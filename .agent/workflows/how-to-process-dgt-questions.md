---
description: Как обработать новые вопросы DGT (скачивание, AI-перевод, загрузка в Supabase)
---

Этот воркфлоу описывает процесс обработки новых DGT вопросов: от парсинга до загрузки в базу данных.

### 1. Парсинг вопросов
Сначала нужно собрать JSON с вопросами с сайта PracticaVial используя браузерный парсер.
Результат сохранить в `data/new-questions.json`.
(Этот шаг пока выполняется полу-автоматически через `scripts/browser-test-parser.js` или вручную).

### 2. Подготовка и AI-перевод
Запустите скрипт для скачивания картинок и генерации объяснений/переводов через Gemini Flash 2.0.

```bash
# Убедитесь, что GEMINI_API_KEY в .env верный
node scripts/digitize-explanations.js
```

Скрипт будет:
1. Читать `data/test-results.json` (или другой входной файл).
2. Скачивать картинки в `data/images`.
3. Генерировать объяснения и переводы.
4. Сохранять прогресс обратно в `data/test-results.json`.

При ошибках скрипт можно перезапускать — он продолжит с места остановки.

### 3. Загрузка картинок в облако
Загрузите локальные картинки в Supabase Storage (бакет `dgt-images`) и обновите ссылки в JSON.

```bash
node scripts/upload-images-to-supabase.js
```

### 4. Импорт в базу данных
Загрузите финальный JSON в таблицу `dgt_questions`.

```bash
# Флаг --yes для авто-подтверждения
node scripts/import-to-supabase.js ./data/test-results.json --yes
```

### 5. Проверка
Зайдите в приложение или админку Supabase и убедитесь, что вопросы появились и работают.
