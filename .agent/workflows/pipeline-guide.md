# 🏭 Автоматический Конвейер Вопросов

## 📋 Полный процесс от парсинга до БД

### Шаг 1: Парсинг вопросов с PracticaVial

1. Открой страницу с результатами теста:
   ```
   https://teorica.practicavial.com/test-results/...
   ```

2. Открой консоль браузера: `F12` → `Console`

3. Скопируй и вставь содержимое файла:
   ```
   scripts/browser-test-parser.js
   ```

4. Нажми `Enter`

5. Сохрани результат:
   ```javascript
   // В консоли браузера:
   copy(JSON.stringify(window.parsedQuestions, null, 2))
   ```

6. Создай файл `data/raw-questions.json` и вставь JSON

---

### Шаг 2: Запуск автоматического конвейера

#### Вариант A: Автоматический режим (без подтверждений)
```bash
node scripts/pipeline.js data/raw-questions.json --auto
```

#### Вариант B: Интерактивный режим (с подтверждением каждого этапа)
```bash
node scripts/pipeline.js data/raw-questions.json
```

---

## 🔄 Что происходит внутри конвейера:

### 1️⃣ Валидация (мгновенно)
- ✅ Проверка существования файла
- ✅ Проверка формата JSON
- ✅ Подсчёт вопросов

### 2️⃣ AI Обработка (~3-6 сек на вопрос)
- 🤖 Перевод на английский и русский
- 🤖 Генерация объяснений на 3 языках
- 🧠 Использование схем (schema_url) для лучших объяснений
- 💾 Автосохранение после каждого вопроса

### 3️⃣ Загрузка изображений (~2-5 сек на изображение)
- 📥 Скачивание изображений с внешних URL
- 📤 Загрузка в Supabase Storage (bucket: dgt-images)
- 🔗 Замена внешних URL на Supabase URL
- 💾 Обновление JSON с новыми ссылками

### 4️⃣ Импорт в БД (~1 сек на 10 вопросов)
- 📦 Батчами по вопросам
- 🔄 Upsert (обновление существующих)
- ✅ Финальная статистика

---

## 🛡️ Безопасность и восстановление:

- ✅ **Автосохранение**: JSON обновляется после каждого вопроса
- ✅ **Можно прервать**: `Ctrl+C` в любой момент
- ✅ **Можно продолжить**: Запусти конвейер снова — он пропустит уже обработанные вопросы
- ✅ **Rate Limit защита**: Автоматические паузы при достижении лимита API

## 🆘 Решение проблем:

### AI Rate Limit (429 ошибка)
```
⏳ Лимит достигнут. Ждем 30 секунд...
```
Конвейер автоматически подождёт и продолжит.

### Некорректный формат JSON
Проверь что JSON содержит массив объектов:
```json
[
  {
    "question": {...},
    "answers": [...]
  }
]
```

### Нет переменных окружения
Убедись что `.env.local` содержит:
```
GEMINI_API_KEY=...
VITE_SUPABASE_URL=...
SUPABASE_SERVICE_ROLE_KEY=...
```

## 💡 Советы:

1. **Маленькими порциями**: Начни с 10-20 вопросов для теста
2. **Проверяй результат**: После импорта зайди в Supabase и проверь данные
3. **Используй dry-run**: Для проверки без реального импорта
4. **Сохраняй бэкапы**: Копируй JSON после AI обработки

## 🎯 Следующие шаги после импорта:
