# 📘 Руководство по регистрации и настройке AdsGram SDK

## 🎯 Цель
Получить доступ к AdsGram SDK для интеграции Rewarded Video рекламы в приложение.

---

## 📋 Пошаговая инструкция

### Шаг 1: Регистрация на AdsGram

1. **Перейдите на официальный сайт:**
   - **Русская версия:** https://adsgram.ai/ru/
   - **Английская версия:** https://adsgram.ai/
   - Нажмите кнопку **"Создать аккаунт"** / **"Sign Up"**

2. **Зарегистрируйтесь как Publisher:**
   - Заполните форму регистрации
   - Выберите тип аккаунта: **"Publisher"** (для монетизации приложения)
   - Подтвердите email (если требуется)

---

### Шаг 2: Создание площадки (приложения)

1. **В кабинете AdsGram найдите раздел:**
   - "Apps" / "Приложения" / "Площадки"
   - "Create New App" / "Создать приложение"

2. **Заполните информацию о приложении:**
   - **App name:** Skily (или ваше название)
   - **Telegram direct link:** `https://t.me/YOUR_BOT_NAME/YOUR_WEBAPP_MENU_BUTTON` 
     - ⚠️ Это должна быть **прямая ссылка** на Web App в боте
     - Формат: `https://t.me/botname/webapp` или `https://t.me/botname/start?startapp=webapp`
     - **НЕ** просто `https://t.me/botname`!
   - **Web app url:** `https://your-deployment-domain.com` (например, Vercel URL)
     - ⚠️ **ВАЖНО:** Это должен быть **реальный веб-URL**, где хостится ваше приложение
     - **НЕ** `t.me` ссылка! Это должен быть HTTPS URL вида `https://skily-app.vercel.app` или ваш домен
     - AdsGram проверяет этот URL, поэтому он должен быть доступен в интернете
   - **Bot ID:** Ваш Telegram Bot ID (можно найти через @userinfobot)
   - **Test platform:** Поставьте галочку для тестирования (опционально)
   - **Категория:** Education / Образование

### ⚠️ Важно про ошибки в форме:

**Ошибка 1: "Enter a valid direct link" (Telegram direct link)**
- Причина: Ссылка должна быть на Web App, а не просто на бота
- Решение: Используйте ссылку вида `https://t.me/botname/webapp` или создайте Menu Button в BotFather

**Ошибка 2: "Links to t.me are not allowed" (Web app url)**
- Причина: AdsGram требует **реальный веб-URL** (где хостится ваше приложение)
- Решение: Используйте URL вашего деплоя (Vercel, Netlify, или ваш домен)
- Примеры правильных URL:
  - ✅ `https://skily-app.vercel.app`
  - ✅ `https://skilyapp.com`
  - ✅ `https://your-app.netlify.app`
  - ❌ `https://t.me/botname` (это НЕ веб-URL!)

**Как получить правильные URL:**

1. **Telegram direct link:**
   - Откройте BotFather → `/setmenubutton` → выберите бота
   - Установите Menu Button с URL вашего Web App
   - Telegram автоматически создаст ссылку вида `https://t.me/botname/webapp`
   - Эту ссылку используйте в "Telegram direct link"

2. **Web app url:**
   - Это URL вашего деплоя на Vercel/Netlify/другом хостинге
   - Если еще не задеплоено - задеплойте сначала на Vercel
   - Или используйте постоянный туннель (но для продакшена лучше Vercel)

3. **Отправьте на модерацию:**
   - AdsGram проверяет качество приложения
   - Обычно одобрение занимает 1-3 дня
   - Если у вас уже есть работающее приложение - одобрят быстрее

---

### Шаг 3: Получение API ключей и Unit IDs

После одобрения приложения вы получите:

1. **API Key:**
   - Используется для авторизации запросов к API
   - Хранится в переменных окружения

2. **Unit ID для Rewarded Video:**
   - **Test Unit ID** - для разработки и тестирования
   - **Production Unit ID** - для продакшена
   - Пример формата: `ca-app-pub-xxxxxxxxxxxxxxxx/xxxxxxxxxx`

3. **Документация SDK:**
   - Инструкции по интеграции
   - Примеры кода
   - API Reference

   ```
   UNIT_ID_TEST=test_unit_id_here
   UNIT_ID_PRODUCTION=production_unit_id_here
   ```

2. **Обновите код:**
   - Раскомментируйте SDK интеграцию в `src/hooks/useRewardedAd.ts`
   - Добавьте Unit IDs в конфигурацию

   - Сначала используйте Test Unit ID

   - После проверки переключитесь на Production Unit ID

## ⚠️ Важные моменты

### Про работу AdsGram:

**Да, AdsGram работает ТОЛЬКО в Telegram Mini App!**

AdsGram - это рекламная сеть, специально разработанная для Telegram Mini Apps. Она:
- ✅ Работает **внутри** Telegram Web App (TMA)
- ✅ Использует Telegram Bot API для показа рекламы
- ✅ Не работает на обычных веб-сайтах (вне Telegram)
- ✅ Не работает в мобильных приложениях (не через Telegram)

**Как это работает:**
1. Пользователь открывает ваше приложение **через Telegram** (как Mini App)
2. В вашем приложении вы показываете рекламу через AdsGram SDK
3. Реклама показывается **внутри** Telegram Web App
4. После просмотра пользователь получает награду (монеты, жизни и т.д.)

**Поэтому:**
- Нужен **Web App URL** - где хостится ваше приложение (Vercel/домен)
- Нужен **Telegram Bot** - через которого открывается Mini App
- AdsGram SDK загружается в ваше веб-приложение, но работает только когда оно открыто через Telegram

### Требования:

1. **Требования к трафику:**
   - AdsGram может требовать минимум пользователей (например, 100+ DAU)
   - Но для стартапов часто делают исключения
   - Укажите, что приложение готовится к запуску

2. **Модерация:**
   - Приложение должно работать и быть доступно по Web App URL
   - Интерфейс должен быть понятным
   - Не должно быть ошибок и багов
   - Web App должен открываться через Telegram

3. **Форматы рекламы:**
   - **Rewarded Video** - за награду (монеты, жизни)
   - Это самый дорогой формат (eCPM $15-30 для Tier-1)
   - Именно этот формат мы интегрируем
