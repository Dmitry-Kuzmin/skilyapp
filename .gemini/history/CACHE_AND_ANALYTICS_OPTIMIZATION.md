# Оптимизация кэша и аналитики

## 📋 Вопросы пользователя

1. **Ротация/очистка кэша**: Есть ли механизм удаления старых данных (например, картинки тестов, которые были пройдены неделю назад)? Кэш может расти бесконечно и забить память телефона.
2. **Кэширование аналитики**: Кэшируются ли события аналитики (какие экраны открывал, куда нажимал)? Отправляются ли они пачкой (batch), когда появится интернет?
3. **Background Sync API**: Возможно стоит добавить для надежной отправки данных.

## ✅ Решения

### 1. Умная ротация кэша (`src/utils/cacheRotation.ts`)

#### Механизм очистки

- **По времени**:
  - Изображения тестов: удаляются через 7 дней
  - Данные тестов: удаляются через 30 дней
- **По размеру**:
  - Максимальный размер кэша: 50 MB
  - Очистка начинается при 80% заполнения (40 MB)
  - Удаляются самые старые элементы
- **Периодическая очистка**:
  - Автоматическая очистка каждые 6 часов
  - Очистка при установке Service Worker

#### Метки времени

Все кэшированные ответы получают метку времени `sw-cache-date` в заголовках:
```javascript
headers.set('sw-cache-date', Date.now().toString());
```

Это позволяет отслеживать возраст кэшированных данных.

#### Функции

- `cleanupOldTestImages()` - удаляет старые изображения тестов (7+ дней)
- `cleanupOldTestData()` - удаляет старые данные тестов (30+ дней)
- `enforceCacheSizeLimit()` - контролирует размер кэша (50 MB)
- `getCacheStats()` - возвращает статистику кэша
- `performCacheRotation()` - запускает полную очистку

### 2. Кэширование и батчинг аналитики (`src/utils/analyticsQueue.ts`)

#### Хранение событий

- **IndexedDB**: События сохраняются в IndexedDB для надежности
- **Лимиты**:
  - Максимум 1000 событий в очереди
  - События старше 30 дней удаляются автоматически
  - Размер батча: 50 событий

#### Отправка событий

- **Батчинг**: События группируются по пользователям и отправляются пачками по 50
- **Автоматическая отправка**:
  - При восстановлении сети (`online` event)
  - При возврате на страницу (`visibilitychange`)
  - Через Background Sync API
- **Retry механизм**:
  - До 5 попыток отправки каждого события
  - Задержка 5 секунд между попытками
  - Старые события удаляются после превышения лимита попыток

#### Интеграция

Все события аналитики теперь проходят через очередь:
```typescript
// src/lib/notification-events.ts
export async function dispatchUserEvent(
  userId: string | null | undefined,
  eventType: string,
  payload: Record<string, any> = {},
  options?: DispatchOptions
): Promise<void> {
  // Всегда добавляем в очередь
  await queueAnalyticsEvent(userId, eventType, payload, options?.overrideTemplateType);
}
```

#### Функции

- `queueAnalyticsEvent()` - добавляет событие в очередь
- `sendBatch()` - отправляет батч событий
- `flushQueue()` - принудительно отправляет все события
- `getQueueSize()` - возвращает размер очереди
- `cleanupOldEvents()` - удаляет старые события

### 3. Background Sync API

#### Регистрация синхронизации

При восстановлении сети автоматически регистрируется Background Sync:
```typescript
if ('serviceWorker' in navigator && 'sync' in ServiceWorkerRegistration.prototype) {
  navigator.serviceWorker.ready.then((registration) => {
    return registration.sync.register('analytics-sync');
  });
}
```

#### Service Worker обработка

В `public/sw.js` добавлен обработчик `sync` события:
```javascript
self.addEventListener('sync', (event) => {
  if (event.tag === 'analytics-sync') {
    // Отправляем сообщение клиенту для синхронизации аналитики
    event.waitUntil(
      self.clients.matchAll().then((clients) => {
        clients.forEach((client) => {
          client.postMessage({ type: 'SYNC_ANALYTICS' });
        });
      })
    );
  }
  
  if (event.tag === 'cache-rotation') {
    // Периодическая ротация кэша
    event.waitUntil(performCacheRotation());
  }
});
```

#### Преимущества

- **Надежность**: События отправляются даже если приложение закрыто
- **Автоматика**: Не требует действий от пользователя
- **Эффективность**: Батчинг снижает нагрузку на сервер

## ✅ Результаты

### Кэш

- ✅ Автоматическая очистка старых данных
- ✅ Контроль размера (не растет бесконечно)
- ✅ Метки времени для точной ротации
- ✅ Периодическая очистка каждые 6 часов

### Аналитика

- ✅ Все события кэшируются локально
- ✅ Батчинг для эффективной отправки
- ✅ Background Sync для надежности
- ✅ Retry механизм для надежности
- ✅ Автоматическая очистка старых событий

### Общее

- ✅ **Кэш не растет бесконечно** - автоматическая очистка по времени и размеру
- ✅ **Аналитика не теряется** - все события сохраняются и отправляются при восстановлении связи
- ✅ **Надежность** - Background Sync гарантирует отправку даже при закрытом приложении
- ✅ **Производительность** - батчинг снижает нагрузку на сервер
