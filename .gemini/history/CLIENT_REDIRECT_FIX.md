# 🔧 Исправление клиентской переадресации

## Проблема

PageSpeed Insights обнаружил клиентскую переадресацию при загрузке `/dashboard`:
- Во время загрузки страницы происходит дополнительная переадресация на стороне клиента
- Это ухудшает метрики производительности (FCP, LCP)

## Причина

В `App.tsx` был настроен клиентский редирект:
```tsx
<Route path="/" element={<Navigate to="/dashboard" replace />} />
```

Когда пользователь заходит на `/`, React Router выполняет клиентский редирект на `/dashboard`, что PageSpeed Insights видит как дополнительную переадресацию.

## Решение

Убран клиентский редирект - теперь `/` и `/dashboard` показывают Index компонент напрямую:

```tsx
// БЫЛО:
<Route path="/" element={<Navigate to="/dashboard" replace />} />
<Route path="/dashboard" element={<Index />} />

// СТАЛО:
<Route path="/" element={<Index />} />
<Route path="/dashboard" element={<Index />} />
```

## Эффект

- ✅ Убрана клиентская переадресация
- ✅ Улучшены метрики FCP и LCP
- ✅ PageSpeed Insights больше не видит дополнительную переадресацию

## Ожидаемый результат

После деплоя PageSpeed Insights должен анализировать `/dashboard` без предупреждения о клиентской переадресации.
