# 📦 Информация о сжатии на Vercel

## ✅ Brotli сжатие включено автоматически

Vercel **автоматически использует Brotli** для всех статических файлов (JS, CSS, изображения), если браузер поддерживает.

### Как это работает:

1. **Vercel автоматически определяет** поддержку Brotli через заголовок `Accept-Encoding`
2. Если браузер поддерживает Brotli → используется Brotli (15-20% лучше Gzip)
3. Если браузер не поддерживает Brotli → используется Gzip (fallback)

### Текущая конфигурация:

В `vercel.json` уже настроен правильный заголовок:
```json
{
  "key": "Vary",
  "value": "Accept-Encoding"
}
```

Это позволяет Vercel выбирать оптимальный алгоритм сжатия.

### Проверка:

Чтобы проверить, что Brotli работает:
1. Открой DevTools → Network
2. Проверь заголовок `Content-Encoding: br` (Brotli) или `Content-Encoding: gzip`
3. Современные браузеры (Chrome, Firefox, Safari) поддерживают Brotli

### Результаты сжатия:

Текущие размеры после сжатия (Brotli/Gzip):
- `vendor.js`: 1,172 KB → **~300-350 KB** (Brotli) / **363 KB** (Gzip)
- `index.js`: 302 KB → **~80-90 KB** (Brotli) / **103 KB** (Gzip)
- `index.css`: 471 KB → **~45-50 KB** (Brotli) / **58 KB** (Gzip)

**Вывод:** Brotli уже работает автоматически на Vercel, дополнительная настройка не требуется.
