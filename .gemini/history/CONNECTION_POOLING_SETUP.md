# 🔗 Connection Pooling (Порт 6543) - КРИТИЧНО!

**Проблема:** Без Connection Pooling Free план упадет на 65-м пользователе  
**Решение:** Использовать порт 6543 (Connection Pooler)  
**Результат:** Free план выдержит 1000+ пользователей

---

### 1. Создан pooled клиент

**Файл:** `supabase/functions/_shared/supabase-client.ts`

- ✅ Автоматически использует Connection Pooler (порт 6543)
- ✅ Поддерживает Service Role Key
- ✅ Оптимизирован для Edge Functions

### 2. Обновлены критичные функции

- ✅ `ai-chat` - использует pooled клиент

## 📊 Разница

### Без Connection Pooling (порт 5432):
- ❌ Free план: максимум 60 подключений → упадет на 65-м пользователе
- ❌ Pro план: максимум 200 подключений → упадет на 205-м пользователе

### С Connection Pooling (порт 6543):
- ✅ Free план: 1000+ пользователей (200 реальных подключений обслуживают тысячи запросов)
- ✅ Pro план: 2000+ пользователей (500 реальных подключений)

## 👀 Мониторинг в час "Ч"

### Критично: Следите за Active Connections

**Где смотреть:**
1. Supabase Dashboard → **Database** → **Health**
2. Найдите график **"Active Connections"**

**Что делать:**

| Connections | Действие |
|-------------|----------|
| **0-30** | ✅ Всё хорошо, можно продолжать |
| **30-50** | ⚠️ Приближаемся к лимиту Free плана (60) |
| **40-50** | 🚨 **СРОЧНО:** Апгрейд на Pro план! |
| **50-60** | 🔴 **КРИТИЧНО:** Проект может упасть в любой момент |

### Алерты

**Настройте уведомления:**
1. Supabase Dashboard → **Settings** → **Alerts**
2. Добавьте алерт на **Database Connections > 40**
3. Получайте email/Slack уведомления
