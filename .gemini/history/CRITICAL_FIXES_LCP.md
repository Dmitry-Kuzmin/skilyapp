# 🚨 Критические исправления для LCP и FCP

**Дата:** 5 декабря 2025  
**Проблема:** FCP 4.5s, LCP 5.9s - HTML заблокирован CSS/JS

## ✅ Выполненные исправления

### 1. LCP элемент оптимизирован ✅

**Проблема:**
- Hero section использовал `background-image` для noise.svg
- `fetchpriority="high"` не работает с `background-image` в CSS
- Изображение загружалось без приоритета

**Решение:**
- Заменён `background-image` на `<img>` элемент
- Добавлен `fetchPriority="high"` (React атрибут)
- Добавлен `loading="eager"` (не lazy)
- Добавлен `decoding="async"` (неблокирующее декодирование)
- Добавлен `aria-hidden="true"` (декоративное изображение)

**Файл:** `src/components/dashboard-new/Dashboard.tsx`

**Ожидаемый результат:**
- LCP элемент загружается с высоким приоритетом
- Улучшение LCP на 1-2 секунды

### 2. CSS блокировка (требует проверки)

**Текущая ситуация:**
- `import "./index.css"` в `main.tsx` - синхронный импорт
- Vite инжектирует CSS в `<head>` автоматически
- Критичный CSS уже инлайнен в `index.html` (skeleton)

**Проблема:**
- Если `index.css` большой (>50KB), он блокирует рендеринг
- FCP ждёт загрузки CSS

**Решение (частично реализовано):**
- Критичный CSS уже инлайнен в `<head>` (skeleton loader)
- `optimizeCssLoading` плагин делает неблокирующую загрузку CSS
- НО: нужно проверить, работает ли это на production

**Требуется:**
- Проверить размер `index.css` в production
- Убедиться, что `optimizeCssLoading` работает
- Возможно, нужно инлайнить больше критичного CSS

### 3. JS загрузка (уже оптимизировано)

**Текущая ситуация:**
- `main.tsx` загружается как ES module (`type="module"`)
- ES modules автоматически defer
- Vendor.js и index.js имеют preload (через `optimizeCssLoading`)

**Статус:** ✅ Оптимизировано
