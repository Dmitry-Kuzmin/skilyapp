# 🔑 Объяснение ключей Cryptomus

## 📋 Типы ключей в Cryptomus

В Cryptomus есть несколько типов ключей, каждый для своих целей:

---

## 1️⃣ **CRYPTOMUS_MERCHANT_ID** (ID мерчанта)

### Что это:
- **Не ключ, а идентификатор** вашего бизнес-аккаунта (мерчанта)
- Уникальный ID, который Cryptomus присваивает вашему бизнесу
- Используется для идентификации вашего аккаунта при API запросах

### Где найти:
1. Откройте **Cryptomus Dashboard**
2. Перейдите в **Настройки мерчанта** (Merchant Settings)
3. В разделе **API Integration** найдите **"ID мерчанта"** (Merchant ID)
4. Скопируйте значение (например: `4d822fef-9ba5-4840-a770-b168291864bc`)

### ✅ Пример из вашего аккаунта:
```
CRYPTOMUS_MERCHANT_ID=4d822fef-9ba5-4840-a770-b168291864bc
```

### Как используется в коде:
```typescript
// Отправляется в заголовке запроса
headers: {
  "merchant": cryptomusMerchantId,  // ← Здесь
  "sign": signature,
}
```

---

## 2️⃣ **CRYPTOMUS_PAYMENT_KEY** (Ключ для платежей)

### Что это:
- **Секретный ключ** для создания подписи (signature) запросов
- Используется для **безопасной аутентификации** при создании платежей
- **НЕ путать** с другими типами ключей!

### Где найти:
1. Откройте **Cryptomus Dashboard**
2. Перейдите в **Настройки мерчанта** (Merchant Settings)
3. В разделе **API Integration** найдите **"Платежный API ключ"** (Payment API key)
4. Нажмите на иконку копирования, чтобы скопировать полный ключ
5. ⚠️ **ВАЖНО:** Ключ показывается частично (например: `n49K********RNNK`), но при копировании вы получите полный ключ

### ✅ Пример из вашего аккаунта:
```
CRYPTOMUS_PAYMENT_KEY=n49K...RNNK  (полный ключ при копировании)
```

### ⚠️ ВАЖНО:
- **НЕ используйте** "Пользовательский API-ключ" (User API-key) — он для личного аккаунта
- **НЕ используйте** "Выплатной API ключ" (Payout API key) — он для вывода средств
- Нужен именно **Payment Key** для создания платежей

### Как используется в коде:
```typescript
// Используется для создания подписи запроса
const signature = await createSignature(payloadString, cryptomusPaymentKey);

// Отправляется в заголовке
headers: {
  "merchant": cryptomusMerchantId,
  "sign": signature,  // ← Подпись, созданная из Payment Key
}
```

---

## 3️⃣ Другие ключи (НЕ нужны для нашей интеграции)

### **Выплатной API ключ (Payout API key)**
- **Для чего:** Вывод средств с бизнес-баланса на ваш кошелек
- **Где:** Настройки → API → "Выплатной API ключ"
- **НЕ нужен** для приема платежей

### **Пользовательский API-ключ (User API-key)**
- **Для чего:** Работа с личным аккаунтом и конвертером
- **Где:** Настройки → API → "Пользовательский API-ключ"
- **НЕ нужен** для бизнес-платежей

---

## 🔍 Как найти нужные ключи

### Если не видите Payment Key в разделе API:

1. **Проверьте статус мерчанта:**
   - Мерчант должен быть **активным** (прошел модерацию)
   - Если статус "Модерация" — дождитесь одобрения

2. **Проверьте раздел "Мерчанты":**
   - Перейдите в **Мерчанты** → [Ваш мерчант]
   - Откройте **Настройки** → **API**
   - Там должен быть **Payment Key**

3. **Обратитесь в поддержку:**
   - Если ключа нет, возможно, нужно включить API для мерчанта
   - Напишите в поддержку Cryptomus

## 📝 Пример использования в коде

```typescript
// 1. Получаем ключи из переменных окружения
const cryptomusMerchantId = Deno.env.get("CRYPTOMUS_MERCHANT_ID");
const cryptomusPaymentKey = Deno.env.get("CRYPTOMUS_PAYMENT_KEY");

// 2. Создаем payload для запроса
const payload = {
  amount: "10.00",
  currency: "USD",
  order_id: "order_123",
  // ...
};

// 3. Создаем подпись из Payment Key
const payloadString = JSON.stringify(payload);
const signature = await createSignature(payloadString, cryptomusPaymentKey);

// 4. Отправляем запрос с Merchant ID и подписью
const response = await fetch("https://api.cryptomus.com/v1/payment", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "merchant": cryptomusMerchantId,  // ← ID мерчанта
    "sign": signature,                  // ← Подпись из Payment Key
  },
  body: payloadString,
});
```
