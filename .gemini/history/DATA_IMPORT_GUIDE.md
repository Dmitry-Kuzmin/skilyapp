## 📋 Какие данные нужно импортировать?

### Обязательные данные (для работы приложения):

1. **`questions_new`** - вопросы для тестов
   - Минимум: 10-20 вопросов для каждой темы
   - Формат: Excel файл с колонками (см. `src/utils/importQuestions.ts`)

2. **`answer_options`** - варианты ответов
   - Связаны с `questions_new`
   - Обычно импортируются вместе с вопросами

3. **`subtopics`** - подтемы
   - Связаны с `topics` (которые уже есть)
   - Минимум: 2-3 подтемы для каждой темы

4. **`materials`** - учебные материалы
   - Связаны с `subtopics`
   - Минимум: 1 материал для каждой подтемы

### Опциональные данные:

5. **`road_signs`** - дорожные знаки
   - Для игры с дорожными знаками
   - Формат: CSV или Excel

6. **`language_terms`** - термины
   - Для словаря терминов
   - Формат: CSV или Excel

7. **`user_progress`** - прогресс пользователей
   - Создается автоматически при использовании приложения

## ⚠️ Важные замечания

1. **Порядок импорта важен:**
   - Сначала `topics` (уже есть)
   - Затем `subtopics` (зависят от `topics`)
   - Затем `materials` (зависят от `subtopics`)
   - Затем `questions_new` (зависят от `topics`)
   - Затем `answer_options` (зависят от `questions_new`)

2. **Внешние ключи:**
   - Убедитесь, что все внешние ключи существуют перед импортом
   - Например, `topic_id` в `questions_new` должен ссылаться на существующий `topics.id`

3. **RLS политики:**
   - Импорт через SERVICE_ROLE_KEY обходит RLS
   - Импорт через интерфейс приложения требует прав admin

## 🔧 Прямой доступ к базе данных

**Вопрос:** Могу ли я напрямую вносить изменения в базу Supabase?

**Ответ:** Да, но с ограничениями:

1. **Через Supabase JS клиент (рекомендуется):**
   - Используйте `SERVICE_ROLE_KEY` для полного доступа
   - Создавайте скрипты на Node.js
   - Пример: `scripts/import-data-from-old-db.js`

2. **Через Supabase Dashboard:**
   - SQL Editor: https://supabase.com/dashboard/project/yffjnqegeiorunyvcxkn/sql/new
   - Table Editor: https://supabase.com/dashboard/project/yffjnqegeiorunyvcxkn/editor
   - Можно вручную добавлять/редактировать данные

3. **Через Supabase CLI:**
   - `supabase db push` - для миграций
   - `supabase db reset` - для сброса базы
   - Прямое выполнение SQL через `psql`

4. **Через REST API:**
   - Используйте `SERVICE_ROLE_KEY` для обхода RLS
   - Ограничения: нельзя выполнять произвольный SQL

## 🆘 Если что-то не работает

1. **Проверьте права доступа:**
   - Убедитесь, что используете `SERVICE_ROLE_KEY` для импорта
   - Проверьте RLS политики

2. **Проверьте внешние ключи:**
   - Убедитесь, что все связанные записи существуют
   - Например, `topic_id` должен ссылаться на существующий `topics.id`

3. **Проверьте формат данных:**
   - Убедитесь, что данные соответствуют схеме таблицы
   - Проверьте типы данных (UUID, TEXT, INTEGER и т.д.)

4. **Обратитесь за помощью:**
   - Проверьте логи ошибок
   - Используйте SQL Editor для диагностики
