### **Приоритет 1: Исправление Race Condition в Edge Function** ✅

**Файл:** `supabase/functions/duel-manager/index.ts`

#### Изменения:

1. **Уменьшена задержка с 500ms до 200ms** (строка 2463)
   - Минимизирует окно для race condition
   - 200ms достаточно для commit в БД

2. **Добавлена проверка статуса дуэли ПЕРЕД подсчетом ответов** (строки 2467-2485)
   ```typescript
   // Проверяем статус дуэли ПЕРВЫМ делом
   const { data: currentDuelStatus } = await supabase
     .from('duels')
     .select('status')
     .eq('id', duel_id)
     .single();

   // Если дуэль уже завершена - возвращаем finished: true
   if (currentDuelStatus?.status === 'finished') {
     return { finished: true, reason: 'already_finished_by_opponent' };
   }
   ```

3. **Убрана двойная проверка с задержкой 300ms** (строки 2509-2516)
   - Было: первая проверка + 300ms + вторая проверка
   - Стало: одна проверка (быстрее для пользователя)

4. **Добавлено атомарное обновление статуса** (строки 2571-2606)
   ```typescript
   // КРИТИЧНО: Атомарное обновление через WHERE
   const { data: updateResult } = await supabase
     .from('duels')
     .update({ status: 'finished', finished_at: new Date().toISOString() })
     .eq('id', duel_id)
     .eq('status', 'active') // ← КРИТИЧНО: только если статус еще 'active'
     .select('id, status');

   // Если не обновлено - другой игрок уже завершил
   if (!updateResult || updateResult.length === 0) {
     return { finished: true, reason: 'already_finished_by_opponent_race' };
   }
   ```

#### Эффект:
- ✅ Устранен race condition при одновременном завершении
- ✅ Только один игрок может обновить статус с 'active' на 'finished'
- ✅ Второй игрок получит `finished: true` и сразу перейдет к результатам
- ✅ Добавлено логирование для мониторинга race conditions

### **Приоритет 3: Улучшение обработки ошибок** ✅

**Файл:** `src/components/duel/DuelWaitingReplay.tsx`

#### Изменения:

1. **Разделены try-catch блоки** (строки 88-128)
   - Звук, toast и onDuelFinished обрабатываются отдельно
   - Ошибка в одном не блокирует остальные

2. **Добавлена повторная попытка при ошибке** (строки 115-122)
   ```typescript
   try {
     onDuelFinished();
   } catch (error) {
     // Сбрасываем флаги для повторной попытки
     onDuelFinishedCalledRef.current = false;
     isDuelFinishedRef.current = false;
     
     // Повторная попытка через 2 секунды
     setTimeout(() => {
       if (!onDuelFinishedCalledRef.current) {
         safeCallOnDuelFinished();
       }
     }, 2000);
   }
   ```

3. **Улучшено информирование пользователя**
   - При ошибке показывается toast с сообщением о повторной попытке
   - Пользователь не застрянет на экране ожидания

#### Эффект:
- ✅ Повышена надежность перехода к результатам
- ✅ Автоматическая повторная попытка при ошибке
- ✅ Улучшен UX (пользователь видит что происходит)
