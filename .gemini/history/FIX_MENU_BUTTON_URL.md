# 🔧 Исправление URL для Menu Button

## Проблема
Кнопка "🚀 В БОЙ" открывает GitHub Pages (404 ошибка) вместо приложения.

## Причина
В секретах Supabase установлен неправильный URL (возможно, старый GitHub Pages URL).

## ✅ Решение

### 1. Проверьте текущий URL в секретах

1. Откройте **Supabase Dashboard**
2. Перейдите: **Edge Functions** → **telegram-bot** → **Settings** → **Secrets**
3. Найдите секрет `MINI_APP_URL`
4. Проверьте его значение

### 2. Установите правильный URL

**Правильный URL для продакшена:**
```
https://skilyapp.com
```

**Если используете другой деплой:**
```
https://sdadim-dgt-prep.vercel.app
```

### 3. Обновите секрет

В Supabase Dashboard:
1. **Edge Functions** → **telegram-bot** → **Settings** → **Secrets**
2. Найдите `MINI_APP_URL`
3. Если его нет — нажмите **Add new secret**
4. Установите:
   - **Name:** `MINI_APP_URL`
   - **Value:** `https://skilyapp.com`
5. Сохраните

### 4. Перезапустите функцию (опционально)

После обновления секрета можно перезапустить функцию:
```bash
supabase functions deploy telegram-bot
```

### 5. Проверьте логи

После обновления секрета отправьте `/start` боту и проверьте логи:
- Должно быть: `[setupMenuButtonForUser] 📋 URL: https://skilyapp.com`
- Если видите другой URL — секрет не обновился

## ⚠️ Важно

- URL должен начинаться с `https://`
- URL должен быть доступен (не 404)
- После изменения секрета может потребоваться несколько секунд для применения

## 📋 Чеклист

- [ ] URL установлен на `https://skilyapp.com` (или правильный продакшен URL)
- [ ] Отправлен `/start` боту после обновления
- [ ] Проверены логи — URL правильный
- [ ] Кнопка открывает приложение без 404
