# 📚 Что такое Git Commit (простыми словами)

## 🎯 Что такое "коммитить"?

**Git Commit** = Сохранить изменения в истории проекта

Это как сделать "снимок" (snapshot) вашего кода на данный момент.

## 📝 Аналогия

Представь что ты пишешь сочинение:
- **Несохраненный файл** = изменения в памяти (еще не "закоммичены")
- **Сохранить файл** = `git commit` (сохранить в историю)
- **История файлов** = git история коммитов

## 🔧 Как это работает

### 1. **Сделал изменения в коде**
```bash
# Изменил файл RaceGame.tsx
# Но изменения еще не сохранены в git
```

### 2. **Добавить изменения (Stage)**
```bash
git add src/pages/games/RaceGame.tsx
# Или все файлы:
git add .
```
**Это значит:** "Хочу сохранить эти изменения"

### 3. **Коммитить (Commit)**
```bash
git commit -m "feat: добавил круговой таймер"
```
**Это значит:** "Сохранить изменения с описанием"

### 4. **Отправить на сервер (Push)**
```bash
git push
```
**Это значит:** "Отправить сохраненные изменения на сервер, чтобы другие могли их увидеть"

## 🎯 Простой пример

```bash
# 1. Сделал изменения
# Изменил файл: src/pages/games/RaceGame.tsx

# 2. Проверить что изменилось
git status

# 3. Добавить к коммиту
git add src/pages/games/RaceGame.tsx

# 4. Коммитить с описанием
git commit -m "feat: добавил круговой таймер в игру Гонка"

# 5. Отправить на сервер
git push
```

## ✅ Зачем это нужно?

1. **История изменений** - видно что и когда изменилось
2. **Откат изменений** - можно вернуться к предыдущей версии
3. **Работа в команде** - другие видят твои изменения
4. **Бэкап** - код сохранен в git репозитории

## 🚨 Важно!

**Коммитить нужно:**
- После завершения задачи
- Перед переключением на другую задачу
- Перед закрытием чата с агентом
- Перед перезапуском приложения

**Коммитить НЕ нужно:**
- После каждого маленького изменения (можно накопиться)
- Если код не работает (сначала исправь)

## 📋 Чеклист перед коммитом

- [ ] Код работает
- [ ] Нет ошибок
- [ ] Понятное описание коммита
- [ ] Все нужные файлы добавлены

## 💡 Примеры хороших коммитов

```
✅ Хорошо:
git commit -m "feat: добавил игру Четыре варианта"
git commit -m "fix: исправил баг с таймером"
git commit -m "refactor: переработал UI игры Гонка"

❌ Плохо:
git commit -m "изменения"
git commit -m "fix"
git commit -m "asdf"
```
