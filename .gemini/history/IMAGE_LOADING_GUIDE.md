# Руководство по организации изображений в тестах

## 📋 Обзор

Это руководство объясняет, как правильно организовать загрузку и отображение изображений в тестах приложения.

## ✅ Формат путей в базе данных

### Правильные форматы:

1. **Имя файла (рекомендуется):**
   ```
   1110.jpeg
   ```
   - Файл должен находиться в корне bucket `questions`
   - Самый простой и чистый вариант

2. **Путь с подпапками:**
   ```
   Image_test/Public/1110.jpeg
   ```
   - Файл находится в подпапке `Image_test/Public/` внутри bucket `questions`
   - Используйте, если файлы организованы по папкам

3. **Полный URL (если изображение из внешнего источника):**
   ```
   https://example.com/image.jpg
   ```
   - Используется как есть, без обработки

### ❌ Неправильные форматы:

- `questions/1110.jpeg` - не указывайте имя bucket в пути
- `/1110.jpeg` - лишний слэш в начале (автоматически удаляется, но лучше не использовать)
- `1110.jpeg/` - лишний слэш в конце (автоматически удаляется)

## 🗂️ Организация файлов в Supabase Storage

### Структура bucket `questions`:

```
questions/
├── 1110.jpeg
├── 1111.jpeg
├── 1112.jpeg
└── Image_test/
    └── Public/
        ├── 1110.jpeg
        ├── 1111.jpeg
        └── ...
```

### Рекомендации:

1. **Для простоты:** Храните все изображения в корне bucket
   - Путь в БД: `1110.jpeg`
   - Легко управлять и находить файлы

2. **Для организации:** Используйте подпапки по темам или типам
   - Путь в БД: `Image_test/Public/1110.jpeg`
   - Удобно для большого количества файлов

3. **Убедитесь, что bucket публичный:**
   - В настройках Supabase Storage установите bucket `questions` как публичный
   - Иначе изображения не будут доступны для загрузки

## 🔧 Как работает система

### Функция `getImageUrl`:

```typescript
getImageUrl(imageUrl: string, bucket: string = 'questions'): string | null
```

**Что делает:**
1. Проверяет, является ли `imageUrl` полным URL (начинается с `http://` или `https://`)
2. Если да - возвращает как есть
3. Если нет - нормализует путь и создает публичный URL из Supabase Storage
4. Возвращает полный URL вида: `https://[project].supabase.co/storage/v1/object/public/questions/1110.jpeg`

### Компонент `QuestionImageComponent`:

**Особенности:**
- Автоматически обрабатывает загрузку изображений
- Показывает состояние загрузки
- Скрывает изображение при ошибке (не показывает битую картинку)
- Логирует ошибки в консоль для отладки

## 📝 Примеры использования


## 🐛 Решение проблем

### Изображение не отображается:

1. **Проверьте путь в базе данных:**
   - Убедитесь, что путь указан правильно (например, `1110.jpeg`)
   - Проверьте, нет ли лишних пробелов

2. **Проверьте существование файла:**
   - Откройте Supabase Storage → bucket `questions`
   - Убедитесь, что файл существует по указанному пути

3. **Проверьте публичность bucket:**
   - В настройках bucket должно быть включено "Public bucket"
   - Иначе изображения не будут доступны

4. **Проверьте консоль браузера:**
   - Откройте DevTools (F12) → Console
   - Ищите сообщения об ошибках загрузки изображений

### Все вопросы используют одно изображение:

**Проблема:** В Google Sheets все строки имеют одинаковый `image_url` (например, `1110.jpeg`)

**Решение:**
1. Если это ошибка - обновите `image_url` для каждого вопроса на уникальное значение
2. Если это правильно - убедитесь, что файл существует в Storage

## ✅ Чеклист при добавлении изображений

- [ ] Файл загружен в Supabase Storage bucket `questions`
- [ ] Путь в базе данных указан правильно (например, `1110.jpeg`)
- [ ] Bucket настроен как публичный
- [ ] Имя файла соответствует пути в БД
- [ ] Файл имеет правильный формат (JPEG, PNG, WebP)
- [ ] Размер файла не превышает 10MB (рекомендуется < 1MB)
- [ ] Изображение отображается при тестировании

## ❓ Частые вопросы

**Q: Могу ли я использовать разные bucket'ы для разных типов контента?**  
A: Да, укажите bucket в функции: `getImageUrl(imageUrl, 'road_signs')`

**Q: Что делать, если файл не найден?**  
A: Компонент автоматически скроет изображение. Проверьте путь в БД и существование файла в Storage.

**Q: Можно ли использовать внешние URL?**  
A: Да, просто укажите полный URL в поле `image_url` в базе данных.

**Q: Как оптимизировать загрузку изображений?**  
A: Используйте форматы WebP или сжатые JPEG, оптимизируйте размер файлов (< 500KB), используйте lazy loading (уже реализовано).
