# Решение: Supabase на лендинге

**Проблема:** На лендинге используется Supabase для:
1. **Реферальная программа** - загрузка информации о реферере (`profiles`)
2. **Партнерская программа** - загрузка информации о партнере (`partners`)

**Вопрос:** Если вынести Supabase в lazy providers, эта функциональность сломается.

---

## ✅ РЕШЕНИЕ: Условная загрузка Supabase

### Стратегия:

1. **Минимальный Supabase client** для лендинга (только чтение, без React Query)
2. **Lazy load полных провайдеров** только для приложения (`/app/*`)
3. **Условная загрузка** - Supabase загружается только если нужен (есть `referral_code` или `partner_code`)

---

## 📋 ПЛАН РЕАЛИЗАЦИИ

### Вариант 1: Условная загрузка Supabase (РЕКОМЕНДУЕТСЯ)

**Идея:** Загружать Supabase только когда он действительно нужен (есть referral/partner код).

**Преимущества:**
- ✅ Лендинг без referral/partner кода остается легким
- ✅ Функциональность не ломается
- ✅ Минимальные изменения в коде

**Реализация:**

1. **Создать легкий Supabase client для лендинга:**
   ```typescript
   // src/integrations/supabase/landingClient.ts
   import { createClient } from '@supabase/supabase-js';
   
   // Минимальный client только для чтения
   export const landingSupabase = createClient(
     import.meta.env.VITE_SUPABASE_URL,
     import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY
   );
   ```

2. **Lazy load Supabase в Landing.tsx:**
   ```typescript
   // src/pages/Landing.tsx
   const Landing = () => {
     const [supabase, setSupabase] = useState<any>(null);
     
     useEffect(() => {
       // Загружаем Supabase только если нужен
       const referralCode = sessionStorage.getItem('referral_code');
       const partnerCode = sessionStorage.getItem('partner_code');
       
       if (referralCode || partnerCode) {
         // Lazy load Supabase только когда нужен
         import('@/integrations/supabase/landingClient').then(({ landingSupabase }) => {
           setSupabase(landingSupabase);
         });
       }
     }, []);
     
     // Используем supabase только если загружен
     if (supabase) {
       // Запросы к БД
     }
   };
   ```

3. **Полные провайдеры для приложения:**
   - `AppProviders` с Query/Supabase для `/app/*`
   - Лендинг использует только легкий client

**Ожидаемый результат:**
- Лендинг без referral/partner: ~150-200 KB (без Supabase)
- Лендинг с referral/partner: ~250-300 KB (с минимальным Supabase)
- Приложение: ~417 KB (с полными провайдерами)

### Вариант 2: Минимальный Supabase в initial bundle

**Идея:** Оставить минимальный Supabase client в initial bundle, но вынести React Query.

**Преимущества:**
- ✅ Проще реализовать
- ✅ Функциональность гарантированно работает
- ⚠️ Supabase все еще в initial bundle (~50-80 KB)

**Реализация:**

1. **Supabase client остается в initial bundle:**
   ```typescript
   // src/integrations/supabase/client.ts - остается как есть
   export const supabase = createClient(...);
   ```

2. **React Query выносится в lazy providers:**
   - Только Query/Supabase провайдеры в lazy
   - Supabase client доступен сразу

**Ожидаемый результат:**
- Initial bundle: 417 KB → 300-350 KB (уменьшение за счет Query)
- PSI Mobile: 66 → 70-75 (небольшой прирост)

---

### Вариант 3: Разделение лендинга (если нужен 90+ PSI)

**Идея:** Создать отдельный легкий лендинг без Supabase, реферальная/партнерская логика на отдельной странице.

**Преимущества:**
- ✅ Максимальный PSI (85-90+)
- ✅ Чистое разделение
- ⚠️ Требует больше работы

**Реализация:**

1. **Легкий лендинг `/`** - без Supabase, только статический контент
2. **Реферальная страница `/join/:code`** - с Supabase (уже есть `InviteLanding.tsx`)
3. **Партнерская страница `/partner/:code`** - с Supabase

**Ожидаемый результат:**
- Лендинг: ~150 KB, PSI 85-90+
- Реферальная/партнерская: ~300 KB, PSI 70-75
