# 🎯 ПЛАН УСТРАНЕНИЯ LONG TASKS

**Цель:** Достичь 60 FPS, Long Tasks < 50ms, плавный интерфейс

---

## 📊 АНАЛИЗ ПРОБЛЕМЫ:

### Найденные источники Long Tasks:

1. **NotificationsPanel** (критично):
   - Рендерит до 20+ уведомлений без виртуализации
   - Framer Motion анимации на каждом элементе
   - Тяжелые вычисления `formatDistanceToNow` для каждого элемента
   - Нет мемоизации рендера элементов

2. **Framer Motion** (76 файлов):
   - `AnimatePresence` на больших списках
   - Отсутствие `LayoutGroup` для оптимизации
   - Тяжелые анимации при каждом рендере

3. **Dashboard** (Index.tsx):
   - Множество компонентов рендерятся одновременно
   - Нет chunking для тяжелых вычислений

---

## 🔧 РЕШЕНИЯ:

### 1. Виртуализация NotificationsPanel ⚡

**Проблема:** Рендеринг 20+ элементов блокирует UI на 200-300ms

**Решение:**
- Установить `@tanstack/react-virtual` (легче чем react-window)
- Виртуализировать только видимые элементы (viewport)
- Мемоизировать каждый элемент списка

**Ожидаемый результат:** Long Task < 50ms даже при 100+ уведомлениях

---

### 2. Оптимизация Framer Motion 🎬

**Проблема:** Анимации блокируют main thread

**Решение:**
- Заменить `AnimatePresence` на `LayoutGroup` где возможно
- Упростить анимации (opacity вместо transform где возможно)
- Использовать `will-change` CSS для GPU acceleration
- Отключить анимации на мобильных (prefers-reduced-motion)

**Ожидаемый результат:** Плавные 60 FPS анимации

---

### 3. Chunking тяжелых вычислений 🧩

**Проблема:** Тяжелые вычисления блокируют рендер

**Решение:**
- Использовать `useTransition` для неблокирующих обновлений
- Разбить большие списки на чанки (batch rendering)
- Использовать `requestIdleCallback` для фоновых задач

**Ожидаемый результат:** UI остается отзывчивым

---

### 4. Мемоизация и оптимизация рендеров 💾

**Проблема:** Лишние ре-рендеры компонентов

**Решение:**
- `React.memo` для тяжелых компонентов
- `useMemo` для дорогих вычислений
- `useCallback` для обработчиков событий
- Оптимизировать `formatDistanceToNow` (кеширование)

**Ожидаемый результат:** Меньше ре-рендеров, быстрее рендер
