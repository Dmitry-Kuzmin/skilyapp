# 🔒 Как сделать репозиторий приватным

## ✅ Самый простой способ

### Шаг 1: Перейти в настройки репозитория

1. Открыть: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/settings
2. Прокрутить вниз до раздела **"Danger Zone"**

### Шаг 2: Изменить видимость

1. Найти кнопку **"Change repository visibility"**
2. Нажать **"Change visibility"**
3. Выбрать **"Make private"**
4. Подтвердить, введя название репозитория: `Dmitry-Kuzmin/sdadim-dgt-prep`
5. Нажать **"I understand, change repository visibility"**

## ⚠️ ВАЖНО: Что нужно знать

### ✅ Плюсы приватного репозитория:
- 🔐 Код виден только вам (и тем, кому вы дадите доступ)
- 🛡️ Секреты не видны посторонним
- 🔒 Полный контроль над доступом
- ✅ Можно оставить ключи в коде (но всё равно лучше через env)

### ⚠️ Минусы:
- 🔍 Репозиторий не индексируется поисковиками
- 👥 Нужно вручную добавлять коллабораторов
- 💰 Если используете GitHub Actions, может быть лимит на бесплатные минуты (но обычно достаточно)

### 🚨 КРИТИЧНО: Ротация ключей всё равно нужна!

**Даже после закрытия репозитория** нужно ротировать ключи, которые были в публичном репозитории:
- Service Role Key
- Telegram Bot Token (если был)
- Любые другие секреты

**Почему?** Потому что они уже были видны, пока репозиторий был публичным. Кто-то мог их скопировать.

## 📋 Чеклист после закрытия

- [ ] Репозиторий сделан приватным
- [ ] Ротированы все ключи, которые были в публичном коде:
  - [ ] Supabase Service Role Key
  - [ ] Telegram Bot Token (если был)
  - [ ] Stripe Secret Key (если был)
  - [ ] Другие API ключи
- [ ] Обновлены ключи в:
  - [ ] GitHub Secrets
  - [ ] Vercel Environment Variables
  - [ ] Supabase Edge Functions Secrets
  - [ ] Локальный `.env.local`
- [ ] Проверено, что всё работает после ротации

## 🔄 Если нужно вернуть публичный доступ

1. Settings → Danger Zone → Change repository visibility
2. Выбрать "Make public"
3. **НО**: перед этим убедиться, что все секреты удалены из кода!

---

**Рекомендация**: Сделать приватным + ротировать ключи = максимальная безопасность! 🔐
