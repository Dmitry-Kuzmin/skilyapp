### Почему Score не изменился?

Оптимизация `PremiumCard` и `DuelPassInfo` через `React.memo` и `useCallback` предотвращает лишние ре-рендеры, но не влияет напрямую на начальную загрузку страницы, которая измеряется PageSpeed Insights.

**PageSpeed Insights измеряет:**
- Время загрузки ресурсов (FCP, LCP)
- Размеры bundle'ов
- Работу в основном потоке (TBT)
- Layout shifts (CLS)

**React.memo и useCallback помогают:**
- Уменьшить работу в основном потоке во время взаимодействия
- Предотвратить лишние ре-рендеры после загрузки
- Улучшить производительность во время использования приложения

**Но не влияют на:**
- Размеры JavaScript bundle'ов
- Время загрузки ресурсов
- Начальный рендеринг

## 🎯 Следующие шаги для достижения 90+

### Приоритет 1: Уменьшить размеры bundle'ов

**Проблема:** Большие JavaScript bundle'ы замедляют загрузку

**Текущие размеры:**
- `vendor.js`: 1.1M (342 KB gzip)
- `CSS`: 470.73 KB (57.96 KB gzip)

**Решения:**
1. **Дополнительное code splitting:**
   - Выделить тяжелые библиотеки (framer-motion, date-fns, zustand)
   - Lazy load некритических компонентов

2. **Tree shaking:**
   - Убедиться, что неиспользуемый код удаляется
   - Проверить импорты (использовать named imports вместо default)

3. **Оптимизация CSS:**
   - Удалить unused CSS
   - Разделить CSS на критические и некритические части

**Ожидаемый эффект:** -100-200ms FCP, +5-10 пунктов Score

### Приоритет 2: Уменьшить работу в основном потоке (TBT)

**Проблема:** JavaScript блокирует основной поток на 3.9 сек

**Решения:**
1. **Lazy load некритических компонентов:**
   - Компоненты, которые не видны сразу (модалки, панели)
   - Тяжелые библиотеки (framer-motion, если используется не везде)

2. **Оптимизация рендеринга:**
   - Продолжить оптимизацию других тяжелых компонентов
   - Использовать `React.memo` для компонентов, которые редко меняются

3. **Отложить некритический JavaScript:**
   - Загружать аналитику после загрузки страницы
   - Отложить инициализацию некритических функций

**Ожидаемый эффект:** -500-1000ms TBT, +10-15 пунктов Score

### Приоритет 3: Оптимизация изображений

**Решения:**
1. **Конвертация в WebP:**
   - Все изображения должны быть в WebP формате
   - Использовать `srcset` для responsive images

2. **Оптимизация размеров:**
   - Убедиться, что все изображения имеют правильные размеры
   - Использовать `width` и `height` атрибуты (уже сделано)

3. **Lazy loading:**
   - Убедиться, что все некритические изображения используют `loading="lazy"`

**Ожидаемый эффект:** Улучшение LCP на 200-300ms, +2-3 пункта Score

### Приоритет 4: Оптимизация загрузки ресурсов

**Решения:**
1. **Preload критических ресурсов:**
   - Убедиться, что все критические ресурсы preload'ятся
   - Проверить порядок загрузки

2. **Preconnect к внешним доменам:**
   - Добавить preconnect для внешних ресурсов (если используются)

3. **DNS prefetch:**
   - Добавить dns-prefetch для внешних доменов

**Ожидаемый эффект:** Улучшение FCP на 100-200ms, +2-3 пункта Score
