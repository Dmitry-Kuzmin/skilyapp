# 🚗 Настройка базы данных ПДД России

## 📋 План действий

### 1. Структура базы данных
- Используем существующую таблицу `questions_new` 
- Добавляем поле `country` для разделения DGT (Испания) и ПДД (Россия)
- Создаём таблицу `pdd_russia_tickets` для билетов

### 2. Источники данных

#### Вариант А: Парсинг официального сайта ГИБДД
- **URL**: https://гибдд.рф/exam или https://www.gibdd.ru/exam/
- **Плюсы**: Официальные данные, всегда актуально
- **Минусы**: Нужен парсер, может быть защита от парсинга

#### Вариант Б: GitHub репозитории (если найдём)
- Ищем открытые репозитории с JSON/CSV данными
- Проверяем лицензию и актуальность

#### Вариант В: Ручной импорт
- Скачиваем данные с pdd-bilet.ru или pdd-tests.ru
- Конвертируем в нужный формат
- Импортируем через скрипт

### 3. Структура данных ПДД России

```typescript
interface PDDQuestion {
  ticket_number: number;      // Номер билета (1-40)
  question_number: number;    // Номер вопроса в билете (1-20)
  question_text: string;      // Текст вопроса
  image_url?: string;         // URL изображения (если есть)
  answers: {
    text: string;
    is_correct: boolean;
  }[];
  explanation?: string;        // Пояснение к ответу
  topic?: string;             // Тема (например, "Дорожные знаки")
  category?: string;          // Категория (A, B, C, D, M)
}
```

### 4. Следующие шаги

1. ✅ Создать миграцию для добавления поля `country` в `questions_new`

3. ✅ Создать парсер для сайта ГИБДД

5. ✅ Протестировать импорт

- Для коммерческого использования может потребоваться лицензия
