# Roadmap to Performance Score 90+

**Текущий статус:** Performance 64 (было 59) ✅  
**Цель:** Performance 90+  
**Отчёт:** https://pagespeed.web.dev/analysis/https-skilyapp-com/gafr8yu13y?form_factor=mobile

## 🎯 Честная оценка текущего состояния

### ✅ Победы:
1. **TBT: 80ms** - блестящий результат! Сайт "летает" когда загрузится
2. **Vendor разбит** - убрали 1MB мусора из стартовой загрузки
3. **CLS: 0** - отлично, нет layout shift
4. **Performance: 64** - улучшение на 5 пунктов

### ⚠️ Проблемы:
1. **FCP/LCP: ~4.6s** - всё ещё долго
2. **Speed Index: 6.6s** - без изменений
3. **index.js: 670KB** - всё ещё большой
4. **Unused JavaScript: 350 KiB** - можно убрать
5. **Unused CSS: 45 KiB** - можно убрать

## 🚀 План действий (Roadmap to 90)

### Шаг 1: Preload критических chunks ✅ (В ПРОЦЕССЕ)

**Что нужно:**
- Preload для `vendor.js` (React, Supabase, TanStack)
- Preload для `index.js` (основной код)
- `fetchpriority="high"` для критических ресурсов

**Статус:** Добавлено в `vite.config.ts` плагин `optimizeCssLoading`

### Шаг 2: Оптимизация index.js (670KB → ~400-500KB)

**Проблема:** Основной код всё ещё большой

**Решения:**
1. **Lazy-load больше компонентов:**
   - `Dashboard` компонент
   - `ExamReadinessWidget`
   - `LumiSearchWidget`
   - Тяжёлые модалки

2. **Убрать unused JavaScript (350 KiB):**
   - Запустить `npm run build:analyze`
   - Найти неиспользуемые импорты
   - Удалить мёртвый код

3. **Оптимизировать импорты:**
   - Проверить tree-shaking
   - Убедиться что все импорты named (не wildcard)

### Шаг 3: Оптимизация CSS (убрать 45 KiB unused)

**Решения:**
1. **PurgeCSS/Tailwind:**
   - Убедиться что Tailwind правильно удаляет неиспользуемые классы
   - Проверить `tailwind.config.ts`

2. **Code splitting CSS:**
   - Уже включено: `cssCodeSplit: true`
   - Проверить что работает правильно

### Шаг 4: Оптимизация LCP элемента

**Что нужно:**
1. **Определить LCP элемент:**
   - В PageSpeed Insights нажать на "Largest Contentful Paint element"
   - Это картинка? Текст? Компонент?

2. **Если это картинка:**
   - Добавить `fetchpriority="high"`
   - Конвертировать в WebP
   - Добавить `loading="eager"` (не lazy)
   - Оптимизировать размер

3. **Если это текст:**
   - Убедиться что шрифты preload
   - Использовать `font-display: swap`
   - Оптимизировать размер шрифтов

### Шаг 5: Оптимизация шрифтов

**Что проверить:**
1. Формат шрифтов (woff2 предпочтительно)
2. Preload критических шрифтов
3. `font-display: swap` для быстрого отображения
4. Subset шрифтов (только нужные символы)

### Шаг 6: Оптимизация изображений

**Что нужно:**
1. Конвертировать все изображения в WebP
2. Добавить fallback для старых браузеров
3. Lazy loading для изображений ниже fold
4. Responsive images (`srcset`)
