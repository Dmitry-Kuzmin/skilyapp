# Server Time Offset - Защита от неверного времени на устройстве

## ❓ Проблема

**Вопрос**: "Если у пользователя на планшете сбиты часы (спешат на час или год), его ответы могут случайно перезатереть правильные ответы или заблокировать их изменение. Мы используем сырое локальное время (Date.now()) или корректируем его относительно времени сервера?"

## ✅ Решение: Server Time Offset

**Ответ**: Используем **Server Time Offset** - вычисляем разницу между временем устройства и сервером при первом подключении и корректируем все timestamp.

### Как это работает:

1. **При первом подключении**:
   - Получаем время сервера через HTTP заголовок `Date`
   - Вычисляем offset: `offset = serverTime - clientTime`
   - Сохраняем offset в localStorage

2. **При сохранении ответа**:
   - Используем серверное время: `serverTime = Date.now() + offset`

3. **При сравнении timestamp**:
   - Конвертируем локальное время в серверное: `localTimeServer = localTime + offset`
   - Сравниваем с серверным временем корректно

### Пример:

```
Устройство пользователя: время 12:00 (но реально на сервере 11:00)
Offset = -3600000 ms (устройство спешит на 1 час)

Ответ сохранен локально:
  localTime = 12:00 (Date.now())
  serverTime = 12:00 + (-1 час) = 11:00 ✅

При синхронизации:
  serverTimestamp = 11:30 (реальное серверное время)
  localTimestampServer = 12:00 + (-1 час) = 11:00
  
  Сравнение: 11:30 > 11:00 ✅
  Правильно: используем серверный ответ (он новее)
```

## 📋 Реализация

### Файл: `src/utils/serverTime.ts`

**Основные функции**:

1. **`initServerTime()`** - Инициализация при старте приложения
2. **`getServerTime()`** - Получить текущее серверное время
3. **`toServerTime(localTimestamp)`** - Конвертировать локальное в серверное
4. **`toLocalTime(serverTimestamp)`** - Конвертировать серверное в локальное
5. **`getServerTimeOffset()`** - Получить offset
6. **`forceSyncServerTime()`** - Принудительная синхронизация

### Интеграция в TestSession:

```typescript
// При сохранении ответа
timestamp: await getServerTime() // Используем server time

// При сравнении в testSync.ts
const localTimestampServer = toServerTime(localTimestamp);
if (serverTimestamp > localTimestampServer) {
  // Серверный новее
}
```

## 🔐 Гарантии

### ✅ Корректное сравнение

- Все timestamp конвертируются в серверное время
- Сравнение всегда корректно, даже если часы сбиты
- Offset кэшируется на 1 час (пересинхронизация)

### ✅ Защита от неверного времени

- Устройство спешит на 1 час → offset корректирует
- Устройство отстает на год → offset корректирует
- Часы изменены во время теста → пересинхронизация через 1 час

### ✅ Производительность

- Offset кэшируется в localStorage
- Пересинхронизация только раз в час
- Минимальная задержка при получении server time

## 📊 Метод получения Server Time

### Используем HTTP заголовок `Date`

```typescript
const response = await fetch(`${supabaseUrl}/rest/v1/`, {
  method: 'HEAD',
});
const serverTime = new Date(response.headers.get('Date')).getTime();
```

**Преимущества**:
- ✅ Нет дополнительных запросов (используем существующий endpoint)
- ✅ Точность высокая (учет latency)
- ✅ Работает везде (не требует специального API)

### Latency Compensation

```typescript
const startTime = Date.now();
const response = await fetch(...);
const endTime = Date.now();
const latency = (endTime - startTime) / 2;
const adjustedServerTime = serverTime + latency;
```

## 🔄 Автоматическая пересинхронизация

- **Валидность offset**: 1 час
- **Автоматическая проверка**: при каждом использовании
- **Принудительная синхронизация**: если offset устарел
