# Таймаут для Network First стратегии

## 🔴 Проблема

Network First стратегия без таймаута может привести к долгому ожиданию, если сервер медленно отвечает или не отвечает вообще. Приложение будет висеть и ждать ответа сети до победного.

## ✅ Решение

Добавлен таймаут 3 секунды для всех Network First запросов:

### 1. Данные тестов (questions, answer_options, tests, topics)

```javascript
// КРИТИЧНО: Network First стратегия с таймаутом 3 секунды
Promise.race([
  // Пытаемся загрузить из сети
  fetch(request),
  // Таймаут: если сеть не отвечает за 3 секунды - переключаемся на кэш
  new Promise((resolve) => {
    setTimeout(() => resolve(null), 3000);
  }),
])
```

**Результат:** Если сервер не отвечает за 3 секунды - мгновенно переключаемся на кэш.

### 2. HTML запросы

```javascript
// Для HTML используем Network First с таймаутом 3 секунды
const response = await Promise.race([
  fetch(request, { cache: 'no-cache' }),
  new Promise((_, reject) => {
    setTimeout(() => reject(new Error('Network timeout')), 3000);
  }),
]);
```

**Результат:** HTML страницы также переключаются на кэш при таймауте.

## 📊 Как это работает

1. **Запрос отправляется в сеть**
2. **Одновременно запускается таймер на 3 секунды**
3. **Если сеть отвечает быстро (< 3 сек) - используем ответ из сети**
4. **Если таймаут (3 сек) - мгновенно переключаемся на кэш**
5. **Если нет в кэше - возвращаем ошибку 503**

## 🎯 Преимущества

- ✅ **Мгновенное переключение на кэш** при медленном сервере
- ✅ **Нет зависаний** приложения
- ✅ **Быстрая работа** даже при плохом интернете
- ✅ **Надежность** - всегда есть fallback на кэш

## ⚙️ Настройка таймаута

Таймаут установлен на **3 секунды** - это оптимальный баланс между:
- Быстрым переключением на кэш
- Даванием серверу времени ответить

Можно изменить в `public/sw.js`:
```javascript
setTimeout(() => resolve(null), 3000); // Измените 3000 на нужное значение (мс)
```
