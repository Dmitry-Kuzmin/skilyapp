# План внедрения гибридного SSR для SEO

## 🎯 Цель

Внедрить SSR для публичных страниц (Blog, Article, Landing) для улучшения SEO и привлечения органического трафика, сохранив SPA для приложения.

## 📋 Страницы для SSR

### ✅ SSR (публичные, SEO-критичные):
- `/` (Landing) — лендинг для неавторизованных
- `/blog` — список статей
- `/article/:slug` — отдельные статьи

### ❌ SPA (приложение, производительность):
- `/dashboard` — дашборд
- `/tests`, `/games` — тесты и игры
- `/learning` — обучение
- Все остальные страницы приложения

## 🛠️ Варианты реализации

### Вариант 1: Vite SSR (рекомендуется) ⭐

**Плюсы:**
- Минимальные изменения в коде
- Используем существующий Vite
- Гибкая настройка (SSR только для нужных роутов)
- Быстрая разработка

**Минусы:**
- Нужно настроить сервер для SSR (Vercel Edge Functions или Node.js)
- Немного сложнее деплой

**Время:** 1-2 недели

**Шаги:**
1. Установить `vite-plugin-ssr` или настроить SSR в Vite
2. Создать `server.tsx` для SSR рендеринга
3. Настроить роутинг для SSR (только публичные страницы)
4. Обновить `vercel.json` для SSR роутов
5. Протестировать и задеплоить

### Вариант 2: Next.js

**Плюсы:**
- Полный контроль над SSR
- Отличная SEO поддержка из коробки
- Много документации и примеров

**Минусы:**
- Нужно переписать роутинг
- Больше изменений в коде
- Может быть сложнее с Telegram WebApp API

**Время:** 2-3 недели

### Вариант 3: Pre-rendering (компромисс)

**Плюсы:**
- Самый простой вариант
- Не нужен сервер для SSR
- Статические HTML файлы

**Минусы:**
- Не работает для динамического контента
- Нужно перегенерировать при изменении контента
- Меньше гибкости

**Время:** 1 неделя

## 🔧 Технические детали

### Структура SSR роутов:

```typescript
// server.tsx
export async function render(url: string) {
  // Определяем, нужен ли SSR для этого роута
  const publicRoutes = ['/', '/blog', '/article'];
  const needsSSR = publicRoutes.some(route => url.startsWith(route));
  
  if (needsSSR) {
    // SSR рендеринг
    const html = await renderToString(<App />);
    return html;
  }
  
  // Для остальных роутов возвращаем SPA
  return null;
}
```

### Vercel конфигурация:

```json
{
  "functions": {
    "server.tsx": {
      "runtime": "nodejs18.x"
    }
  },
  "rewrites": [
    {
      "source": "/(blog|article|)",
      "destination": "/server.tsx"
    }
  ]
}
```

## ⚠️ Важные моменты

1. **Telegram WebApp API** — работает только в браузере, нужно проверять `typeof window !== 'undefined'`
2. **Hydration** — нужно убедиться, что SSR и клиентский рендеринг совпадают
3. **Мета-теги** — должны быть в SSR HTML для SEO
4. **Structured Data** — должны быть в SSR HTML
5. **Изображения** — нужно оптимизировать для SSR (WebP, lazy loading)
