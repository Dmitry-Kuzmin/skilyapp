# Руководство по тестированию системы сезонов Duel Pass

## 📋 Чек-лист проверки

### 1. Проверка базы данных

Убедитесь, что миграция применена:
```sql
-- Проверить существование таблиц
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN (
  'duel_pass_seasons',
  'duel_pass_season_rewards',
  'user_season_progress',
  'season_challenges',
  'user_challenge_progress'
);

-- Проверить активный сезон
SELECT * FROM duel_pass_seasons WHERE is_active = true;

-- Проверить награды сезона
SELECT COUNT(*) FROM duel_pass_season_rewards WHERE season_id = 1;
-- Должно быть 30 уровней
```

### 3. Тестирование в приложении

#### Шаг 1: Запустите приложение
```bash
npm run dev
```

#### Шаг 2: Войдите в систему
- Откройте http://localhost:8080
- Войдите через Telegram или веб-авторизацию

#### Шаг 3: Проверьте главную страницу
1. **Проверьте компонент Duel Pass**
   - Должен отображаться прогресс сезона
   - Должны быть видны уровни (1-30)
   - Должен быть виден текущий сезон "Operación Asfalto"

2. **Проверьте компонент Season Challenges**
   - Должен отображаться виджет с челленджами
   - Должны быть видны Daily, Weekly и Season челленджи
   - Прогресс должен быть 0/целевое значение

#### Шаг 4: Протестируйте начисление SP

**Тест 1: Ежедневный вход**
1. Заберите ежедневный бонус
2. Проверьте в консоли браузера (F12 → Console):
   - Должен быть вызов `season-sp` с `source_type: 'daily_login'`
   - Должен быть вызов `season-challenges-track`
3. Обновите страницу
4. Проверьте, что SP увеличились в Duel Pass
5. Проверьте, что прогресс челленджей обновился

**Тест 2: Прохождение теста**
1. Пройдите любой тест
2. После завершения проверьте:
   - В консоли должен быть вызов `season-sp` с `source_type: 'test_completed'` или `'test_perfect'`
   - Должен быть вызов `season-challenges-track`
3. Вернитесь на главную страницу
4. Проверьте, что:
   - SP увеличились
   - Прогресс челленджей обновился (например, "3 идеальных теста" или "150 вопросов")

**Тест 3: Дуэль**
1. Сыграйте дуэль
2. После завершения проверьте:
   - В консоли должен быть вызов `season-sp` с `source_type: 'duel_win'` или `'duel_lose'`
   - Должен быть вызов `season-challenges-track`
3. Проверьте обновление прогресса

#### Шаг 5: Проверьте завершение челленджа

**Тест: Завершение Daily челленджа**
1. Выполните условие челленджа (например, пройдите 3 теста без ошибок)
2. После завершения проверьте:
   - В консоли должен быть вызов `season-challenges-reward`
   - Должны быть начислены SP и монеты
3. Обновите страницу
4. Проверьте, что:
   - Челлендж отмечен как завершенный (зеленая галочка)
   - Награды отображаются
   - SP и монеты увеличились

#### Шаг 6: Проверьте Premium Pass бонус

1. Если у вас Premium или Trial:
   - При начислении SP должен применяться бонус +20%
   - Например: базовое начисление 25 SP → с Premium 30 SP

### 4. Проверка в Supabase Dashboard

#### Проверка прогресса пользователя
```sql
-- Проверить прогресс пользователя в сезоне
SELECT 
  usp.*,
  s.name_ru as season_name
FROM user_season_progress usp
JOIN duel_pass_seasons s ON s.id = usp.season_id
WHERE usp.user_id = 'YOUR_USER_ID';
```

#### Проверка прогресса челленджей
```sql
-- Проверить прогресс по челленджам
SELECT 
  ucp.*,
  sc.title_ru,
  sc.target_value,
  sc.reward_sp,
  sc.reward_coins
FROM user_challenge_progress ucp
JOIN season_challenges sc ON sc.id = ucp.challenge_id
WHERE ucp.user_id = 'YOUR_USER_ID';
```

#### Проверка транзакций
```sql
-- Проверить транзакции с SP и монетами
SELECT * FROM transactions 
WHERE user_id = 'YOUR_USER_ID' 
AND metadata->>'source' = 'season_challenge'
ORDER BY created_at DESC
LIMIT 10;
```

### 5. Возможные проблемы и решения

#### Проблема: Челленджи не отображаются
**Решение:**
- Проверьте, что миграция применена
- Проверьте, что создан активный сезон
- Проверьте консоль браузера на ошибки

#### Проблема: SP не начисляются
**Решение:**
- Проверьте логи Edge Functions в Supabase Dashboard
- Проверьте, что функции развернуты
- Проверьте консоль браузера на ошибки вызовов

#### Проблема: Прогресс челленджей не обновляется
**Решение:**
- Проверьте логи `season-challenges-track` функции
- Убедитесь, что `source_type` соответствует `target_type` в челлендже
- Проверьте, что `metadata` передается правильно

#### Проблема: Награды не начисляются
**Решение:**
- Проверьте логи `season-challenges-reward` функции
- Убедитесь, что челлендж действительно завершен (progress >= target_value)
- Проверьте, что `season-sp` и RPC `increment_profile_value` отрабатывают без ошибок
