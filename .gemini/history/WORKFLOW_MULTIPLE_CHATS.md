# 🎯 Руководство по работе с несколькими чатами агентов

## ⚠️ Проблема
Когда работаешь в нескольких чатах одновременно, легко потерять контроль над изменениями и сломать код.

## ✅ Решение: Правильный Workflow

### 1. **Один чат = одна задача**
```
✅ ПРАВИЛЬНО:
- Чат #1: Работа над игрой "Гонка"
- Чат #2: Работа над игрой "Четыре варианта"
- Чат #3: Исправление багов

❌ НЕПРАВИЛЬНО:
- Чат #1 и #2 одновременно работают над одной игрой
```

### 2. **Всегда коммить изменения перед переключением**
```bash
# Перед переключением на другой чат:
git add .
git commit -m "feat: что-то сделал"
git push
```

### 3. **Проверяй статус перед началом работы**
```bash
# В начале каждого чата:
git status          # Проверить что изменилось
git pull            # Получить последние изменения
git log --oneline -5  # Посмотреть последние коммиты
```

### 4. **Используй ветки для разных задач**
```bash
# Создай ветку для каждой задачи:
git checkout -b feature/four-variants-game
git checkout -b fix/race-game-bug
git checkout -b refactor/ui-improvements

# Работай в ветке, коммить, пушить:
git add .
git commit -m "feat: add four variants game"
git push -u origin feature/four-variants-game
```

### 5. **Правила для каждого чата**

#### 📝 В начале работы в новом чате:
1. **Проверь статус:**
   ```bash
   git status
   git pull
   ```

2. **Сообщи агенту:**
   - На какой задаче ты работаешь
   - Есть ли незакоммиченные изменения
   - Какие файлы уже изменены

3. **Попроси агента проверить:**
   - Нет ли конфликтов
   - Какие файлы будут изменены

#### ✅ В конце работы в чате:
1. **Закоммить все изменения:**
   ```bash
   git add .
   git commit -m "feat: описание изменений"
   git push
   ```

2. **Сообщи в следующем чате:**
   - Что было сделано
   - Какие файлы изменены
   - Коммит хеш для проверки

### 6. **Шаблон сообщения для нового чата**

```
Привет! Я работаю над [ОПИСАНИЕ ЗАДАЧИ].

Перед началом работы:
1. Проверь git status - есть ли незакоммиченные изменения?
2. Проверь git log - какие последние коммиты?
3. Убедись что нет конфликтов с другими задачами

Файлы которые могут быть изменены:
- [список файлов]

Можешь начать работу?
```

### 7. **Что делать если есть конфликты**

```bash
# Если видишь конфликты:
git status                    # Посмотреть конфликты
git pull                      # Попробовать получить изменения
# Если конфликты:
git merge --abort             # Отменить мердж
# Или решить конфликты вручную
```

### 8. **Чеклист перед началом работы в чате**

- [ ] `git status` - проверен статус
- [ ] `git pull` - получены последние изменения
- [ ] Сообщено агенту о текущей задаче
- [ ] Сообщено о файлах которые будут изменены
- [ ] Агент подтвердил что нет конфликтов

### 9. **Чеклист перед завершением чата**

- [ ] Все изменения закоммичены
- [ ] Изменения запушены в репозиторий
- [ ] Сообщено в следующий чат о сделанных изменениях
- [ ] Указан коммит хеш для проверки

### 10. **Полезные команды**

```bash
# Посмотреть что изменилось в файле:
git diff src/pages/games/RaceGame.tsx

# Посмотреть последние коммиты:
git log --oneline -10

# Посмотреть какие файлы изменены:
git status --short

# Отменить изменения в файле:
git restore src/pages/games/RaceGame.tsx

# Посмотреть кто и что изменил:
git blame src/pages/games/RaceGame.tsx
```

## 🎯 Резюме

**Главное правило:** Всегда коммить и пушить перед переключением на другой чат. Это гарантирует что:
- Другие чаты видят последние изменения
- Нет потери работы
- Легко отследить что было сделано
- Нет конфликтов

**Шаблон работы:**
1. Начало чата → `git pull` → проверка статуса
2. Работа → изменения
3. Конец чата → `git add .` → `git commit` → `git push`
4. Следующий чат → сообщить о коммите → повторить
