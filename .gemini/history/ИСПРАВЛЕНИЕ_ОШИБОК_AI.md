# 🔧 Исправление орфографических ошибок AI

## Проблема

AI делает орфографические ошибки в ответах на русском языке.

**Причина:**
- Модель Llama-3.3-70b хуже работает с русским языком
- Системный промпт не акцентировал внимание на грамотности

## Решение

### 1. Изменён приоритет моделей
Теперь используется **Mixtral** (лучше для русского):
```
1. mixtral-8x7b-32768        // Лучшая для русского
2. llama-3.3-70b-versatile   // Резерв
3. llama-3.1-8b-instant      // Быстрая
```

### 2. Улучшен системный промпт
Добавлен акцент на грамотность:
```
КРИТИЧЕСКИ ВАЖНО:
- Пиши ГРАМОТНО, БЕЗ ОРФОГРАФИЧЕСКИХ ОШИБОК
- Проверяй каждое слово перед отправкой
- Используй правильную пунктуацию
- Всегда пиши букву "ё" там, где она нужна
```

## Что нужно сделать

### Шаг 1: Обнови код в Supabase

1. Открой Supabase Dashboard → Edge Functions → ai-chat → Code
2. Выдели весь код (Ctrl+A или Cmd+A)
3. Скопируй обновлённый код из `AI_CHAT_CODE.txt`
4. Вставь в редактор (Ctrl+V или Cmd+V)
5. Нажми **"Deploy updates"**

### Шаг 2: Проверь работу

1. Перезагрузи страницу приложения
2. Задай вопрос AI помощнику
3. Должно быть меньше ошибок

### Шаг 3: Проверь логи

В логах должно быть:
```
[AI Chat] Trying Groq API with model: mixtral-8x7b-32768...
[AI Chat] ✅ Groq API success with model: mixtral-8x7b-32768
```

## Дополнительные варианты

### Вариант 1: Попробуй Gemini (лучше для русского)

Gemini часто работает лучше с русским языком. Добавь секрет `LOVABLE_API_KEY`:
1. Открой Supabase Dashboard → Settings → Edge Functions → Secrets
2. Добавь секрет `LOVABLE_API_KEY` (если есть)
3. Функция автоматически переключится на Gemini, если Groq не подходит

### Вариант 2: Используй только Mixtral

Если хочешь использовать только Mixtral, измени в коде:
```typescript
const models = [
  'mixtral-8x7b-32768',  // Только Mixtral
];
```

### Вариант 3: Добавь температуру 0.3

Для более точных и грамотных ответов можно снизить температуру:
```typescript
temperature: 0.3,  // Было 0.7
```

Это сделает ответы более консистентными, но менее креативными.

После изменений:

## Если ошибки сохраняются

1. **Добавь LOVABLE_API_KEY** — Gemini лучше для русского
2. **Сделай fallback на Gemini обязательным** — если Groq делает ошибки
3. **Снизь температуру до 0.3** — для более точных ответов
4. **Добавь постобработку** — проверка орфографии перед отправкой (сложнее)
