## ✅ Что исправлено

### 1. Извлечение ключевых терминов

**Было:**
```typescript
const knowledge = await searchDGTKnowledge(lastUserMessage, topicNumber);
// Искал по полному вопросу: "Вопрос: En un estrechamiento sin señales..."
```

**Стало:**
```typescript
const extractKeyTerms = (text: string): string => {
  // Извлекаем испанские термины: estrechamiento, prioridad, señal...
  const spanishTerms = text.match(/\b(estrechamiento|prioridad|señal...)\w*/gi);
  
  // Извлекаем русские термины: сужение, приоритет, знак...
  const russianTerms = text.match(/\b(сужени[ея]|приоритет...)\w*/gi);
  
  return [...spanishTerms, ...russianTerms].join(' ');
};

const searchQuery = extractKeyTerms(lastUserMessage);
// Теперь ищет: "estrechamiento prioridad señal"
```

**Результат:**
- ✅ Поиск по **ключевым словам**, а не по всему вопросу
- ✅ Извлекает **испанские термины** (estrechamiento, autopista, etc.)
- ✅ Извлекает **русские термины** (сужение, приоритет, etc.)

### 2. Поиск по двум языкам (SQL)

**Создана новая миграция:** `20251110_update_search_function.sql`

**Было:**
```sql
to_tsvector('russian', dk.content) @@ plainto_tsquery('russian', search_query)
-- Искал ТОЛЬКО по-русски
```

**Стало:**
```sql
-- Поиск по русскому
to_tsvector('russian', dk.content) @@ plainto_tsquery('russian', search_query)
OR
-- Поиск по испанскому
to_tsvector('spanish', dk.content) @@ plainto_tsquery('spanish', search_query)
OR
-- Простой поиск (универсальный)
to_tsvector('simple', dk.content) @@ plainto_tsquery('simple', search_query)
OR
-- Fallback: поиск подстроки
dk.content ILIKE '%' || search_query || '%'
```

**Результат:**
- ✅ Находит по **испанским терминам** (estrechamiento)
- ✅ Находит по **русским терминам** (сужение)
- ✅ Fallback на **простой поиск**
- ✅ **Больше результатов** (3 → 5)
