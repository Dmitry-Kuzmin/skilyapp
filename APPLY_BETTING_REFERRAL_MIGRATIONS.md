# Применение миграций для системы ставок и рефералов

## Важно!

Эти миграции добавляют две новые фичи:
1. **Дуэли на ставки** - играйте на монеты с друзьями
2. **Реферальная система** - приглашайте друзей и зарабатывайте

## Как применить миграции

### Вариант 1: Через Supabase SQL Editor (Рекомендуется)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Выберите ваш проект
3. Перейдите в **SQL Editor**
4. Скопируйте и выполните содержимое файла `supabase/migrations/20251111100000_add_duel_betting_system.sql`
5. Скопируйте и выполните содержимое файла `supabase/migrations/20251111100001_add_referral_system.sql`

### Вариант 2: Через Supabase CLI

```bash
npx supabase db push
```

(Но может быть конфликт с существующими миграциями - тогда используйте Вариант 1)

## Что добавляют миграции

### Миграция 1: Система ставок
- Добавляет поля `bet_amount`, `bet_type`, `commission_taken`, `rematch_pot` в таблицу `duels`
- Создает таблицу `duel_transactions` для отслеживания всех операций с монетами
- Добавляет функцию `process_duel_payout` для автоматического расчета выигрыша
- Комиссия 10% с каждой дуэли на ставки
- При ничьей ставки переносятся на реванш

### Миграция 2: Реферальная система
- Добавляет поля `referral_code`, `referred_by`, `total_referrals` в таблицу `profiles`
- Создает таблицу `referrals` для отслеживания приглашенных друзей
- Добавляет функцию `generate_referral_code` для генерации уникальных кодов
- Добавляет функцию `check_referral_milestone` - автоматически начисляет +100 монет когда друг зарабатывает 50
- Генерирует реферальные коды для всех существующих пользователей
- Триггер на обновление монет для автоматического начисления наград

## После применения миграций

1. **Перезапустите приложение** - `npm run dev`
2. **Проверьте реферальный код** - откройте `/referrals`
3. **Протестируйте ставки** - создайте дуэль с ставкой
4. **Поделитесь реферальной ссылкой** - пригласите друга

## Проблемы?

Если миграции не применились, проверьте:
1. Есть ли у вас права на изменение схемы БД
2. Нет ли конфликтов с существующими таблицами
3. Логи в Supabase Dashboard → Database → Logs
