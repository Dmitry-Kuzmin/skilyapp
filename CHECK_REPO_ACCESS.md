# 🔍 Как проверить, кто просматривал или скачивал репозиторий

## ⚠️ Ограничения GitHub

**К сожалению, GitHub НЕ показывает:**
- ❌ Кто конкретно просматривал репозиторий
- ❌ Кто скачивал код (клонировал репозиторий)
- ❌ IP-адреса посетителей
- ❌ Детальную статистику просмотров

**GitHub показывает только:**
- ✅ Общую статистику просмотров (без имён)
- ✅ Количество звёзд, форков, клонов
- ✅ Активность в Insights (но без персональных данных)

---

## 📊 Что можно проверить

### 1. Общая статистика (Insights)

1. Перейти: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/graphs
2. Раздел **"Traffic"**:
   - Количество просмотров страницы репозитория
   - Количество уникальных посетителей
   - Количество клонов (но без имён)
   - Популярные страницы

**Ограничение:** Данные только за последние 14 дней, без персональной информации.

### 2. Количество форков

1. Перейти: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/network/members
2. Увидеть список форков (если есть)

**Что это даёт:** Можете увидеть, кто форкнул репозиторий, но не факт, что они скачали код.

### 3. Количество звёзд

1. Перейти: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/stargazers
2. Увидеть список тех, кто поставил звезду

**Что это даёт:** Люди, которые заинтересовались проектом, но не факт, что они скачали код.

### 4. История коммитов (если были изменения)

1. Перейти: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/commits
2. Проверить, не было ли подозрительных коммитов

---

## 🔍 Альтернативные способы проверки

### 1. Проверка логов Supabase

Если кто-то использовал Service Role Key из вашего репозитория:

1. Перейти: https://supabase.com/dashboard/project/yffjnqegeiorunyvcxkn/logs
2. Проверить логи доступа:
   - **API Logs** - запросы к API
   - **Auth Logs** - попытки входа
   - **Database Logs** - запросы к БД

**Что искать:**
- Подозрительные запросы с незнакомых IP
- Массовые запросы
- Запросы в нерабочее время

### 2. Проверка логов Telegram Bot

Если кто-то использовал Telegram Bot Token:

1. Проверить логи бота (если есть доступ)
2. Проверить количество пользователей
3. Проверить подозрительную активность

### 3. Проверка Stripe (если был ключ)

1. Перейти: https://dashboard.stripe.com/logs
2. Проверить подозрительные транзакции или запросы

---

## 🚨 КРИТИЧНО: Что делать независимо от проверки

### ⚠️ Ротация ключей обязательна!

**Даже если вы не видите активности, нужно ротировать ключи:**

1. **Service Role Key** - был в публичном репозитории
2. **Telegram Bot Token** - если был в коде
3. **Stripe Secret Key** - если был в коде
4. **Другие API ключи**

**Почему:**
- Кто-то мог скачать код через `git clone` (не видно в статистике)
- Кто-то мог скопировать ключи из веб-интерфейса GitHub
- Боты могли проиндексировать репозиторий
- Ключи могли попасть в кэши поисковиков

## 📋 План действий

### Шаг 1: Проверить доступную статистику

1. ✅ Открыть Insights: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/graphs
2. ✅ Проверить Traffic (последние 14 дней)
3. ✅ Проверить форки и звёзды
4. ✅ Проверить историю коммитов

### Шаг 2: Проверить логи сервисов

1. ✅ Supabase Logs (API, Auth, Database)
2. ✅ Telegram Bot логи (если доступны)
3. ✅ Stripe Logs (если был ключ)

### Шаг 3: Ротация ключей (ОБЯЗАТЕЛЬНО!)

**Независимо от результатов проверки:**

1. ✅ Ротировать Service Role Key
2. ✅ Ротировать Telegram Bot Token (если был)
3. ✅ Ротировать Stripe Secret Key (если был)
4. ✅ Обновить ключи в:
   - GitHub Secrets
   - Vercel Environment Variables
   - Supabase Edge Functions Secrets
   - Локальный `.env.local`

### Шаг 4: Мониторинг после ротации

1. ✅ Настроить алерты в Supabase (если возможно)
2. ✅ Регулярно проверять логи
3. ✅ Мониторить подозрительную активность

---

## 🛡️ Дополнительная защита

### 1. Настроить алерты в Supabase

- Уведомления о подозрительных запросах
- Лимиты на количество запросов
- Мониторинг необычной активности

### 2. Использовать IP whitelist (если возможно)

- Ограничить доступ к API только с ваших IP
- Настроить firewall правила

### 3. Регулярный аудит

- Еженедельно проверять логи
- Проверять количество пользователей
- Мониторить использование ресурсов

## ✅ Вывод

**Что можно проверить:**
- ✅ Общую статистику просмотров (без имён)
- ✅ Количество форков и звёзд
- ✅ Логи сервисов (Supabase, Telegram, Stripe)

**Что нельзя проверить:**
- ❌ Конкретных людей, которые просматривали
- ❌ Кто скачал код через `git clone`
- ❌ IP-адреса посетителей

**Главное:**
- 🔐 Ротация ключей обязательна независимо от проверки
- 🛡️ Лучше перестраховаться и ротировать все ключи
- 📊 Регулярно мониторить логи сервисов

---

**Рекомендация:** Проверьте доступную статистику, но в любом случае ротируйте ключи. Безопасность важнее, чем точное знание, кто мог получить доступ! 🔒
