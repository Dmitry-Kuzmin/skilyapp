# План исправления критических проблем

## 💣 Мина №1: Service Worker и Access Token

**Проблема**: Service Worker не имеет доступа к localStorage, где Supabase хранит токен. Токен может протухнуть пока приложение закрыто.

**Решение**: 
1. **Безопасный вариант**: Отложить отправку до открытия приложения (если токен протух)
2. **Альтернатива**: Дублировать токен в IndexedDB при каждом обновлении
3. **Проверка**: При 401 ошибке - отложить отправку до открытия приложения

## 💣 Мина №2: Дедупликация (Идемпотентность)

**Проблема**: При обрыве связи может быть отправлен дубль события.

**Решение**:
1. ✅ Уникальный `event.id` уже генерируется на клиенте
2. ⚠️ Нужно проверить: проверяет ли сервер дубликаты по `event.id`
3. Добавить проверку на сервере перед записью

## 💣 Мина №3: iOS beforeunload

**Проблема**: Apple не рекомендует `beforeunload` для iOS.

**Решение**: Заменить на `pagehide` с проверкой `persisted`.
