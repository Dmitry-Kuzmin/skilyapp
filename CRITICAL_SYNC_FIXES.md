# Критические исправления синхронизации

## Вопрос 1: Background Sync - Service Worker должен отправлять сам

**Проблема**: Если приложение закрыто, `postMessage` не сработает.

**Решение**: Service Worker должен сам отправлять аналитику через fetch, а не будить клиент.

### Текущая (неправильная) реализация:
```javascript
self.addEventListener('sync', (event) => {
  if (event.tag === 'analytics-sync') {
    // НЕПРАВИЛЬНО: Будим клиент
    self.clients.matchAll().then((clients) => {
      clients.forEach((client) => {
        client.postMessage({ type: 'SYNC_ANALYTICS' });
      });
    });
  }
});
```

### Правильная реализация:
```javascript
self.addEventListener('sync', (event) => {
  if (event.tag === 'analytics-sync') {
    event.waitUntil(
      // ПРАВИЛЬНО: Service Worker сам отправляет
      sendAnalyticsBatchFromIndexedDB()
    );
  }
});
```

## Вопрос 2: iOS Safari не поддерживает Background Sync

**Проблема**: iOS Safari не поддерживает Background Sync API.

**Решение**: Fallback через `visibilitychange` + `Page Visibility API`.

### Стратегия:
1. **Android/Desktop**: Background Sync API (работает когда приложение закрыто)
2. **iOS Safari**: `visibilitychange` + `beforeunload` (работает при сворачивании)
3. **Все браузеры**: Отправка при восстановлении сети (`online` event)

## Вопрос 3: Лимит кэша 50MB маловат

**Проблема**: 50MB = ~250 картинок = ~8 билетов. Мало для активных студентов.

**Решение**: Увеличить до 200MB (современные телефоны легко потянут).

## Вопрос 4: Приоритеты событий

**Проблема**: При переполнении очереди удаляются старые события без учета важности.

**Решение**: Система приоритетов:
- **HIGH**: Результат экзамена, оплата, важные достижения
- **MEDIUM**: Прогресс теста, дуэли
- **LOW**: Клики, просмотры, навигация

Важные события защищены от удаления.

## Вопрос 5: Защита данных активного теста

**Проблема**: Данные активного теста могут быть удалены при очистке кэша.

**Решение**: 
- Помечать данные активного теста
- Не удалять их при очистке
- Удалять только после завершения теста

## Дополнительные вопросы:

1. **Дедупликация на сервере**: Нужна проверка
2. **Retry для разных ошибок**: 400 vs 500 - разная стратегия
3. **Мониторинг**: Нужны метрики для кэша/очереди
4. **GDPR**: Возможность удаления данных пользователем
5. **Fallback IndexedDB**: Уже есть localStorage
