# 🌐 Подтверждение домена в Cryptomus

## 📋 Что это?

Cryptomus требует подтверждения домена для безопасности. Это гарантирует, что вы владеете доменом, с которого будут приниматься платежи.

---

## 🔧 Как подтвердить домен

Cryptomus предлагает **два способа** подтверждения домена:

### 📄 Способ 1: Через HTML файл (РЕКОМЕНДУЕТСЯ - проще и быстрее)

#### Шаг 1: Скачайте HTML файл

1. В личном кабинете Cryptomus найдите раздел "Подтверждение домена"
2. Выберите способ "Загрузить HTML-файл"
3. Скачайте файл (например: `cryptomus_4d822fef.html`)

#### Шаг 2: Загрузите файл в корневую папку сайта

**Для Vite/React проекта:**

1. Скопируйте файл в папку `public/` вашего проекта
   ```bash
   cp ~/Downloads/cryptomus_4d822fef.html public/
   ```

2. Файл уже должен быть в `public/cryptomus_4d822fef.html` ✅

3. Задеплойте проект на Vercel/Netlify:
   ```bash
   git add public/cryptomus_4d822fef.html
   git commit -m "Add Cryptomus domain verification file"
   git push
   ```

4. После деплоя файл будет доступен по адресу:
   ```
   https://skilyapp.com/cryptomus_4d822fef.html
   ```

#### Шаг 3: Проверьте доступность

1. Откройте в браузере: `https://skilyapp.com/cryptomus_4d822fef.html`
2. Должен отобразиться текст: `cryptomus=4d822fef`

#### Шаг 4: Подтвердите в Cryptomus

1. Вернитесь в личный кабинет Cryptomus
2. Нажмите кнопку **"Проверить"** или **"Подтвердить"**
3. Cryptomus проверит доступность файла
4. Если всё правильно — домен будет подтвержден ✅

---

### 📝 Способ 2: Через DNS TXT запись (альтернатива)

#### Шаг 1: Получите код подтверждения

1. В личном кабинете Cryptomus найдите раздел "Подтверждение домена"
2. Выберите способ "Добавить TXT запись в DNS"
3. Скопируйте код подтверждения (например: `cryptomus=4d822fef`)
   - Ваш код будет уникальным, не используйте пример!

#### Шаг 2: Добавьте TXT запись в DNS

#### Вариант А: Если домен на Vercel/Netlify

1. Откройте настройки домена в Vercel/Netlify
2. Перейдите в раздел DNS
3. Добавьте новую TXT запись:
   - **Тип**: TXT
   - **Имя**: `@` (для корневого домена) или `www` (для поддомена)
   - **Значение**: `cryptomus=ваш_код` (например: `cryptomus=4d822fef`)
4. Сохраните изменения

#### Вариант Б: Если домен на другом хостинге

1. Войдите в панель управления вашего хостинга/регистратора домена
2. Найдите раздел "DNS" или "Управление DNS"
3. Добавьте новую TXT запись:
   - **Тип**: TXT
   - **Имя (Host)**: `@` или оставьте пустым (для корневого домена)
   - **Значение (Value)**: `cryptomus=ваш_код`
   - **TTL**: 3600 (или оставьте по умолчанию)
4. Сохраните изменения

#### Вариант В: Если используете Cloudflare

1. Войдите в Cloudflare Dashboard
2. Выберите ваш домен
3. Перейдите в "DNS" → "Records"
4. Нажмите "Add record"
5. Заполните:
   - **Type**: TXT
   - **Name**: `@` (для корневого домена)
   - **Content**: `cryptomus=ваш_код`
   - **TTL**: Auto
6. Нажмите "Save"

---

## ⏱️ Шаг 3: Дождитесь распространения DNS

- Обычно занимает **5-30 минут**
- Может занять до **24 часов** в редких случаях
- Проверить можно через онлайн-инструменты:
  - [MXToolbox](https://mxtoolbox.com/TXTLookup.aspx)
  - [DNS Checker](https://dnschecker.org/)

### Как проверить вручную:

```bash
# В терминале (macOS/Linux)
dig TXT yourdomain.com

# Или
nslookup -type=TXT yourdomain.com
```

Должна появиться запись с `cryptomus=ваш_код`

---

## ✅ Шаг 4: Подтвердите в Cryptomus

1. Вернитесь в личный кабинет Cryptomus
2. Нажмите кнопку **"Проверить"** или **"Подтвердить"**
3. Если всё правильно — домен будет подтвержден ✅
4. Если ошибка — подождите еще немного и попробуйте снова

## 🔍 Примеры для разных провайдеров

## ⚠️ Частые ошибки

### Ошибка: "Домен не подтвержден"

**Причины:**
- DNS запись еще не распространилась (подождите 30-60 минут)
- Неправильный формат записи (должно быть `cryptomus=код`, без пробелов)
- Неправильное имя записи (должно быть `@` для корневого домена)

**Решение:**
1. Проверьте запись через `dig TXT yourdomain.com`
2. Убедитесь, что формат правильный
3. Подождите еще немного и попробуйте снова

### Ошибка: "Запись не найдена"

**Причины:**
- Запись еще не добавлена
- Неправильное имя домена

**Решение:**
1. Убедитесь, что запись сохранена в DNS
2. Проверьте, что используете правильный домен (без `www` если добавили для `@`)
