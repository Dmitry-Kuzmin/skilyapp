# 🧪 Тестирование Cryptomus платежей

## 📋 Важная информация

**Cryptomus НЕ имеет полноценной тестовой среды** как Stripe (нет тестовых карт или sandbox режима).

Однако можно протестировать:
1. ✅ Создание платежа (получение URL для оплаты)
2. ✅ Webhook обработку (через тестовый webhook)
3. ✅ Полный цикл с минимальной суммой

---

## 🧪 Способ 1: Тестирование создания платежа

### Шаг 1: Создайте тестовый платеж

1. Откройте приложение
2. Попробуйте купить Premium или монеты
3. Выберите метод оплаты "Крипто"
4. Должна открыться страница оплаты Cryptomus

### Шаг 2: Проверьте URL

URL должен быть вида:
```
https://pay.cryptomus.com/pay/...
```

Если URL открывается — создание платежа работает! ✅

### Шаг 3: Проверьте логи

В Supabase Dashboard → Edge Functions → cryptomus-payment → Logs:
- Должно быть: `[cryptomus-payment] Processing purchase`
- Не должно быть ошибок

---

## 🧪 Способ 2: Тестирование Webhook

Cryptomus предоставляет возможность тестировать webhook без реального платежа.

### Шаг 1: Отправьте тестовый webhook

Используйте API Cryptomus для отправки тестового webhook:

```bash
curl -X POST https://api.cryptomus.com/v1/test-webhook/wallet \
  -H "Content-Type: application/json" \
  -H "merchant: 4d822fef-9ba5-4840-a770-b168291864bc" \
  -H "sign: ВАША_ПОДПИСЬ" \
  -d '{
    "url_callback": "https://yffjnqegeiorunyvcxkn.supabase.co/functions/v1/cryptomus-webhook",
    "currency": "USDT",
    "network": "TRC20",
    "order_id": "test_order_123",
    "uuid": "ВАШ_UUID_КОШЕЛЬКА"
  }'
```

### Шаг 2: Проверьте логи webhook

В Supabase Dashboard → Edge Functions → cryptomus-webhook → Logs:
- Должно быть: `[cryptomus-webhook] Received event`
- Должна быть обработка тестового платежа

---

## 🧪 Способ 3: Тестирование с минимальной суммой

### Вариант А: Создайте тестовый продукт с минимальной суммой

1. В Cryptomus Dashboard создайте продукт с минимальной суммой (например, $0.10)
2. Используйте этот продукт для тестирования
3. Совершите реальный платеж на минимальную сумму
4. Проверьте обработку

### Вариант Б: Используйте существующий продукт с минимальной суммой

Если у вас есть продукт с минимальной суммой:
1. Создайте платеж через приложение
2. Оплатите минимальную сумму реальными средствами
3. Проверьте обработку webhook и начисление

---

## 🔍 Что проверить после тестового платежа

### 1. В Supabase Dashboard:

**Таблица `purchases`:**
- Должна появиться запись со статусом `pending`
- Должен быть заполнен `cryptomus_order_id`
- После оплаты статус должен стать `completed`

**Таблица `transactions`:**
- После обработки webhook должна появиться запись
- Тип: `coins_purchase_cryptomus` или `premium_purchase_*`

**Таблица `profiles`:**
- Если Premium — должен обновиться `premium_until`
- Если монеты — должен увеличиться `coins`

### 2. В Cryptomus Dashboard:

**Платежи:**
- Должен появиться платеж
- Статус должен быть `paid` после оплаты

**Webhooks:**
- Должна быть запись о доставке webhook
- Статус должен быть `delivered`

## ⚠️ Важные замечания

### Минимальные суммы:

- **USDT (TRC20):** обычно минимум $1-5
- **BTC:** обычно минимум $10-20
- **ETH:** обычно минимум $5-10

Проверьте минимальные суммы в Cryptomus Dashboard перед тестированием.

### Комиссии:

- Cryptomus берет комиссию ~2% за транзакцию
- Учитывайте это при тестировании

### Безопасность:

- Тестовые платежи используют **реальные деньги**
- Используйте минимальные суммы для тестирования
- Не тестируйте на больших суммах
