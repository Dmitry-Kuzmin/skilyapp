# 🔧 Исправление ошибки 401 "Missing authorization header"

## ❌ Проблема

При настройке webhook в Cryptomus вы получаете ошибку:
```json
{"code":401,"message":"Missing authorization header"}
```

## ✅ Решение

Supabase Edge Functions по умолчанию требуют авторизацию, но Cryptomus не отправляет заголовки авторизации. Нужно сделать webhook **публичным**.

---

## 📋 Способ 1: Через Supabase Dashboard (РЕКОМЕНДУЕТСЯ)

### Шаг 1: Откройте настройки функции

1. Перейдите в **Supabase Dashboard**
2. Откройте **Edge Functions** → **cryptomus-webhook**
3. Перейдите на вкладку **Settings** (или **Configuration**)

### Шаг 2: Отключите проверку JWT

1. Найдите настройку **"Verify JWT"** или **"Require Authentication"**
2. **Отключите** её (установите в `false` или снимите галочку)
3. Сохраните изменения

### Шаг 3: Проверьте webhook

1. Вернитесь в **Cryptomus Dashboard**
2. Проверьте webhook — ошибка 401 должна исчезнуть

---

## 📋 Способ 2: Через конфигурацию (уже сделано)

Я уже добавил настройку в `supabase/config.toml`:

```toml
[functions.cryptomus-webhook]
verify_jwt = false
```

И задеплоил функцию. Но если ошибка осталась, используйте **Способ 1** через Dashboard.

---

## 🔍 Проверка работы

### Тест 1: Проверка URL

Откройте в браузере (должен вернуть ошибку метода, но не 401):
```
https://yffjnqegeiorunyvcxkn.supabase.co/functions/v1/cryptomus-webhook
```

Если получаете 401 — настройка не применилась, используйте **Способ 1**.

### Тест 2: Проверка в Cryptomus

1. В **Cryptomus Dashboard** → **Настройки** → **Webhooks**
2. Нажмите **"Проверить"** или **"Test"** на вашем webhook
3. Должен вернуться успешный ответ (не 401)

---

## ⚠️ Безопасность

Webhook защищен **подписью от Cryptomus**:

1. Cryptomus отправляет подпись в заголовке `sign`
2. Наш webhook проверяет подпись через `CRYPTOMUS_PAYMENT_KEY`
3. Если подпись неверна — запрос отклоняется

Поэтому безопасно делать webhook публичным — он защищен криптографической подписью.
