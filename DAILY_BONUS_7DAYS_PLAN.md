# 📅 План перехода Daily Bonus на 7-дневный цикл

## 💡 Идея

Изменить Daily Bonus с 90 дней на **7-дневный повторяющийся цикл**.

### Преимущества 7-дневного цикла:

✅ **Проще для пользователя**
- Не нужно держать streak 90 дней
- Пропустил день - не так критично (начнется новая неделя)
- Более понятная механика (недельный цикл)

✅ **Меньше нагрузки на БД**
- Всего 7 записей в `daily_bonus_def` вместо 90
- Меньше данных для хранения

✅ **Постоянная мотивация**
- Каждую неделю новый цикл
- Пользователь не теряет мотивацию после пропуска
- Всегда есть шанс начать заново

✅ **Лучше для retention**
- Недельный цикл психологически проще
- Меньше фрустрации при потере streak
- Больше шансов вернуться после пропуска

### Недостатки:

⚠️ **Меньше долгосрочной мотивации**
- Нет вызова "90 дней подряд"
- Но это компенсируется Duel Pass (30 дней сезон)

⚠️ **Меньше "железной воли"**
- 90 дней - это серьезный вызов
- Но для большинства пользователей это слишком сложно

---

## 🔄 Варианты реализации

### Вариант 1: Циклический 7-дневный streak (РЕКОМЕНДУЕТСЯ)

**Логика:**
- Streak считается по модулю 7: `day_number = (current_streak % 7) || 7`
- После 7 дня streak сбрасывается на 1 (новая неделя)
- В БД хранится общий счетчик дней, но награда берется по циклу

**Пример:**
- День 1-7: награды 1-7
- День 8: награда дня 1 (новая неделя)
- День 14: награда дня 7
- День 15: награда дня 1 (новая неделя)

**Плюсы:**
- ✅ Постоянная мотивация
- ✅ Простая логика
- ✅ Пользователь всегда может начать заново

**Минусы:**
- ⚠️ Нет долгосрочного вызова (но это компенсируется Duel Pass)

### Вариант 2: Просто максимум 7 дней

**Логика:**
- Streak максимум 7 дней
- После 7 дня streak остается на 7
- Пользователь получает награду дня 7 каждый день после 7 дня

**Плюсы:**
- ✅ Очень простая логика

**Минусы:**
- ❌ После 7 дня нет мотивации продолжать
- ❌ Пользователь получает одинаковую награду каждый день

**Вывод:** Не рекомендуется

## 📋 План изменений

### 1. Изменить логику вычисления day_number

**Было:**
```typescript
const currentReward = dashboardData.weeklyRewards.find(r => r.day_number === newStreak);
```

**Станет:**
```typescript
// Циклический расчет: день 1-7 повторяется
const dayNumber = newStreak % 7 || 7; // Если 0, то 7
const currentReward = dashboardData.weeklyRewards.find(r => r.day_number === dayNumber);
```

## 🎯 Рекомендуемая реализация

### Вариант: Циклический 7-дневный streak

**Логика:**
1. Streak хранится как общий счетчик (1, 2, 3, ... 100, 101, ...)
2. Награда берется по циклу: `day_number = streak % 7 || 7`
3. UI показывает недельный прогресс (1-7 дней)
4. После 7 дня показывается уведомление "Неделя завершена!"

**Преимущества:**
- ✅ Простая логика
- ✅ Постоянная мотивация
- ✅ Пользователь всегда может начать заново
- ✅ Меньше нагрузки на БД

**Пример работы:**
- День 1: streak=1, day_number=1, награда дня 1
- День 7: streak=7, day_number=7, награда дня 7, уведомление "Неделя завершена!"
- День 8: streak=8, day_number=1, награда дня 1 (новая неделя)
- День 14: streak=14, day_number=7, награда дня 7, уведомление "Неделя завершена!"
- День 15: streak=15, day_number=1, награда дня 1 (новая неделя)
