# 📊 Анализ системы Daily Bonus и сравнение с Duel Pass

## 🔍 Текущая логика Daily Bonus

### Что происходит при получении бонуса

При клике на "Получить бонус" в `handleClaimBonus` (Index.tsx):

1. **Обновление streak в БД:**
   - Обновляется `user_daily_bonus` (current_streak, last_claimed_date)
   - Вычисляется новый streak (до 90 дней)

2. **Награда из daily_bonus_def:**
   - Берется награда по номеру дня (day_number = streak)
   - Награда содержит: `{xp, coins, boost, badge}`
   - Примеры:
     - День 1: `{"xp": 10, "coins": 0}`
     - День 7: `{"xp": 50, "coins": 10, "badge": "7day_streak"}`
     - День 30: `{"xp": 100, "coins": 50}`

3. **Обновление XP напрямую:**
   ```typescript
   // Обновляем XP сразу (без ожидания Edge Functions)
   supabase.from('profiles').update({
     xp: (dashboardData.profile.xp || 0) + (currentReward.reward.xp || 0)
   })
   ```

4. **Вызов Edge Functions (асинхронно, не блокируя):**
   - `coins-earn` с `reward_type: 'daily_login'` → **+5 монет** (или +7.5 для Premium)
   - `season-sp` с `source_type: 'daily_login'` → **+15 SP** для Duel Pass
   - `duel-pass-xp` с `source_type: 'daily_login'` → **+15 XP** (старая система?)

### Проблемы в текущей логике

#### ❌ Проблема 1: Двойное начисление монет
- Daily Bonus дает монеты из `daily_bonus_def` (например, 10 монет за день 7)
- **ПЛЮС** `coins-earn` дает еще 5 монет
- **Итого:** Пользователь получает монеты дважды (из daily_bonus_def + из coins-earn)

#### ❌ Проблема 2: Двойное начисление XP
- Daily Bonus дает XP из `daily_bonus_def` (например, 50 XP за день 7)
- **ПЛЮС** `duel-pass-xp` дает еще 15 XP
- **Итого:** Пользователь получает XP дважды

#### ❌ Проблема 3: Старая система Duel Pass
- `duel-pass-xp` использует старую систему с `duel_pass_rewards` (10 уровней)
- `season-sp` использует новую систему с `duel_pass_season_rewards` (30 уровней)
- **Конфликт:** Две системы работают параллельно

#### ❌ Проблема 4: Несоответствие логики
- В `handleClaimBonus` монеты из `daily_bonus_def` **НЕ начисляются** напрямую
- Только XP начисляется напрямую
- Монеты должны начисляться через `coins-earn`, но там фиксированная сумма (5 монет)

## 🎮 Сравнение Daily Bonus и Duel Pass

### Daily Bonus (Ежедневная серия)
- **Цель:** Мотивация ежедневного входа
- **Механика:** Streak до 90 дней
- **Награды:** 
  - XP (10-100+ в зависимости от дня)
  - Монеты (0-50+ в зависимости от дня)
  - Бусты (в некоторые дни)
  - Бейджи (на вехах: 7, 14, 30, 60, 90 дней)
- **Частота:** Раз в день
- **Сброс:** При пропуске дня streak сбрасывается

### Duel Pass (Сезонная система)
- **Цель:** Мотивация активности в течение сезона
- **Механика:** 30 уровней, каждый требует 100 SP
- **Награды:**
  - Монеты (20-500 на уровнях)
  - Бусты
  - Косметика (скины, бейджи, стикеры)
  - Трофеи
- **Частота:** Постоянно (за дуэли, тесты, челленджи)
- **Сброс:** Каждый сезон (30 дней)

### Пересечения

| Действие | Daily Bonus | Duel Pass |
|----------|-------------|-----------|
| Ежедневный вход | ✅ +XP, +монеты, +бусты | ✅ +15 SP |
| Прохождение тестов | ❌ | ✅ +25 SP |
| Дуэли | ❌ | ✅ +30 SP (победа) |
| Челленджи | ❌ | ✅ +100-1500 SP |

**Вывод:** Daily Bonus и Duel Pass **дополняют друг друга**, но есть дублирование на ежедневном входе.

## 🐛 Найденные баги в логике

### Баг 1: Монеты не начисляются из daily_bonus_def
```typescript
// В handleClaimBonus:
// ✅ XP начисляется напрямую
supabase.from('profiles').update({
  xp: (dashboardData.profile.xp || 0) + (currentReward.reward.xp || 0)
})

// ❌ Монеты НЕ начисляются из currentReward.reward.coins
// Вместо этого вызывается coins-earn с фиксированными 5 монетами
```

**Проблема:** Если в `daily_bonus_def` указано 20 монет за день 6, пользователь получит только 5 монет из `coins-earn`.

### Баг 2: Двойное начисление XP
- XP из `daily_bonus_def` начисляется напрямую
- `duel-pass-xp` начисляет еще 15 XP
- **Итого:** Пользователь получает больше XP, чем должно быть

### Баг 3: Старая система Duel Pass все еще работает
- `duel-pass-xp` использует `duel_pass_rewards` (10 уровней) - **УСТАРЕВШАЯ**
- `season-sp` использует `duel_pass_season_rewards` (30 уровней) - **НОВАЯ**
- Обе системы работают параллельно, что может вызывать путаницу
