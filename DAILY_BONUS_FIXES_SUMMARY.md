## 🐛 Потенциальные проблемы и решения

### Проблема 1: RLS блокирует вставку косметики

**Решение:**
- Функция `grant_random_loot` использует `SECURITY DEFINER`, что обходит RLS
- Добавлены политики INSERT для пользователей в миграции `20251122000002_fix_cosmetics_rls_for_daily_bonus.sql`

### Проблема 2: Telegram пользователи не могут получить награды

**Решение:**
- Создана функция `can_access_daily_bonus()` с fallback для Telegram
- Создана функция `can_access_cosmetics()` с fallback для Telegram
- Обновлены все RLS политики

### Проблема 3: Стикеры не созданы в базе

**Решение:**
- Проверено наличие стикеров в миграции `20251116100000_create_cosmetics_system.sql`
- Добавлены дополнительные стикеры в миграцию `20251122000002_fix_cosmetics_rls_for_daily_bonus.sql`

## 🧪 Как проверить начисление наград

### В консоли браузера:

1. Откройте DevTools (F12)
2. Перейдите на вкладку Console
3. Получите ежедневный бонус
4. Проверьте логи:
   - `[handleClaimBonus] Sticker granted:` - стикер начислен
   - `[handleClaimBonus] Boost granted:` - boost начислен
   - `[handleClaimBonus] Seasonal badge granted:` - бейдж начислен
   - `[handleClaimBonus] Error granting...` - ошибка (нужно исправить)

### В базе данных:

```sql
-- Проверить стикеры пользователя
SELECT * FROM user_stickers WHERE user_id = 'your-user-id' AND obtained_from = 'daily_bonus';

-- Проверить скины пользователя
SELECT * FROM user_skins WHERE user_id = 'your-user-id' AND obtained_from = 'daily_bonus';

-- Проверить бейджи пользователя
SELECT * FROM user_badges WHERE user_id = 'your-user-id' AND obtained_from = 'daily_bonus';

-- Проверить boost инвентарь
SELECT * FROM boost_inventory WHERE user_id = 'your-user-id' AND boost_type = 'double_sp';
```
