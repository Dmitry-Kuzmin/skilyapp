# 🔍 Диагностика проблем синхронизации

## ❌ Текущая проблема

Синхронизация завершается с предупреждениями:
- ✅ Темы обработано: 0
- ❌ Вопросов добавлено: 0
- ❌ Вопросов обновлено: 0
- ❌ Всего обработано: 0
- ⚠️ Вопросов пропущено: 60

## 🔍 Причины пропуска вопросов

1. **Отсутствует source_id** - первая колонка пустая
2. **Темы не найдены в базе** - topic_number не соответствует существующим темам
3. **Неправильный формат данных** - данные в неправильном формате
4. **Отсутствуют обязательные поля** - question_ru, question_es, question_en не заполнены

## ✅ Пошаговая диагностика

### Шаг 1: Проверьте структуру Google Sheets

1. **Откройте вашу таблицу Google Sheets**
2. **Перейдите на вкладку "Questions"**
3. **Проверьте первую строку (заголовки):**

   Должен быть такой порядок колонок:
   ```
   A: source_id
   B: topic_number
   C: difficulty
   D: is_premium
   E: type
   F: image_url
   G: sign_code
   H: source
   I: question_ru
   J: question_es
   K: question_en
   L: explanation_ru
   M: explanation_es
   N: explanation_en
   O: tags
   P: answer_1_ru
   Q: answer_1_es
   R: answer_1_en
   S: is_correct_1
   ... и так далее
   ```

4. **Проверьте вторую строку (первая строка данных):**
   - Колонка A (source_id): должен быть заполнен (например, `GS-1`, `GS-2`)
   - Колонка B (topic_number): должен быть числом от 1 до 10
   - Колонка I (question_ru): должен быть заполнен
   - Колонка J (question_es): должен быть заполнен
   - Колонка K (question_en): должен быть заполнен

### Шаг 2: Проверьте topics в базе данных

1. **Откройте SQL Editor:**
   ```
   https://supabase.com/dashboard/project/yffjnqegeiorunyvcxkn/sql/new
   ```

2. **Выполните запрос:**
   ```sql
   SELECT number, title_ru FROM topics ORDER BY number;
   ```

3. **Проверьте, что topics существуют:**
   - Должны быть темы с номерами от 1 до 10
   - Если topics нет, сначала синхронизируйте вкладку "Topics" в Google Sheets

### Шаг 3: Проверьте логи функции

1. **Откройте логи:**
   ```
   https://supabase.com/dashboard/project/yffjnqegeiorunyvcxkn/functions/sync-google-sheets/logs
   ```

2. **Ищите сообщения:**
   - `Parsed X columns for row Y` - показывает, сколько колонок распознано
   - `Вопрос пропущен: отсутствует source_id` - первая колонка пустая
   - `Тема X не найдена в базе` - topic_number неправильный или тема не существует
   - `Вопрос пропущен: отсутствуют тексты вопроса` - question_ru/es/en не заполнены

### Шаг 4: Проверьте формат данных

**source_id (колонка A):**
- ✅ Правильно: `GS-1`, `GS-2`, `Q-1`, `QUESTION-1`
- ❌ Неправильно: пусто, пробелы, только числа без префикса

**topic_number (колонка B):**
- ✅ Правильно: `1`, `2`, `3`, ..., `10`
- ❌ Неправильно: `0`, `11`, `тема 1`, `Topic 1`, пусто

**question_ru/es/en (колонки I, J, K):**
- ✅ Правильно: любой текст
- ❌ Неправильно: пусто, только пробелы

## 🔧 Решение проблем

### Проблема 1: "Отсутствует source_id"

**Решение:**
1. Добавьте `source_id` в первую колонку (A)
2. Используйте формулу для автоматической генерации:
   ```
   ="GS-" & ROW()-1
   ```
   Это создаст: GS-1, GS-2, GS-3, ...

### Проблема 2: "Темы не найдены в базе"

**Решение:**
1. Проверьте, что `topic_number` соответствует существующим темам (1-10)
2. Если topics нет в базе, сначала синхронизируйте вкладку "Topics"
3. Проверьте, что в Google Sheets вкладка "Topics" заполнена правильно

### Проблема 3: "Отсутствуют обязательные поля"

**Решение:**
1. Убедитесь, что `question_ru`, `question_es`, `question_en` заполнены
2. Проверьте, что данные находятся в правильных колонках (I, J, K)

### Проблема 4: "Неправильный формат данных"

**Решение:**
1. Проверьте, что `topic_number` - это число, а не текст
2. Проверьте, что нет лишних пробелов в начале/конце ячеек
3. Используйте функцию `TRIM()` в Google Sheets для очистки пробелов
