# 📦 Объяснение деплоя

## 🤔 Нужно ли деплоить и в GitHub, и в Vercel?

**НЕТ!** Теперь процесс такой:

---

## 📋 Текущая ситуация

### GitHub (хранение кода)

GitHub = хранилище кода (как Dropbox для кода)

**Что делаешь:**
- ✅ `git push` — сохраняешь код в GitHub
- ✅ Версионирование (история изменений)
- ✅ Бэкап кода

**Что НЕ делаешь:**
- ❌ НЕ деплоишь сайт туда (GitHub Pages отключен)

### Vercel (хостинг сайта)

Vercel = хостинг сайта (где живёт твой сайт)

**Что делаешь:**
- ✅ Деплоишь сайт на Vercel
- ✅ Пользователи видят сайт на Vercel

**Как деплоишь:**
- Через Vercel CLI: `vercel --prod`
- Или через GitHub Actions (автоматически)

---

## 🔄 Два способа деплоя

### Вариант 1: Vercel CLI (вручную)

**Процесс:**
1. Изменяешь код локально
2. Коммитишь: `git commit -m "изменения"`
3. Пушишь в GitHub: `git push origin main`
4. **Отдельно** деплоишь на Vercel: `vercel --prod`

**Когда использовать:**
- ✅ Когда хочешь полный контроль
- ✅ Когда экспериментируешь
- ✅ Когда автодеплой не работает

**Минус:**
- ❌ Нужно помнить деплоить вручную

---

### Вариант 2: GitHub Actions (автоматически)

**Процесс:**
1. Изменяешь код локально
2. Коммитишь: `git commit -m "изменения"`
3. Пушишь в GitHub: `git push origin main`
4. **Автоматически** GitHub Actions деплоит на Vercel

**Когда использовать:**
- ✅ Когда хочешь автодеплой
- ✅ Удобно для регулярной разработки

**Плюс:**
- ✅ Не нужно думать о деплое — происходит автоматически

**Настройка:**
Смотри `DEPLOY_WITHOUT_AUTODEPLOY.md` — там инструкция по настройке GitHub Actions.

---

## 📊 Сравнение

| Действие | GitHub | Vercel | Описание |
|----------|--------|--------|----------|
| `git push` | ✅ Да | ❌ Нет | Сохраняешь код |
| Деплой сайта | ❌ Нет | ✅ Да | Загружаешь сайт |
| Хранение кода | ✅ Да | ❌ Нет | Репозиторий |
| Хостинг сайта | ❌ Нет | ✅ Да | Работающий сайт |

---

## 💡 Рекомендация

### Сейчас (пока автодеплой не работает):

1. **Работаешь над кодом** локально
2. **Сохраняешь в GitHub**: `git push`
3. **Деплоишь на Vercel**: `vercel --prod` (вручную)

### В будущем (когда настроишь GitHub Actions):

1. **Работаешь над кодом** локально
2. **Сохраняешь в GitHub**: `git push`
3. **Автоматически** деплоится на Vercel

---

## 🎯 Итог

**GitHub:**
- ✅ Хранит код
- ✅ История изменений
- ✅ Работа в команде (если будет)

**Vercel:**
- ✅ Хостит сайт
- ✅ Пользователи видят сайт здесь
- ✅ Деплоишь через CLI или GitHub Actions

**Не нужно:**
- ❌ Деплоить в GitHub (там только код)
- ❌ Думать что они связаны (кроме автодеплоя)

---

**Простыми словами:**
- GitHub = архив кода
- Vercel = место где работает сайт
- Push в GitHub ≠ деплой сайта
- Деплой на Vercel = сайт обновляется
