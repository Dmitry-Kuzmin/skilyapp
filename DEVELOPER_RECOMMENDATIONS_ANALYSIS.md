## 🎯 МОЯ РЕКОМЕНДАЦИЯ: Гибридный подход

### Почему именно так?

1. **У вас уже есть разделение:**
   - `Landing.tsx` - для неавторизованных
   - `Dashboard` - для авторизованных
   - Но оба загружают весь vendor.js

2. **Проблема:**
   - Даже на лендинге загружается Supabase, React Query, Radix UI
   - Это не нужно для простого лендинга

3. **Решение:**
   - Создать легкий лендинг без тяжелых зависимостей
   - Оставить тяжелое SPA для авторизованных пользователей

## 📋 КОНКРЕТНЫЙ ПЛАН ДЕЙСТВИЙ

### Этап 1: Быстрый выигрыш (1-2 дня)

#### 1.1. Локальный билд для SSG (из рекомендации #1)

**Что сделать:**
```bash
# Создать скрипт для локального билда и деплоя
npm run build:local-deploy
```

**Скрипт:**
```json
{
  "scripts": {
    "build:local-deploy": "npm run build && npm run prerender && vercel deploy dist --prod"
  }
}
```

**Результат:**
- SSG будет работать (HTML с контентом)
- LCP улучшится на 0.5-1 секунду
- PSI Mobile: 66 → 70-72

#### 1.2. Оптимизация критического пути

**Что сделать:**
- Убедиться, что `Landing.tsx` не импортирует тяжелые зависимости
- Проверить, что Supabase/React Query не инициализируются на лендинге

**Результат:**
- PSI Mobile: 70-72 → 72-75

### Этап 2: Разделение лендинга и приложения (1-2 недели)

#### 2.1. Создать легкий лендинг

**Что сделать:**

1. **Создать отдельный entry point для лендинга:**
   ```typescript
   // src/landing/main.tsx
   import { createRoot } from 'react-dom/client';
   import { LandingApp } from './LandingApp';
   
   // Минимальные зависимости: только React, без Supabase/Query
   const root = createRoot(document.getElementById('root')!);
   root.render(<LandingApp />);
   ```

2. **Создать легкий LandingApp:**
   ```typescript
   // src/landing/LandingApp.tsx
   // БЕЗ Supabase, React Query, Radix UI
   // Только базовые компоненты
   ```

3. **Настроить Vite для двух entry points:**
   ```typescript
   // vite.config.ts
   build: {
     rollupOptions: {
       input: {
         main: './index.html',
         landing: './landing.html'
       }
     }
   }
   ```

**Результат:**
- Лендинг: ~150KB gzipped (вместо 830KB)
- PSI Mobile для лендинга: 85-90+
- Приложение остается на 66-70 (это нормально)

#### 2.2. Настроить роутинг

**Что сделать:**
- Лендинг: `/` (легкий, без авторизации)
- Приложение: `/app` или `app.skilyapp.com` (тяжелое SPA)

**Результат:**
- SEO работает на лендинге (90+ PSI)
- Пользователи получают быстрый лендинг
- Приложение работает как раньше

## 💡 МОЕ ФИНАЛЬНОЕ МНЕНИЕ

### Что делать СЕЙЧАС:

1. **Остановиться на текущем результате** (66 Mobile, 93 Desktop) - это нормально для SPA
2. **Реализовать локальный билд для SSG** - быстрый выигрыш без больших изменений
3. **Если бизнес требует 90+ на мобиле** - делать вариант B (разделение лендинга)

### Что НЕ делать:

1. ❌ **Не пытаться добить 90+ на текущем SPA** - это бесполезная трата времени
2. ❌ **Не делать lazy loading Dashboard** - ухудшит LCP
3. ❌ **Не мигрировать на Next.js сейчас** - слишком радикально

### Критерии для принятия решения:

**Если:**
- Пользователи не жалуются на скорость
- Конверсии в порядке
- SEO не критичен (брендовый трафик)

**То:**
- Остановиться на текущем результате
- Сделать только локальный билд для SSG
- Перестать "воевать" за 90+

**Если:**
- Нужен SEO трафик
- Маркетинг требует 90+ PSI
- Есть ресурсы на 1-2 недели

**То:**
- Делать вариант B (разделение лендинга)
- Это даст максимальный выхлоп за разумное время
