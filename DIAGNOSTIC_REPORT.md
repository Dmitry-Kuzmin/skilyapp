## 3️⃣ Real-time vs Polling

### Текущее использование Real-time

**Используется для:**
1. ✅ **Дуэли** (`useDuelRealtime`) - **ОПРАВДАНО**
   - Нужна мгновенная синхронизация между игроками
   - Fallback на heartbeat/polling уже есть

2. ⚠️ **Уведомления** (`useNotifications`) - **МОЖНО ЗАМЕНИТЬ**
   - Real-time для уведомлений не критичен
   - Можно использовать polling каждые 30-60 секунд
   - Это снизит нагрузку на Supabase

**Рекомендации:**

1. **Заменить real-time на polling для уведомлений:**
   ```typescript
   // Вместо real-time subscription:
   useQuery({
     queryKey: ['notifications', profileId],
     queryFn: () => fetchNotifications(profileId),
     refetchInterval: 30000, // 30 секунд
     staleTime: 20000, // 20 секунд
   });
   ```

2. **Оставить real-time только для дуэлей:**
   - ✅ Уже правильно настроено
   - ✅ Есть fallback на heartbeat
