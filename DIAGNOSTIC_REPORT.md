## 3. 🏁 Results & Persistence (Финал и Данные)

### ⚠️ Нормальное завершение
**Статус:** 🟡 Требует внимания

**Найдено:**
- `finishDuel` создает snapshot ПЕРЕД вызовом `onDuelFinished`
- Snapshot сохраняется в localStorage через `saveDuelResultSnapshot`
- `useDuelResults` использует `initialSnapshot` (передается напрямую из памяти)
- Fallback на localStorage snapshot если `initialSnapshot` не передан
- Retry логика: 10 попыток с экспоненциальной задержкой (до 10 секунд)

**Потенциальные проблемы:**
- ⚠️ **КРИТИЧНО:** Если пользователь перезагрузит страницу на экране результатов, `initialSnapshot` будет потерян
- ⚠️ Snapshot в localStorage может быть устаревшим (максимум 5 минут)

**Рекомендации:**
- ✅ **КРИТИЧНО:** Улучшить восстановление после reload:
  - Проверять `loadDuelResultSnapshot` при монтировании `DuelResult`
  - Если snapshot найден, использовать его вместо запроса к БД
  - Добавить индикатор загрузки при восстановлении

## 4. 🔌 Resilience & Edge Cases (Стресс-тест)

### ⚠️ Reload на Результатах
**Статус:** 🟡 Требует внимания

**Найдено:**
- `useDuelResults` проверяет snapshot в localStorage
- `useActiveDuel` восстанавливает состояние из localStorage
- Snapshot имеет максимальный возраст 5 минут
- `activeDuel` имеет максимальный возраст 15 минут (STALE_DUEL_AGE_MS)

**Потенциальные проблемы:**
- ⚠️ **КРИТИЧНО:** Если пользователь перезагрузит страницу на экране результатов:
  - `initialSnapshot` будет потерян (он только в памяти)
  - `useDuelResults` попытается загрузить из БД, но может быть race condition
  - Snapshot в localStorage может быть устаревшим или отсутствовать

**Рекомендации:**
- ✅ **КРИТИЧНО:** Улучшить восстановление:
  1. При монтировании `DuelResult` проверять `loadDuelResultSnapshot(duelId)`
  2. Если snapshot найден и не устарел, использовать его
  3. Если snapshot не найден, использовать `useDuelResults` с retry
  4. Показывать индикатор загрузки во время восстановления

### ✅ Дисконнект (Technical Win)

## 🚨 Критические Проблемы

### 1. ⚠️ Reload на экране результатов
**Приоритет:** 🔴 Высокий

**Проблема:**
- При перезагрузке страницы на экране результатов `initialSnapshot` теряется
- Snapshot в localStorage может быть устаревшим или отсутствовать
- `useDuelResults` может не найти данные в БД (race condition)

**Решение:**
```typescript
// В DuelResult.tsx при монтировании:
useEffect(() => {
  if (!initialSnapshot && duelId) {
    const savedSnapshot = loadDuelResultSnapshot(duelId);
    if (savedSnapshot && savedSnapshot.duelId === duelId) {
      // Использовать savedSnapshot
      setRestoredSnapshot(savedSnapshot);
    }
  }
}, [duelId, initialSnapshot]);
```

### 2. ⚠️ Дисконнект - Technical Win (Обновлено)
**Приоритет:** 🟢 Низкий (найдено и проверено)

**Статус:**

- ✅ Логика реализована корректно
- ⚠️ Таймаут 60 секунд (больше, чем 15-20 в чеклисте)
