### 2. Избыточное логирование (Низкий приоритет)

**Проблема:**
- Множество `console.warn` и `console.error` в компоненте `OilSplashAttack.tsx` (19+ вызовов)
- Debug логи в production коде (`useDuelRealtime.ts`)

**Рекомендация:**
- Использовать условное логирование через `import.meta.env.DEV`
- Рассмотреть использование библиотеки для логирования (например, `pino` или `winston`)
- Удалить debug логи перед production билдом

**Пример:**
```typescript
const isDev = import.meta.env.DEV;
if (isDev) {
  console.warn('[OilSplashAttack] Debug message');
}
```
