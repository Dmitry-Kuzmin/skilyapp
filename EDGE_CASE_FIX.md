# 🔧 Исправление Edge Case: Обработка null для season_progress

## Проблема

После применения миграции Super RPC v2.1 обнаружен edge case:

**Проблема:** Если `season_progress` равен `null` (для нового пользователя), код:
1. Не обрабатывал `null` корректно
2. Уходил в fallback и вызывал `get_or_create_season_progress` автоматически
3. Это создавало лишний запрос, который мы хотели избежать

## Решение

### ✅ Что исправлено:

1. **`useDuelPassInfo.ts`**
   - Изменена проверка: теперь работает даже если `season_progress = null`
   - Добавлена обработка `null`: используется дефолтный объект с `level: 0`, `season_points: 0`
   - **НЕ создается прогресс автоматически** - только когда пользователь реально начинает играть

2. **`useDuelPassData.ts`**
   - Аналогично: обрабатывает `null` как дефолтные значения
   - Уровень для нового пользователя: `0` (не `1`)
   - Добавлено логирование: `isNewUser: !dashboardData.season_progress`

### 📋 Логика обработки:

```typescript
// Если прогресса нет (null) - используем дефолтные значения
const progress = dashboardData.season_progress || {
  season_points: 0,
  level: 0, // Уровень 0 для нового пользователя
};
```

### 🎯 Правильное поведение:

- ✅ **Новый пользователь:** Видит "Уровень 0", "0 SP"
- ✅ **Нет автоматического создания:** Прогресс создается только при реальной активности
- ✅ **Fallback работает:** Если Super RPC не вернул данные, делаем отдельные запросы
- ✅ **Graceful degradation:** Приложение не падает, просто показывает нулевые значения

### ⚠️ Важно:

**Прогресс создается только когда:**
- Пользователь реально начинает играть (получает SP)
- Или нажимает кнопку "Начать сезон" (если такая есть в UI)

**НЕ создается:**
- При загрузке dashboard
- При просмотре модалки Duel Pass
- При простом открытии приложения
