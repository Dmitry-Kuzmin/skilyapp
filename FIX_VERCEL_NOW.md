# ⚡ СРОЧНО: Запустить деплой на Vercel

## 🚨 Проблема
- GitHub: последний коммит **30 минут назад** ✅
- Vercel: последний деплой **10 часов назад** ❌
- Автодеплой не работает!

---

## ✅ Решение (2 минуты)

### Шаг 1: Ручной деплой через Vercel Dashboard

1. Открой: https://vercel.com/dashboard
2. Найди проект `sdadim-dgt-prep`
3. Перейди в **Deployments**
4. Нажми кнопку **"Create Deployment"** (справа вверху, синяя кнопка)
5. Выбери:
   - **Branch:** `main`
   - Остальное оставь по умолчанию
6. Нажми **Deploy**
7. Дождись завершения (2-5 минут)

**Готово!** Сайт обновится с последним кодом.

---

### Шаг 2: Проверка после деплоя

После завершения деплоя:
1. Открой сайт (ссылка в Vercel Dashboard)
2. Проверь что данные загружаются
3. Открой DevTools (F12) → Console
4. Не должно быть ошибок `VITE_SUPABASE_URL is not set`

---

## 🔧 Исправление автодеплоя (5 минут)

После того как сайт работает, исправь автодеплой:

### Вариант 1: Переподключить репозиторий

1. **Settings** → **Git**
2. Нажми **Disconnect** (красная кнопка)
3. Подожди 10 секунд
4. Нажми **Connect Git Repository**
5. Выбери `Dmitry-Kuzmin/sdadim-dgt-prep`
6. Выбери ветку `main`
7. Нажми **Connect**
8. Vercel автоматически запустит первый деплой

### Вариант 2: Проверить вебхук в GitHub

1. Открой: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/settings/hooks
2. Ищи вебхук от Vercel (обычно содержит `vercel.com`)
3. Проверь:
   - ✅ Активен (зелёная галочка)
   - ✅ Последняя доставка была недавно

**Если вебхука нет или он не работает:**
- Переподключи репозиторий (Вариант 1 выше)

## 🆘 Если ручной деплой не работает

1. Проверь переменные окружения в Vercel:
   - **Settings** → **Environment Variables**
   - Должны быть: `VITE_SUPABASE_URL` и `VITE_SUPABASE_PUBLISHABLE_KEY`

2. Проверь логи деплоя:
   - **Deployments** → последний деплой → **Build Logs**
   - Ищи ошибки

3. Если есть ошибки - скопируй их, и мы разберёмся.
