# 🔄 Правильный Workflow для работы с Lovable и Cursor

> **📌 ВАЖНО:** Перед работой обязательно прочитайте [`CURSOR_RULES.md`](./CURSOR_RULES.md) - там описаны правила, какие файлы можно изменять, а какие НЕЛЬЗЯ!

## 🎯 Проблема

**Конфликты возникают потому что:**
- ✅ Lovable **автоматически** коммитит изменения после каждого запроса
- ✅ Cursor работает с той же веткой и делает коммиты
- ✅ Оба инструмента работают **одновременно** с одной и той же веткой
- ❌ Результат: конфликты при `git pull` / `git push`

## ✅ Решение: Правильный Workflow

### **Вариант 1: Разделение по веткам (Рекомендуется)**

Используйте **разные ветки** для каждого инструмента:

```bash
# В Cursor - создайте свою рабочую ветку
git checkout -b cursor-work

# После работы в Cursor:
git add .
git commit -m "feat: изменения через Cursor"
git push origin cursor-work

# Затем в Lovable или через GitHub:
# - Создайте Pull Request из cursor-work в main
# - Или мержите напрямую через GitHub
```

**Преимущества:**
- ✅ Нет конфликтов во время работы
- ✅ Видно, кто что сделал
- ✅ Можно проверить изменения перед мержем

---

### **Вариант 2: Синхронизация перед работой (Простой)**

**Правило:** Всегда синхронизируйтесь **ПЕРЕД** началом работы:

```bash
# В Cursor - ПЕРЕД началом работы:
git pull origin <ваша-ветка>  # или git pull --rebase

# После работы:
git add .
git commit -m "feat: изменения"
git pull --rebase origin <ваша-ветка>  # Синхронизируем с Lovable
git push origin <ваша-ветка>
```

**Важно:**
- 🔄 **Всегда** делайте `git pull` перед началом работы
- 🔄 Используйте `--rebase` для более чистой истории
- ⚠️ Если Lovable работал, будут конфликты - решайте их сразу

---

### **Вариант 3: Определить главный инструмент (Строгий)**

**Если Lovable - главный:**
- ✅ В Lovable работайте как обычно
- ✅ В Cursor: **только читайте** код, не коммитьте
- ✅ Или делайте изменения в отдельной ветке и мерьте через PR

**Если Cursor - главный:**
- ✅ В Cursor работайте как обычно
- ✅ В Lovable: используйте **read-only** режим или временно отключите авто-коммиты

---

## 🛠️ Практические команды для Cursor

### **Ежедневный workflow:**

```bash
# 1. УТРОМ: Синхронизация с Lovable
git fetch origin
git pull --rebase origin <ваша-ветка>

# 2. РАБОТА: Делайте изменения в Cursor

# 3. ПЕРЕД КОММИТОМ: Снова синхронизация
git pull --rebase origin <ваша-ветка>

# 4. КОММИТ: Делайте коммит
git add .
git commit -m "feat: описание изменений"

# 5. ОТПРАВКА: Отправляем в репозиторий
git push origin <ваша-ветка>
```

### **Если возник конфликт:**

```bash
# 1. Git покажет конфликт
git pull --rebase origin <ваша-ветка>

# 2. Откройте файлы с конфликтами (<<<<<<, ======, >>>>>>)
# 3. Решите конфликты вручную
# 4. Продолжите rebase:
git add <файлы-с-конфликтами>
git rebase --continue

# 5. Отправьте изменения:
git push origin <ваша-ветка>
```

## 📋 Чек-лист перед работой в Cursor

- [ ] ✅ `git pull --rebase` - синхронизация с Lovable
- [ ] ✅ Работа в Cursor
- [ ] ✅ `git pull --rebase` - еще раз перед коммитом
- [ ] ✅ `git add .` - добавить изменения
- [ ] ✅ `git commit -m "..."` - закоммитить
- [ ] ✅ `git push` - отправить в репозиторий

## 🎯 Рекомендация для вашего случая

**Используйте Вариант 2 (Синхронизация перед работой):**

1. **Перед началом работы в Cursor:**
   ```bash
   git pull --rebase origin 2025-11-03-9ng1-gqCtX
   ```

2. **После работы:**
   ```bash
   git pull --rebase origin 2025-11-03-9ng1-gqCtX
   git add .
   git commit -m "feat: описание"
   git push origin 2025-11-03-9ng1-gqCtX
   ```

3. **Если Lovable работал параллельно:**
   - Git покажет конфликты
   - Решите их вручную
   - Продолжите с `git rebase --continue`

---

## ⚠️ Важные правила

1. **НИКОГДА не коммитьте без синхронизации** - сначала `git pull`
2. **Используйте `--rebase`** - более чистая история коммитов
3. **Решайте конфликты сразу** - не откладывайте
4. **Проверяйте статус:** `git status` перед коммитом
5. **Сохраняйте изменения:** `git stash` если нужно быстро переключиться

## 🔧 Автоматизация (опционально)

Можно создать скрипт для автоматической синхронизации:

```bash
# create: sync.sh
#!/bin/bash
git pull --rebase origin 2025-11-03-9ng1-gqCtX
git add .
git commit -m "$1"
git push origin 2025-11-03-9ng1-gqCtX
```

Использование:
```bash
chmod +x sync.sh
./sync.sh "feat: мое изменение"
```

## 📞 Что делать если все сломалось?

1. **Сохраните текущие изменения:**
   ```bash
   git stash
   ```

2. **Сбросьте на последний коммит:**
   ```bash
   git reset --hard origin/2025-11-03-9ng1-gqCtX
   ```

3. **Верните изменения:**
   ```bash
   git stash pop
   ```

4. **Начните заново:**
   ```bash
   git pull --rebase origin 2025-11-03-9ng1-gqCtX
   ```

---

**Главное правило:** Всегда синхронизируйтесь ПЕРЕД началом работы! 🔄
