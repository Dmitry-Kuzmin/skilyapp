# Как эффективно сообщать о проблемах и ошибках

## 🎯 Главное правило

**Текст + код > Скриншот** для большинства случаев.

## ✅ Когда нужен скриншот

1. **Визуальные проблемы**:
   - Неправильное отображение элементов UI
   - Проблемы с анимациями
   - Ошибки верстки/стилей

2. **Сложные сценарии**:
   - Когда текст не передает проблему полностью
   - Когда нужно показать последовательность действий

3. **Ошибки в консоли браузера** (если текст ошибки слишком длинный)

## ❌ Когда скриншот НЕ нужен

1. **Ошибки в коде**:
   - Используй копирование текста ошибки из консоли
   - Показывай соответствующие строки кода

2. **Проблемы с логикой**:
   - Описывай проблему текстом
   - Показывай релевантный код

3. **Простые вопросы**:
   - Текстовое описание проблемы
   - Вопрос без скриншота

## 📝 Рекомендуемый формат сообщения

### ✅ Хороший формат:

```
Проблема: кнопка "Войти" не работает

Ошибка в консоли:
TypeError: undefined is not an object (evaluating 'f.useLayoutEffect')

Файл: src/components/LoginButton.tsx:45

Что делал:
1. Нажал кнопку "Войти"
2. Открылась модалка
3. Появилась ошибка в консоли
```

### ❌ Плохой формат:

```
[скриншот белого экрана]
```

## 💡 Эффективные способы передачи информации

### 1. **Текстовые ошибки**

Вместо скриншота консоли:
- Открой DevTools (F12)
- Скопируй текст ошибки
- Вставь в сообщение

**Пример:**
```
❌ Не работает:
[скриншот консоли с ошибкой]

✅ Работает:
Ошибка: ReferenceError: Newspaper is not defined
Файл: src/components/Layout.tsx:23
```

### 2. **Код ошибки**

Покажи релевантный код:
```tsx
// src/components/Layout.tsx
const navigation = [
  { name: "Блог", href: "/blog", icon: Newspaper }, // ❌ Newspaper не импортирован
];
```

### 3. **Структурированное описание**

Для сложных проблем:
```
Проблема: [краткое описание]

Шаги воспроизведения:
1. [шаг 1]
2. [шаг 2]
3. [шаг 3]

Ожидаемое поведение: [что должно было произойти]
Фактическое поведение: [что произошло]

Окружение:
- Браузер: Safari
- ОС: macOS
- Версия: [если есть]
```

## 🖼️ Если скриншот все же нужен

1. **Используй инструменты выделения**:
   - Выдели только проблемную область
   - Не делай скриншот всего экрана

2. **Сжимай изображения**:
   - Используй инструменты сжатия (например, TinyPNG)
   - PNG → JPG (для скриншотов JPG обычно достаточно)

3. **Описывай контекст**:
   - Добавляй текстовое описание к скриншоту
   - Указывай, что именно на скриншоте важно

## 🧹 Автоматическая очистка

Система автоматически очищает старые скриншоты (> 14 дней) для экономии места.

Если нужно очистить вручную:
```bash
cd ~/Desktop/Sdadim/sdadim-dgt-prep
./scripts/cleanup-cursor-large-files.sh
```

## 📊 Статистика

- **Один скриншот**: ~2-5MB
- **100 скриншотов**: ~200-500MB
- **Автоочистка**: старые (> 14 дней) удаляются автоматически

## 🎓 Лучшие практики

1. **Начинай с текста** - опиши проблему словами
2. **Добавляй код** - покажи релевантные строки
3. **Используй скриншоты выборочно** - только когда действительно нужно
4. **Удаляй старые скриншоты** - запускай очистку периодически

## 💬 Примеры эффективной коммуникации

### Пример 1: Ошибка импорта

**❌ Плохо:**
```
[скриншот белого экрана]
```

**✅ Хорошо:**
```
Ошибка: ReferenceError: Newspaper is not defined
Файл: src/components/Layout.tsx:23

Нужно добавить импорт:
import { Newspaper } from "lucide-react";
```

### Пример 2: Проблема с UI

**✅ Хорошо:**
```
Проблема: кнопка "Войти" не видна на мобильных устройствах

Файл: src/components/Layout.tsx

Скриншот мобильной версии: [скриншот]
Текст ошибки: нет ошибки, просто не видно кнопку

Стили кнопки:
className="hidden sm:flex" // возможно, нужно изменить
```

### Пример 3: Логика не работает

**✅ Хорошо:**
```
Проблема: счетчик очков не обновляется после ответа

Файл: src/pages/TestSession.tsx
Функция: handleAnswer

Текущий код:
const handleAnswer = (answer: string) => {
  setScore(score + 1); // ❌ Проблема здесь
};

Проблема: state обновляется асинхронно, нужно использовать callback
```

## 🚀 Итог

- **80% проблем** можно решить без скриншотов (текст + код)
- **20% проблем** требуют скриншотов (визуальные баги)
- **Автоочистка** управляет старыми скриншотами автоматически
- **Эффективная коммуникация** = быстрее решение проблем
