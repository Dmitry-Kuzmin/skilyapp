## 🎯 План оптимизации

### Приоритет 1: Оптимизация CSS (высокий приоритет)

**Проблема:** CSS bundle 470.73 KB (57.96 KB gzip) - слишком большой

**Решения:**
1. **Проверить unused CSS:**
   - Использовать PurgeCSS или аналоги
   - Удалить неиспользуемые стили

2. **Разделить CSS на критические и некритические:**
   - Критические стили (above-the-fold) → inline в `<head>`
   - Некритические стили → lazy load

3. **Оптимизировать Tailwind:**
   - Проверить, что используется только нужное
   - Убедиться, что JIT mode включен

**Ожидаемый эффект:** -20-30 KB gzip, +3-5 пунктов Score

### Приоритет 2: Дополнительное code splitting (средний приоритет)

**Проблема:** vendor.js все еще большой (1.1M)

**Решения:**
1. **Проверить, можно ли выделить еще библиотеки:**
   - `@tanstack/react-query` (если используется не везде)
   - `@supabase` (если можно lazy load)

2. **Оптимизировать импорты:**
   - Использовать named imports вместо default
   - Убедиться, что tree shaking работает

**Ожидаемый эффект:** -50-100 KB gzip, +2-3 пункта Score

### Приоритет 3: Оптимизация изображений (средний приоритет)

**Решения:**
1. **Конвертация в WebP:**
   - Все изображения должны быть в WebP формате
   - Использовать `srcset` для responsive images

2. **Оптимизация размеров:**
   - Убедиться, что все изображения имеют правильные размеры
   - Использовать `width` и `height` атрибуты (уже сделано)

**Ожидаемый эффект:** Улучшение LCP на 200-300ms, +2-3 пункта Score

### Приоритет 4: Оптимизация загрузки ресурсов (низкий приоритет)

**Решения:**
1. **Preload критических ресурсов:**
   - Убедиться, что все критические ресурсы preload'ятся
   - Проверить порядок загрузки

2. **Preconnect к внешним доменам:**
   - Добавить preconnect для внешних ресурсов (если используются)

**Ожидаемый эффект:** Улучшение FCP на 100-200ms, +1-2 пункта Score
