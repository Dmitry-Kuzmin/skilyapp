## 🔍 Выявленные проблемы

### 1. **Принудительная компоновка (Forced layout)** 🔴
**Проблема:** Браузер вынужден пересчитывать layout несколько раз

**Возможные причины:**
- Чтение layout свойств (offsetWidth, offsetHeight) во время записи
- Изменение DOM во время рендеринга
- Синхронные операции с DOM

**Решение:**
- Использовать `requestAnimationFrame` для батчинга DOM операций
- Избегать чтения layout свойств в циклах
- Использовать `will-change` для оптимизации анимаций

### 2. **Разбивка LCP (LCP breakdown)** 🔴
**Проблема:** LCP элемент загружается медленно

**Возможные причины:**
- LCP элемент (изображение или текст) загружается поздно
- Нет preload для критических ресурсов
- Большой размер LCP элемента

**Решение:**
- Preload критических изображений
- Оптимизировать размер LCP элемента
- Использовать `fetchpriority="high"` для LCP изображений

### 3. **Работа в основном потоке: 3.9 сек** 🔴
**Проблема:** JavaScript блокирует основной поток на 3.9 секунды

**Возможные причины:**
- Большой vendor.js (1.2MB) выполняется синхронно
- Тяжелые вычисления в основном потоке
- Отсутствие code splitting для некритических компонентов

**Решение:**
- Дополнительное code splitting
- Использовать Web Workers для тяжелых вычислений
- Оптимизировать React рендеринг (useMemo, useCallback)

### 4. **Кэширование: 5 KiB экономии** 🟠
**Проблема:** Неэффективный период хранения кэша

**Решение:**
- Увеличить `max-age` для статических ресурсов
- Настроить правильные заголовки Cache-Control

## 🤔 Почему Score ухудшился?

### Возможные причины:

1. **Изменения в коде:**
   - Выделение @radix-ui в отдельный chunk (потом вернули обратно)
   - Изменения в порядке загрузки модулей
   - Новые зависимости или изменения в bundle

2. **Изменения в Lighthouse:**
   - Lighthouse мог обновиться и изменить алгоритм оценки
   - Разные условия тестирования (сеть, устройство)

3. **Временные проблемы:**
   - Vercel мог деплоить старую версию
   - Кэш браузера мог влиять на результаты

## 🔧 План исправлений

### Приоритет 1: Уменьшить работу в основном потоке (TBT)

1. **Дополнительное code splitting:**
   - Выделить тяжелые библиотеки в отдельные chunks
   - Lazy load некритических компонентов

2. **Оптимизация React рендеринга:**
   - Проверить использование useMemo/useCallback
   - Оптимизировать тяжелые компоненты

3. **Web Workers (если нужно):**
   - Вынести тяжелые вычисления в Web Workers

### Приоритет 2: Исправить Forced Layout

1. **Батчинг DOM операций:**
   - Использовать `requestAnimationFrame`
   - Избегать чтения layout свойств в циклах

2. **Оптимизация анимаций:**
   - Использовать `transform` и `opacity` вместо изменения layout свойств
   - Добавить `will-change` для анимируемых элементов

### Приоритет 3: Улучшить LCP

1. **Preload критических ресурсов:**
   - Preload LCP изображения
   - Preload критических шрифтов (если есть)

2. **Оптимизация размера LCP:**
   - Уменьшить размер LCP изображения
   - Оптимизировать LCP текст

### Приоритет 4: Кэширование

1. **Настроить Cache-Control:**
   - Увеличить `max-age` для статических ресурсов
   - Правильные заголовки для разных типов ресурсов
