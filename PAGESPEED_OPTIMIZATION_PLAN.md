## 🎯 Основные проблемы (типичные для Score 65)

### 1. **FCP (First Contentful Paint)** - вероятно > 2.5s
**Проблема:** Пользователь не видит контент быстро

**Решения:**
- ✅ SSG уже работает (prerender)
- ⚠️ Нужно добавить **inline critical CSS** в `<head>`
- ⚠️ Оптимизировать загрузку шрифтов (preload, font-display: swap)
- ⚠️ Уменьшить размер JavaScript бандлов

### 2. **LCP (Largest Contentful Paint)** - вероятно > 4.0s
**Проблема:** Главный контент загружается медленно

**Решения:**
- ✅ SSG уже работает
- ⚠️ Оптимизировать изображения (WebP, размеры, lazy loading)
- ⚠️ Preload критических ресурсов (уже есть в vite.config.ts)
- ⚠️ Уменьшить JavaScript execution time

### 3. **TBT (Total Blocking Time)** - вероятно > 600ms
**Проблема:** JavaScript блокирует основной поток

**Решения:**
- ⚠️ Code splitting (уже есть, но можно улучшить)
- ⚠️ Defer non-critical JavaScript
- ⚠️ Оптимизировать React рендеринг (useMemo, useCallback)
- ⚠️ Уменьшить размер vendor.js (сейчас ~1.2MB uncompressed)

### 4. **CLS (Cumulative Layout Shift)** - вероятно < 0.1 ✅
**Статус:** Вероятно уже хорошо (нужно проверить)

### 5. **Unused JavaScript/CSS**
**Проблема:** Загружается код, который не используется

**Решения:**
- ⚠️ Tree-shaking (уже есть, но можно улучшить)
- ⚠️ Удалить неиспользуемые зависимости
- ⚠️ Lazy load тяжелых библиотек (tiptap, recharts уже вынесены)

## 🔧 План оптимизации (по приоритету)

### Приоритет 1: Критический CSS (FCP -300ms)

**Что сделать:**
1. Добавить inline critical CSS в `<head>` через Vite plugin
2. Остальной CSS загружать асинхронно (уже есть в vite.config.ts)

**Ожидаемый эффект:** FCP улучшится на 200-300ms

### Приоритет 2: Оптимизация шрифтов (FCP -200ms)

**Что сделать:**
1. Preload критических шрифтов
2. Добавить `font-display: swap` в CSS
3. Использовать `woff2` формат (самый сжатый)

**Ожидаемый эффект:** FCP улучшится на 150-200ms

### Приоритет 3: Уменьшение JavaScript бандлов (TBT -400ms)

**Что сделать:**
1. Проверить размер vendor.js (сейчас ~1.2MB uncompressed)
2. Дополнительное code splitting для тяжелых компонентов
3. Lazy load неиспользуемых библиотек
4. Оптимизировать React компоненты (useMemo, useCallback)

**Ожидаемый эффект:** TBT улучшится на 300-400ms

### Приоритет 4: Оптимизация изображений (LCP -500ms)

**Что сделать:**
1. Конвертировать все изображения в WebP
2. Добавить srcset для responsive images
3. Lazy load изображений ниже fold (уже есть LazyImage компонент)
4. Оптимизировать размеры изображений

**Ожидаемый эффект:** LCP улучшится на 400-500ms

### Приоритет 5: Preload критических ресурсов (уже частично есть)

**Что сделать:**
1. Проверить, что preload работает правильно
2. Добавить preload для критических изображений
3. Оптимизировать порядок загрузки ресурсов

**Ожидаемый эффект:** Общее улучшение на 100-200ms
