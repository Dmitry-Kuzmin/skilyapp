## 🎯 Основные проблемы (типичные для Score 65)

### 1. **FCP (First Contentful Paint)** - вероятно > 2.5s
**Проблема:** Пользователь не видит контент быстро

**Решения:**
- ✅ SSG уже работает (prerender)
- ✅ Inline critical CSS уже есть в `<head>`
- ⚠️ Оптимизировать загрузку шрифтов (preload, font-display: swap, size-adjust)
- ⚠️ Уменьшить размер JavaScript бандлов

### 2. **LCP (Largest Contentful Paint)** - вероятно > 4.0s
**Проблема:** Главный контент загружается медленно

**Решения:**
- ✅ SSG уже работает
- ⚠️ Оптимизировать изображения (WebP, размеры, **width/height атрибуты**)
- ⚠️ Preload критических ресурсов (уже есть в vite.config.ts)
- ⚠️ Добавить `fetchpriority="high"` для Hero image
- ⚠️ Уменьшить JavaScript execution time

### 3. **TBT (Total Blocking Time)** - вероятно > 600ms
**Проблема:** JavaScript блокирует основной поток

**Решения:**
- ⚠️ **КРИТИЧНО: Отложить загрузку сторонних скриптов** (Telegram widget, аналитика)
- ⚠️ Code splitting (уже есть, но можно улучшить)
- ⚠️ Defer non-critical JavaScript
- ⚠️ Оптимизировать React рендеринг (useMemo, useCallback)
- ⚠️ **Selective Hydration** для тяжелых компонентов (React 18 Suspense)
- ⚠️ Уменьшить размер vendor.js (проверить: 1.2MB - это сжатый или несжатый?)

### 4. **CLS (Cumulative Layout Shift)** - вероятно < 0.1 ✅
**Статус:** Вероятно уже хорошо, но нужно проверить

**Критично для SSG:**
- ⚠️ **Все изображения должны иметь `width` и `height` атрибуты**
- ⚠️ Использовать CSS `aspect-ratio` для динамических изображений
- ⚠️ Добавить `size-adjust` в `@font-face` для предотвращения скачков текста

### 5. **Unused JavaScript/CSS**
**Проблема:** Загружается код, который не используется

**Решения:**
- ⚠️ Tree-shaking (уже есть, но можно улучшить)
- ⚠️ Удалить неиспользуемые зависимости
- ⚠️ Lazy load тяжелых библиотек (tiptap, recharts уже вынесены)

## 🔧 План оптимизации (по приоритету - ОБНОВЛЕНО)

### Приоритет 1: JavaScript и Сторонние скрипты (TBT -400ms, Interaction)

**КРИТИЧНО: "Убийцы" производительности**

**Проблема:** Сторонние скрипты (Telegram widget, аналитика) часто весят много и выполняются в главном потоке сразу при загрузке. Они одни могут уронить Score на 15-20 пунктов (TBT).

**Что сделать:**
1. **Отложить загрузку Telegram widget** (сейчас загружается с `defer`, но можно лучше):
   - Не грузить, пока пользователь не начнет скроллить или не пройдет 3-5 секунд
   - Или загружать только при взаимодействии (клик, скролл)
   
2. **Проверить размер vendor.js:**
   - Если 1.2MB - это **сжатый** размер → катастрофа, нужно дробить агрессивнее
   - Если 1.2MB - это **несжатый** → Gzip ужмет до ~300-400кб, что приемлемо, но можно лучше
   
3. **Дополнительное code splitting:**
   - Разбить `vendor.js` через `manualChunks` в Vite
   - Lazy load неиспользуемых библиотек
   
4. **Selective Hydration (React 18):**
   - Убедиться, что тяжелые компоненты обернуты в `Suspense`
   - Оптимизировать React рендеринг (useMemo, useCallback)

5. **Опционально: Partytown** (выносит сторонние скрипты в Web Worker):
   - Для будущей аналитики (Google Analytics, Яндекс.Метрика)
   - Пока не критично, так как аналитики нет

**Ожидаемый эффект:** TBT улучшится на 300-400ms, Interaction улучшится

### Приоритет 2: Оптимизация Изображений (LCP -500ms)

**LCP — это 25% оценки Performance Score!**

**Что сделать:**
1. **Конвертировать все изображения в WebP**
   - Использовать современные форматы (WebP, AVIF для поддерживающих браузеров)
   
2. **КРИТИЧНО: Явные размеры (width/height)**
   - Все изображения должны иметь атрибуты `width` и `height` в HTML
   - Это критично для SSG, так как HTML приходит сразу, а картинки догружаются позже
   - Либо использовать CSS `aspect-ratio` для динамических изображений
   
3. **Lazy loading:**
   - Атрибут `loading="lazy"` для всего, что ниже первого экрана
   - Уже есть LazyImage компонент, но нужно проверить использование
   
4. **Hero image оптимизация:**
   - Добавить `fetchpriority="high"` для главной картинки на первом экране
   - Preload критических изображений
   
5. **Responsive images:**
   - Добавить `srcset` для разных размеров экранов
   - Оптимизировать размеры изображений

**Ожидаемый эффект:** LCP улучшится на 400-500ms

### Приоритет 3: Шрифты и CSS (FCP/CLS -200ms)

**Что сделать:**
1. **Preload критических шрифтов:**
   - Уже есть preconnect для fonts.googleapis.com
   - Добавить preload для критических шрифтов
   
2. **Font-display: swap:**
   - Добавить `font-display: swap` в CSS
   - Это предотвращает блокировку рендеринга
   
3. **КРИТИЧНО: size-adjust в @font-face:**
   - Добавить `size-adjust` в `@font-face`, чтобы шрифт-заглушка (Arial) занимал столько же места, сколько финальный шрифт
   - Это уберет скачки текста (CLS)
   
4. **Формат шрифтов:**
   - Использовать `woff2` формат (самый сжатый)
   - Убедиться, что все шрифты в woff2

**Ожидаемый эффект:** FCP улучшится на 150-200ms, CLS улучшится

### Приоритет 4: Сжатие (Compression)

**В плане не упомянуто, но критично!**

**Что сделать:**
1. **Проверить Brotli на Vercel:**
   - Brotli сжимает JS/CSS на 15-20% лучше Gzip
   - Vercel должен поддерживать Brotli автоматически, но нужно проверить
   
2. **Проверить настройки Vite:**
   - Убедиться, что build использует оптимальное сжатие
   - Проверить размеры после Gzip/Brotli

**Ожидаемый эффект:** Общее улучшение на 100-200ms

### Приоритет 5: Preload критических ресурсов (уже частично есть)

**Что сделать:**
1. Проверить, что preload работает правильно
2. Добавить preload для критических изображений
3. Оптимизировать порядок загрузки ресурсов

**Ожидаемый эффект:** Общее улучшение на 100-200ms
