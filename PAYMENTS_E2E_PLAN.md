# 💳 План E2E тестов для платежных систем

## 📋 Актуальные платежные шлюзы

### ✅ Используются:
1. **Cryptomus** - для криптовалюты
2. **Telegram Stars** - для Telegram
3. **Paddle** - для приема евро (в процессе проверки)

### ❌ Не используются:
- **Stripe** - не используется

---

## 🧪 План E2E тестов

### 1. Cryptomus (криптовалюта)

#### Edge Functions:
- `cryptomus-payment` - создание платежа
- `cryptomus-webhook` - обработка webhook

#### Тесты:
- [ ] Создание платежа через `cryptomus-payment`
- [ ] Проверка создания записи в `purchases`
- [ ] Симуляция webhook от Cryptomus (успешный платеж)
- [ ] Проверка активации Premium/начисления монет
- [ ] Симуляция webhook (неуспешный платеж)
- [ ] Проверка обработки ошибок
- [ ] Проверка идемпотентности (повторный webhook)

---

### 2. Telegram Stars

#### Edge Functions:
- `telegram-stars-payment` - создание invoice и обработка платежа
- `stars-payment-retry` - повторная попытка обработки

#### Тесты:
- [ ] Создание invoice через `telegram-stars-payment` (action: create_invoice)
- [ ] Проверка создания записи в `purchases`
- [ ] Симуляция успешного платежа через Telegram API
- [ ] Проверка активации Premium/начисления монет
- [ ] Проверка обработки ошибок
- [ ] Проверка retry механизма через `stars-payment-retry`

---

### 3. Paddle (евро)

#### Edge Functions:
- `paddle-payment` - создание платежа
- `paddle-webhook` - обработка webhook

#### Тесты:
- [ ] Создание платежа через `paddle-payment`
- [ ] Проверка создания записи в `purchases`
- [ ] Симуляция webhook от Paddle (успешный платеж)
- [ ] Проверка активации Premium/начисления монет
- [ ] Симуляция webhook (неуспешный платеж)

- [ ] Проверка идемпотентности

## 🛠 Технологии для E2E тестов

### Вариант 1: Playwright (рекомендуется)
- Поддержка API тестов
- Поддержка браузерных тестов
- Хорошая интеграция с CI/CD

### Вариант 2: Jest + Supertest
- Простые API тесты
- Легко интегрировать
- Хорошо для unit/integration тестов

### Вариант 3: Cypress
- Отличные E2E тесты для UI
- Может тестировать API
- Хорошая документация

## ⚠️ Важные моменты

1. **Тестовые ключи:** Использовать тестовые ключи для всех шлюзов
2. **Webhook симуляция:** Симулировать webhook события, не делать реальные платежи
3. **Идемпотентность:** Проверять что повторные webhook не создают дубликаты
4. **Edge cases:** Тестировать ошибки сети, таймауты, невалидные данные
