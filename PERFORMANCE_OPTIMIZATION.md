## 🛠️ Рекомендации по оптимизации

### 1. Используйте только один редактор

**Проблема:** Cursor и Antigravity одновременно
- Каждый запускает свой TypeScript сервер
- Каждый индексирует файлы проекта
- Удваивает использование памяти

**Решение:**
- ✅ Закройте один из редакторов
- ✅ Работайте в одном редакторе за раз
- ✅ Если нужно переключиться - закройте первый полностью

---

### 2. Настройте исключения для редакторов

#### Для Cursor:

Создайте/обновите `.cursorignore`:

```
# Большие файлы
data/pdf/*.pdf
*.pdf

# Кэш и временные файлы
node_modules/.vite
.vite
dist
.cache

# Логи
*.log
logs/

# Git
.git/

# Системные файлы
.DS_Store
Thumbs.db
```

#### Для Antigravity:

Создайте `.antigravityignore` (аналогично `.cursorignore`)

---

### 3. Ограничьте индексирование TypeScript

#### В Cursor:
1. Откройте настройки (Cmd + ,)
2. Найдите `typescript.tsserver.maxTsServerMemory`
3. Установите значение: `3072` (3GB)

#### В Antigravity:
Аналогично настройте лимит памяти для TypeScript сервера

---

### 4. Используйте скрипт оптимизации

Запускайте перед началом работы:

```bash
./optimize-system.sh
```

Или вручную:

```bash
# Остановить все dev server'ы
pkill -f "vite"
pkill -f "npm run dev"

# Очистить кэш
rm -rf node_modules/.vite
rm -rf .vite

# Удалить .DS_Store
find . -name ".DS_Store" -delete
```

---

### 5. Настройте Vite для меньшего использования ресурсов

Создайте/обновите `vite.config.ts`:

```typescript
export default defineConfig({
  // ... существующие настройки ...
  
  server: {
    // Ограничить количество одновременных соединений
    hmr: {
      overlay: false, // Отключить overlay для экономии ресурсов
    },
  },
  
  build: {
    // Оптимизация сборки
    chunkSizeWarningLimit: 1000,
    rollupOptions: {
      output: {
        manualChunks: {
          // Разделить большие библиотеки на отдельные чанки
          'vendor-react': ['react', 'react-dom'],
          'vendor-router': ['react-router-dom'],
        },
      },
    },
  },
  
  // Отключить source maps в dev (экономит память)
  // sourcemap: false, // Раскомментируйте если проблемы продолжаются
});
```

### 6. Мониторинг ресурсов

#### Проверка процессов:

```bash
# Проверить все Node процессы
ps aux | grep -E "(node|npm|vite)" | grep -v grep

# Проверить использование памяти
top -l 1 -n 10 -stats pid,command,cpu,mem

# Проверить Load Average
uptime
```

#### Нормальные значения:
- **Load Average:** < 4.0 (для 4-ядерного процессора)
- **Память Node:** < 500MB на процесс
- **CPU:** < 50% в idle

## 🚨 Экстренные меры

