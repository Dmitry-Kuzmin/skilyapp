## 🧠 Инструкции для ИИ (Agent Skills)

Для обеспечения максимальной производительности и безопасности при работе с Supabase, следуй этим правилам (основано на Supabase Agent Skills):

### 🗄️ Работа с базой данных (PostgreSQL)
1. **Минимизация запросов:** Всегда старайся объединять запросы через JOIN или использовать RPC для сложных выборок. Одиночные запросы в цикле строго запрещены.
2. **Типизация:** Используй только сгенерированные типы из `src/integrations/supabase/types.ts`. Никаких `any` или ручных интерфейсов для таблиц.
3. **Проверка существования:** Используй `.select('*', { count: 'exact', head: true })` если нужно только проверить наличие записи, не загружая её.
4. **RLS (Row Level Security):** При создании новых таблиц ВСЕГДА создавай RLS политики. Без политик данные открыты — это недопустимо.
5. **Атомарные операции:** Для счетчиков (XP, Coins) всегда используй `increment` через RPC или SQL (`SET coins = coins + val`), а не перезаписывай значение с фронтенда.

### ⚡ Edge Functions (Deno)
1. **Производительность:** Функция должна выполняться < 300мс.
2. **Безопасность:** Проверяй JWT/Auth во всех критических функциях.
3. **Обработка ошибок:** Всегда возвращай понятные JSON-ошибки с правильными HTTP кодами.

### 🧩 UI/UX и Фронтенд
1. **React Query:** Используй `staleTime` и `gcTime` для минимизации лишних сетевых вызовов.
2. **Оффлайн:** Учитывай, что пользователь может быть оффлайн. Используй `onMutate` для Optimistic UI.
3. **Виртуализация:** Списки более 50 элементов должны использовать виртуализацию.
