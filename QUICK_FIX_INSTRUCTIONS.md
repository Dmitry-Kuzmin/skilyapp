# 🚨 СРОЧНОЕ ИСПРАВЛЕНИЕ: RLS политика для transactions

## Проблема
В логах видна ошибка:
```
code: "42501"
message: "new row violates row-level security policy for table \"transactions\""
```

Это означает, что пользователи не могут создавать транзакции при покупке бустов.

## Решение (2 минуты)

### Вариант 1: Через Supabase Dashboard (РЕКОМЕНДУЕТСЯ)

1. **Откройте SQL Editor:**
   ```
   https://supabase.com/dashboard/project/ijijcrucqqnnjbkclqhb/sql/new
   ```

2. **Скопируйте весь код из файла `APPLY_NOW.sql`** и вставьте в редактор

3. **Нажмите "Run"** или `Ctrl+Enter` (Mac: `Cmd+Enter`)

4. **Проверьте результат** - должно быть сообщение об успехе

### Вариант 2: Через Supabase CLI

```bash
# Если у вас установлен Supabase CLI
supabase db push --include-all
```

## После применения

1. ✅ Покупки бустов будут сохраняться в БД
2. ✅ Транзакции будут отображаться в истории
3. ✅ Инвентарь будет обновляться корректно

## Проверка

После применения миграции:
1. Откройте Telegram Web App
2. Купите буст
3. Проверьте консоль - ошибка RLS должна исчезнуть
4. Проверьте историю транзакций - должна появиться запись
5. Проверьте инвентарь - буст должен сохраниться

## Если что-то не работает

Проверьте логи в консоли браузера:
- Должно быть: `[BoostShop] ✅ Транзакция создана успешно`
- Не должно быть: `code: "42501"`
