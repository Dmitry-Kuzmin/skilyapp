# 🏆 План минимальной ранговой системы

## ✅ Принцип: Упрощение без перегрузки

**Используем существующую систему Duel Pass уровня вместо создания новой RP системы.**

---

## 📊 Система рангов (на основе Duel Pass уровня)

### Маппинг уровней → рангов:

| Duel Pass Уровень | Ранг | Название |
|-------------------|------|----------|
| 1-5 | Rookie | Новичок |
| 6-10 | Bronze | Бронза |
| 11-15 | Silver | Серебро |
| 16-20 | Gold | Золото |
| 21-25 | Platinum | Платина |
| 26-30 | Diamond | Алмаз |
| Топ 100 по SP | Master | Мастер |

### Преимущества:
- ✅ Использует существующую систему SP
- ✅ Не требует новых таблиц для RP
- ✅ Легко балансировать
- ✅ Быстрое внедрение (1-2 недели)

---

## 🎁 Награды по рангам

### Бесплатные награды:
- **Rookie** → Базовая рамка профиля
- **Bronze** → Улучшенная рамка
- **Silver** → Редкая рамка
- **Gold** → Скин Skily (базовый)
- **Platinum** → Скин Skily (редкий)
- **Diamond** → Скин Skily (эпический)
- **Master** → Титул "Мастер сезона"

### Premium награды (Duel Pass Premium):
- **Gold+** → Эксклюзивная рамка
- **Platinum+** → Редкий скин Skily
- **Diamond+** → Эпический скин Skily + титул
- **Master+** → Легендарный скин + эксклюзивный титул

## 📋 Что нужно реализовать

### 1. Таблица `rank_rewards` (миграция)
```sql
CREATE TABLE rank_rewards (
  rank TEXT PRIMARY KEY, -- 'rookie', 'bronze', 'silver', etc.

);
```

CREATE FUNCTION get_user_rank(p_user_id UUID, p_season_id INTEGER)

  -- Логика: уровень Duel Pass → ранг

### Рамки рангов:

- **Diamond**: Алмазная рамка с анимацией
- **Master**: Эксклюзивная рамка с эффектами

### Титулы:
- Отображаются под ником в турнирной таблице
- Формат: "Титул | Ранг"
- Пример: "Мастер сезона | Diamond"

## ⚠️ Важные моменты

1. **Не создаем отдельную RP систему** - используем SP
2. **Ранги пересчитываются каждый сезон** - на основе текущего уровня Duel Pass
3. **Награды выдаются один раз** - при первом достижении ранга
4. **Master ранг** - только для топ 100 по SP в конце сезона
