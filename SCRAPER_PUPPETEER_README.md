# Скрапер PracticaVial с Puppeteer

Автоматический скрапер тестов с сайта PracticaVial, использующий Puppeteer для работы как реальный браузер.

## Преимущества Puppeteer

✅ **Автоматическая авторизация** - не нужно вручную копировать cookies  
✅ **Работает с JavaScript** - загружает динамический контент  
✅ **Надежность** - работает как настоящий браузер  
✅ **Простота использования** - один скрипт для всего процесса  

## Установка

1. **Установите зависимости:**

```bash
npm install
```

Это установит:
- `puppeteer` - автоматизация браузера
- `@supabase/supabase-js` - работа с базой данных
- `xlsx` - генерация Excel файлов
- `dotenv` - загрузка переменных окружения

2. **Настройте переменные окружения:**

Убедитесь, что в файле `.env` (в корне проекта) есть следующие переменные:

```env
VITE_SUPABASE_URL=your-supabase-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
```

Или используйте переменные:
```env
SUPABASE_URL=your-supabase-url
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

## Использование

### Способ 1: Интерактивный режим

Просто запустите скрипт, и он запросит логин и пароль:

```bash
npm run scrape:practicavial
```

### Способ 2: С параметрами командной строки

```bash
npm run scrape:practicavial -- --username=your-email@example.com --password=your-password
```

### Способ 3: С выбором тем

```bash
npm run scrape:practicavial -- --username=your-email@example.com --password=your-password --topics=1,2,3
```

### Способ 4: С видимым браузером (для отладки)

```bash
npm run scrape:practicavial -- --username=your-email@example.com --password=your-password --headless=false
```

### Способ 5: С указанием имени выходного файла

```bash
npm run scrape:practicavial -- --username=your-email@example.com --password=your-password --output=my-scrape.xlsx
```

## Параметры командной строки

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--username=USERNAME` | Логин/Email для PracticaVial | `--username=user@example.com` |
| `--password=PASSWORD` | Пароль для PracticaVial | `--password=pass123` |
| `--topics=1,2,3` | Номера тем для скрапинга (через запятую) | `--topics=1,2,3` |
| `--headless=false` | Запустить браузер в видимом режиме | `--headless=false` |
| `--output=FILENAME` | Имя выходного Excel файла | `--output=my-scrape.xlsx` |
| `--help` или `-h` | Показать справку | `--help` |

## Как это работает

1. **Авторизация**: Скрипт автоматически открывает браузер, переходит на страницу логина и авторизуется
2. **Загрузка тем**: Получает список всех тем и тестов с сайта
3. **Скрапинг**: Для каждой темы и теста:
   - Открывает страницу теста
   - Извлекает вопросы, ответы и изображения
   - Сохраняет данные
4. **Сохранение**: 
   - Сохраняет все вопросы в базу данных Supabase
   - Генерирует Excel файл с результатами

## Результаты

После выполнения скрипт создаст:

1. **Excel файл** (`practicavial-scrape.xlsx` по умолчанию) с двумя листами:
   - **Вопросы**: Список всех вопросов с текстом и изображениями
   - **Ответы**: Список всех ответов с указанием правильных

2. **Данные в базе данных**:
   - Вопросы сохраняются в таблицу `questions_new`
   - Ответы сохраняются в таблицу `answer_options`
   - Автоматически связываются с темами по номеру

## Отладка

Если что-то не работает:

1. **Запустите с видимым браузером:**
   ```bash
   npm run scrape:practicavial -- --headless=false
   ```
   Это позволит увидеть, что происходит в браузере

2. **Проверьте логи**: Скрипт выводит подробную информацию о каждом шаге

3. **Проверьте учетные данные**: Убедитесь, что логин и пароль правильные

4. **Проверьте подключение к Supabase**: Убедитесь, что переменные окружения настроены правильно

## Устранение неполадок

### Ошибка: "Не удалось найти поля ввода логина и пароля"

**Решение**: Сайт мог изменить структуру. Запустите с `--headless=false` и проверьте, как выглядит форма логина.

### Ошибка: "Авторизация не удалась"

**Решение**: 
- Проверьте правильность логина и пароля
- Убедитесь, что у вас есть премиум доступ
- Попробуйте авторизоваться вручную в браузере

### Ошибка: "Не удалось найти темы на странице"

**Решение**: 
- Убедитесь, что вы авторизованы
- Проверьте, что у вас есть доступ к тестам
- Запустите с `--headless=false` для отладки

### Ошибка: "Не найдены переменные окружения"

**Решение**: Убедитесь, что файл `.env` существует в корне проекта и содержит правильные переменные.
