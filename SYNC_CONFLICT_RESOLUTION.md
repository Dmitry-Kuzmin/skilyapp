# Разрешение конфликтов при синхронизации прогресса

## ❓ Проблема

**Вопрос пользователя**: "Если я прошел тест оффлайн, а потом зашел с другого устройства, не затрет ли старая версия с сервера мой новый локальный прогресс при синхронизации?"

**Ответ**: Раньше да, могло. Теперь нет! Реализована стратегия слияния (merging strategy) для предотвращения потери данных.

## ✅ Решение: Стратегия слияния (Merging Strategy)

### Как это работает

1. **Версионирование по времени**
   - Каждый ответ имеет метку времени (timestamp)
   - При синхронизации сравниваются временные метки

2. **Приоритет свежести**
   - Самый свежий ответ всегда побеждает
   - Если локальный ответ новее серверного → отправляем на сервер
   - Если серверный ответ новее → используем серверный

3. **Объединение уникальных ответов**
   - Локальные и серверные ответы объединяются
   - Нет дубликатов - каждый вопрос только один раз
   - Не теряются ответы с разных устройств

4. **Безопасное слияние**
   - Если есть конфликт (разные ответы на один вопрос) → используется новейший
   - Все ответы сохраняются - ничего не теряется

### Пример работы

**Сценарий**: Пользователь ответил на вопросы 1-5 на устройстве A оффлайн, затем ответил на вопросы 3-7 на устройстве B (онлайн).

```
Устройство A (оффлайн):
  Вопрос 1: ответ A, timestamp: 10:00
  Вопрос 2: ответ B, timestamp: 10:05
  Вопрос 3: ответ C, timestamp: 10:10
  Вопрос 4: ответ D, timestamp: 10:15
  Вопрос 5: ответ E, timestamp: 10:20

Устройство B (онлайн):
  Вопрос 3: ответ X, timestamp: 11:00  ← новее!
  Вопрос 4: ответ Y, timestamp: 11:05  ← новее!
  Вопрос 5: ответ Z, timestamp: 11:10  ← новее!
  Вопрос 6: ответ F, timestamp: 11:15
  Вопрос 7: ответ G, timestamp: 11:20

Результат слияния:
  Вопрос 1: ответ A (только с устройства A)
  Вопрос 2: ответ B (только с устройства A)
  Вопрос 3: ответ X (сервер новее - с устройства B)
  Вопрос 4: ответ Y (сервер новее - с устройства B)
  Вопрос 5: ответ Z (сервер новее - с устройства B)
  Вопрос 6: ответ F (только с устройства B)
  Вопрос 7: ответ G (только с устройства B)
```

**Итог**: Никакие данные не потеряны, используются новейшие ответы.

## 📋 Реализация

### Файл: `src/utils/testSync.ts`

Основные функции:

1. **`syncTestProgress()`** - Синхронизация с разрешением конфликтов
   - Загружает серверный прогресс
   - Сливает локальный и серверный прогресс
   - Отправляет новейшие ответы на сервер
   - Возвращает информацию о конфликтах

2. **`mergeProgress()`** - Слияние прогрессов
   - Сравнивает временные метки
   - Выбирает новейшие ответы
   - Объединяет уникальные ответы

3. **`loadAndMergeProgress()`** - Загрузка и слияние при старте теста
   - Загружает локальный прогресс
   - Загружает серверный прогресс
   - Объединяет их для отображения

### Интеграция в TestSession

```typescript
// При восстановлении сети
const syncResult = await syncTestProgress(profileId, localProgress);

if (syncResult.success) {
  if (syncResult.conflicts > 0) {
    toast.success(`Прогресс синхронизирован`, {
      description: `Разрешено ${syncResult.conflicts} конфликтов. Отправлено ${syncResult.syncedAnswers} ответов.`,
    });
  }
}
```

## 🔐 Гарантии

### ✅ Не теряются данные

- Все ответы сохраняются (локальные и серверные)
- Объединяются уникальные ответы
- Новейшие ответы всегда побеждают

### ✅ Корректная синхронизация

- Проверка временных меток
- Разрешение конфликтов автоматически
- Информация о количестве конфликтов

### ✅ Работает между устройствами

- Прогресс синхронизируется между устройствами
- Нет конфликтов при работе с разных устройств
- Все ответы доступны везде

## 📊 Статистика синхронизации

После синхронизации возвращается:
- `success` - успешность операции
- `syncedAnswers` - количество отправленных ответов
- `conflicts` - количество разрешенных конфликтов
- `mergedAnswers` - объединенный список ответов

## 🚀 Примеры использования

### 1. Синхронизация при восстановлении сети

```typescript
// Автоматически при восстановлении сети
const syncResult = await syncTestProgress(userId, localProgress);
```

### 2. Загрузка прогресса при старте теста

```typescript
// При загрузке теста - объединяем локальный и серверный
const { mergedAnswers, hasConflicts } = await loadAndMergeProgress(
  userId,
  testId,
  questionIds
);
```

### 3. Проверка конфликтов

```typescript
if (syncResult.conflicts > 0) {
  console.log(`Разрешено ${syncResult.conflicts} конфликтов`);
  // Показать уведомление пользователю
}
```
