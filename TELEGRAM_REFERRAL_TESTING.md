# Как протестировать реферальную ссылку в Telegram Web App

## Проблема
Ссылка `https://skilyapp.com/join/E2A9AB` работает в обычном браузере, но не работает при открытии через Telegram Web App.

## Решение

### Вариант 1: Прямая ссылка (через браузер Telegram)
Если пользователь открывает ссылку `https://skilyapp.com/join/E2A9AB` через Telegram (кликая на ссылку в сообщении), она откроется в браузере Telegram, а не в Web App.

**Как протестировать:**
1. Открой Telegram на телефоне
2. Отправь себе или кому-то сообщение со ссылкой: `https://skilyapp.com/join/E2A9AB`
3. Нажми на ссылку — она откроется в браузере Telegram
4. Проверь, что код сохранился и происходит редирект

### Вариант 2: Через Telegram Web App (рекомендуемый)
Чтобы ссылка открывалась **внутри Web App**, нужно использовать специальный формат.

**Формат ссылки:**
```
https://t.me/ВАШ_БОТ/app?startapp=ref_КОД
```

**Для твоего бота (`sdadimtutbot`) и кода `E2A9AB`:**
```
https://t.me/sdadimtutbot/app?startapp=ref_E2A9AB
```

**Как протестировать:**

#### Способ 1: Через кнопку в боте
1. Открой бота в Telegram: `@sdadimtutbot`
2. Отправь команду `/start`
3. Если у тебя есть кнопка, которая открывает Web App с этой ссылкой — нажми на неё

#### Способ 2: Через сообщение с кнопкой
Создай сообщение в боте с inline-кнопкой:
```json
{
  "text": "Открыть с реферальным кодом",
  "web_app": {
    "url": "https://skilyapp.com/join/E2A9AB"
  }
}
```

Или через startapp параметр:
```json
{
  "text": "Открыть с реферальным кодом",
  "web_app": {
    "url": "https://skilyapp.com?startapp=ref_E2A9AB"
  }
}
```

#### Способ 3: Прямая ссылка в браузере (для теста)
Открой эту ссылку в браузере на телефоне:
```
https://t.me/sdadimtutbot/app?startapp=ref_E2A9AB
```

Телеграм спросит, открыть ли это в Web App — выбери "Да".

### Вариант 3: Через страницу рефералов в приложении
В приложении уже есть функция "Поделиться" на странице `/referrals`:

1. Открой приложение: `https://skilyapp.com`
2. Зайди на страницу рефералов: `/referrals`
3. Нажми кнопку "Поделиться"
4. Если ты в Telegram Web App, откроется диалог шаринга с правильной ссылкой

## Что изменилось в коде

Компонент `ReferralRedirect` теперь:
1. ✅ Показывает визуальную обратную связь (экран загрузки)
2. ✅ Использует `window.location` для редиректа в Telegram (надежнее)
3. ✅ Обрабатывает код из URL: `/join/E2A9AB`
4. ✅ Обрабатывает код из start_param: `ref_E2A9AB`
5. ✅ Обрабатывает код из query параметра: `?startapp=ref_E2A9AB`

## Быстрый тест

**1. Протестируй в обычном браузере:**
```
https://skilyapp.com/join/E2A9AB
```
Должно: показать экран загрузки → сохранить код → редирект на главную

**2. Протестируй через Telegram Web App ссылку:**
```
https://t.me/sdadimtutbot/app?startapp=ref_E2A9AB
```
Должно: открыть Web App → показать экран загрузки → сохранить код → редирект

**3. Проверь сохранение кода:**
После открытия ссылки проверь в DevTools (Console):
```javascript
sessionStorage.getItem('referral_code')
```
Должно вернуть: `E2A9AB`

## Важные замечания

⚠️ **Разница между ссылками:**

- `https://skilyapp.com/join/E2A9AB` 
  - Открывается в браузере Telegram (не в Web App)
  - Работает как обычный сайт
  
- `https://t.me/sdadimtutbot/app?startapp=ref_E2A9AB`
  - Открывается внутри Telegram Web App
  - Полный доступ к Telegram API
  - Лучший UX для пользователей

✅ **Рекомендация:** Используй ссылки формата `https://t.me/БОТ/app?startapp=ref_КОД` для шаринга в Telegram, а обычные ссылки `https://skilyapp.com/join/КОД` для шаринга в других местах (WhatsApp, SMS, email и т.д.).

## Настройка бота

Если нужно изменить username бота, установи переменную окружения:
```env
VITE_TELEGRAM_BOT_USERNAME=sdadimtutbot
```

Сейчас в коде используется хардкод `sdadimtutbot` в `ProfileModal.tsx`, но в `Referrals.tsx` используется переменная окружения.
