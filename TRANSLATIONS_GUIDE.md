# Руководство по организации переводов материалов

## Текущая реализация

### Структура хранения переводов

Переводы хранятся прямо в JSON файлах материалов в формате:
```json
{
  "id": "subtopic-2-1",
  "title_ru": "Заголовок на русском",
  "title_es": "Título en español",
  "title_en": "Title in English",
  "content_ru": "<div>...</div>",
  "content_es": "<div>...</div>",
  "content_en": "<div>...</div>"
}
```

### Расположение файлов

- **Статические материалы**: `public/data/materials/topic-{number}/subtopic-{code}.json`
- **Материалы в Supabase**: таблица `materials` с полями `content_ru`, `content_es`, `content_en`

### Автоматический выбор языка

`MaterialViewer` автоматически использует язык из `LanguageContext`:
1. Если передан проп `language` → используется он
2. Иначе используется язык из контекста приложения
3. Если перевода нет → показывается fallback (испанский/русский/английский)

### Fallback логика

При отсутствии перевода для выбранного языка:
- **Русский**: показывает `content_ru` → `content_es` → `content_en`
- **Испанский**: показывает `content_es` → `content_ru` → `content_en`
- **Английский**: показывает `content_en` → `content_es` → `content_ru`

## Рекомендации по организации переводов

### Вариант 1: Хранение в JSON файлах (текущий) ✅ Рекомендуется

**Плюсы:**
- Простота реализации
- Быстрая загрузка (статичные файлы)
- Легко редактировать
- Не требует изменений в базе данных

**Минусы:**
- Дублирование структуры для каждого языка
- Большой размер файлов

**Когда использовать:**
- Статичные материалы учебника
- Материалы, которые редко меняются
- Когда нужна быстрая загрузка без запросов к БД

### Вариант 2: Хранение в Supabase

**Плюсы:**
- Централизованное хранение
- Легко обновлять через админ-панель
- Можно добавить версионирование

**Минусы:**
- Требует запросов к БД
- Медленнее загрузка

**Когда использовать:**
- Динамические материалы
- Материалы, которые часто обновляются
- Когда нужна админ-панель для редактирования

### Вариант 3: Отдельные файлы переводов

Создать структуру:
```
public/data/materials/
  topic-2/
    subtopic-2-1/
      content.es.json
      content.ru.json
      content.en.json
```

**Плюсы:**
- Четкое разделение языков
- Легко добавлять новые языки
- Можно загружать только нужный язык

**Минусы:**
- Сложнее структура
- Требует изменений в коде загрузки

## Процесс добавления переводов

### Шаг 1: Подготовка материала

1. Создай базовый материал на испанском (оригинал)
2. Сохрани в JSON файл с полями `content_es`, `title_es`

### Шаг 2: Перевод на русский

1. Скопируй `content_es` в `content_ru`
2. Переведи содержимое HTML, сохраняя структуру:
   - Заголовки (`<h1>`, `<h2>`, etc.)
   - Классы CSS (`class="info-box"`, etc.)
   - Структуру HTML

### Шаг 3: Перевод на английский (опционально)

Аналогично шагу 2, но в `content_en`

### Шаг 4: Проверка

1. Открой материал в приложении
2. Переключи язык в настройках
3. Убедись, что перевод отображается корректно
4. Проверь, что fallback работает, если перевода нет

## Пример структуры переведенного материала

```json
{
  "id": "subtopic-2-1",
  "code": "2.1",
  "topic_id": "topic-2",
  "title_ru": "Список различных манёвров – шаги для выполнения манёвра",
  "title_es": "Lista de las distintas maniobras – pasos a seguir para realizar una maniobra",
  "title_en": "List of different maneuvers – steps to follow to perform a maneuver",
  "content_ru": "<div class=\"material-content-wrapper\">\n<h1 class=\"material-title\">Список различных манёвров</h1>\n...",
  "content_es": "<div class=\"material-content-wrapper\">\n<h1 class=\"material-title\">Lista de las distintas maniobras</h1>\n...",
  "content_en": "<div class=\"material-content-wrapper\">\n<h1 class=\"material-title\">List of different maneuvers</h1>\n...",
  "images": [],
  "order": 1,
  "source_pdf": "https://..."
}
```

## Индикация отсутствия перевода

Если перевода нет, MaterialViewer показывает:
- Badge с предупреждением: "Перевод на русский недоступен"
- Информацию о языке оригинала: "Показан оригинал (испанский)"

## Рекомендации по переводу

1. **Сохраняй структуру HTML** - не меняй классы и теги
2. **Переводи только текст** - оставляй технические термины на языке оригинала, если они специфичны
3. **Проверяй форматирование** - убедись, что списки, таблицы и блоки отображаются корректно
4. **Тестируй на разных языках** - проверь все три языка перед публикацией
