# 🎨 DGT Image Validator - Руководство

## Что это?

Интерфейс в стиле Tinder для быстрой проверки генерированных AI изображений.

## Быстрый старт

### 1. Запустить сервер

```bash
node validator-server.js
```

### 2. Открыть браузер

Перейти на: **http://localhost:3030**

## Использование

### Горячие клавиши

- **Y** — Одобрить (✅)
- **N** — Отклонить (❌)  
- **R** — Перегенерировать (🔄)
- **→** — Пропустить (⏭️)

### Процесс валидации

1. **Сравни** оригинал (слева) и AI-версию (справа)
2. **Реши** что делать:
   - ✅ **Одобрить** — AI отлично справился, картинка идет в продакшн
   - ❌ **Отклонить** — AI накосячил, нужна новая генерация
   - 🔄 **Перегенерировать** — Почти хорошо, но нужна доработка (напиши промпт)
   - ⏭️ **Пропустить** — Не уверен, решу потом

## Файлы проекта

```
validator-server.js        # Backend (Express API)
validator-ui.html          # Frontend (UI)
data/
  generated-images/        # Папка с AI-изображениями
    originals/             # Оригинальные от PracticaVial
  validation-decisions.json # Твои решения
```

## API Endpoints

### GET /api/images
Получить список изображений для проверки

**Response:**
```json
{
  "images": [
    {
      "id": "abc-123",
      "originalUrl": "/data/.../original.jpg",
      "generatedUrl": "/data/.../abc-123.png",
      "question": "Текст вопроса"
    }
  ],
  "stats": {
    "total": 1000,
    "pending": 850,
    "approved": 100,
    "rejected": 30,
    "toRegenerate": 20
  }
}
```

### POST /api/decision
Сохранить решение по изображению

**Request:**
```json
{
  "id": "abc-123",
  "action": "approve",  // или "reject", "regenerate"
  "additionalPrompt": "Добавь велосипедиста справа"
}
```

### GET /api/stats
Получить статистику

## Результаты

Все твои решения сохраняются в `data/validation-decisions.json`:

```json
{
  "approved": ["id1", "id2", ...],
  "rejected": ["id3", "id4", ...],
  "toRegenerate": [
    {
      "id": "id5",
      "additionalPrompt": "Убери второй знак"
    }
  ]
}
```

## Что дальше?

После валидации:

1. **Одобренные** → Скрипт загрузки в Supabase
2. **Отклоненные** → Удаляем и перегенерируем
3. **К перегенерации** → Скрипт дополнит промпт и сгенерирует заново

## Troubleshooting

**Картинки не загружаются?**
- Убедись, что сервер запущен (`node validator-server.js`)
- Проверь, что папка `data/generated-images` содержит файлы

**Ошибка "API not found"?**
- Перезапусти сервер
- Проверь порт 3030 (должен быть свободен)

**Нет изображений для проверки?**
- Сначала запусти генератор: `node scripts/generate-images-batch.js --limit=10`
- Дождись завершения
- Обнови страницу валидатора
