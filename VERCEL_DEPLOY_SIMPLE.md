# ✅ Простое решение: Автоматический деплой

## 🎯 Проблема
Ручной деплой через "Create Deployment" не работает, но репозиторий подключен.

## ✅ Решение: Автоматический деплой через Git Push

### Способ 1: Пустой коммит (самый простой)

Просто сделай push в main - Vercel автоматически создаст деплой:

```bash
# Создай пустой коммит для триггера
git commit --allow-empty -m "Trigger Vercel auto-deploy"
git push origin main
```

**Что произойдёт:**
- ✅ GitHub получит коммит
- ✅ Vercel увидит push (репозиторий подключен)
- ✅ Автоматически запустит деплой через 1-2 минуты
- ✅ Появится новый деплой в Vercel Dashboard

### Способ 2: Redeploy последнего деплоя

Если хочешь пересобрать текущую версию без коммита:

1. Vercel Dashboard → **Deployments**
2. Найди последний деплой (даже старый)
3. Нажми **⋯** (три точки) рядом с деплоем
4. Выбери **Redeploy**
5. Выбери **"Use existing Build Cache"** = **OFF**
6. Нажми **Redeploy**

Это пересоберёт последний деплой без нового коммита.

---

## 🔍 Проверка что автодеплой работает

После push проверь:

1. **Vercel Dashboard** → **Deployments**
2. Должен появиться новый деплой через 1-2 минуты
3. Статус будет **Building...** затем **Ready**

---

## 🆘 Если автодеплой не запустился

### Проверь подключение репозитория:

1. **Settings** → **Git**
2. Убедись что:
   - ✅ Репозиторий: `Dmitry-Kuzmin/sdadim-dgt-prep`
   - ✅ Production Branch: `main`
   - ✅ Auto-deploy: включен

### Проверь вебхук в GitHub:

1. GitHub: https://github.com/Dmitry-Kuzmin/sdadim-dgt-prep/settings/hooks
2. Ищи вебхук от Vercel (обычно `vercel.com` или `vercel.app`)
3. Проверь последнюю доставку - должна быть недавно (после push)

**Если вебхука нет:**
- Переподключи репозиторий (Settings → Git → Disconnect → Connect)

## 💡 Рекомендация

**Используй Способ 1 (пустой коммит)** - это самый простой и надёжный способ:
1. Быстро (одна команда)
2. Проверяет что автодеплой работает
3. Создаёт новый деплой с актуальным кодом
