# 🔧 Исправление Puppeteer для Vercel

**Дата:** 5 декабря 2025  
**Проблема:** Puppeteer не может найти Chrome на Vercel

## 🔴 Проблема

```
Error: Could not find Chrome (ver. 143.0.7499.40)
```

**Причина:** Vercel не имеет Chrome установленного по умолчанию, или он в другом месте.

## ✅ Решения

### Вариант 1: Использовать системный Chrome (рекомендуется)
Vercel имеет Chrome в системе, но нужно указать правильный путь.

### Вариант 2: Установить Chrome через puppeteer browsers
Можно добавить установку Chrome в build процесс.

### Вариант 3: Использовать альтернативный подход
- Запускать prerender только локально
- Использовать Vercel's built-in prerendering (если доступно)
- Использовать другой инструмент для SSG

## 🎯 Выбранное решение

Используем **Вариант 1** с fallback:
- Пытаемся найти системный Chrome
- Используем правильные аргументы для Vercel
- Добавляем обработку ошибок

## 📝 Изменения

### scripts/prerender.js:
- Добавлена проверка Vercel окружения
- Поиск системного Chrome
- Правильные аргументы для Vercel
- Обработка ошибок

### package.json:
- `build` - обычная сборка (без prerender)
- `build:vercel` - сборка с prerender (для Vercel)

## ⚠️ Альтернатива

Если это не сработает, можно:
1. Использовать `@vercel/static` для статической генерации
2. Запускать prerender только локально перед деплоем
3. Использовать другой SSG подход
