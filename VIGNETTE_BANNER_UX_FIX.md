# ✅ Исправление UX: Vignette Banner перенесен на управляемый показ

## 🎯 Проблема:

Vignette Banner загружался автоматически на всех страницах веб-версии, включая дашборд. Это создавало плохой UX:
- Баннер появлялся каждый раз при возврате на дашборд
- Перекрывал контент и требовал закрытия
- Раздражал пользователей частыми показами

---

## ✅ Решение:

### 1. Убрана автоматическая загрузка из `index.html`
- Удален код автоматической загрузки Vignette Banner
- Теперь баннер загружается только по требованию

### 2. Создан хук `useVignetteBanner`
- **Расположение:** `src/hooks/useVignetteBanner.ts`
- **Функциональность:**
  - Показывает баннер только в веб-версии (НЕ в Telegram Mini App)
  - Показывает максимум один раз за сессию (sessionStorage)
  - Поддерживает задержку перед показом
  - Проверяет, не загружен ли уже скрипт

### 3. Интеграция в правильные места:

#### ✅ TestResults (результаты теста)
- Показывается после завершения теста
- Задержка 1.5 секунды (чтобы не перекрывать анимацию результатов)
- Один раз за сессию

#### ✅ DuelResult (результаты дуэли)
- Показывается после завершения дуэли
- Задержка 1.5 секунды
- Один раз за сессию

---

## 📋 Где показывается Vignette Banner:

1. ✅ **После завершения теста** (`TestResults.tsx`)
   - Игрок только что закончил тест
   - На экране результатов
   - Идеальный момент для показа

2. ✅ **После завершения дуэли** (`DuelResult.tsx`)
   - Игрок только что закончил дуэль
   - На экране результатов
   - Идеальный момент для показа

3. ❌ **НЕ показывается на дашборде**
   - Убрана автоматическая загрузка
   - Пользователи не будут раздражены частыми показами

---

## 🔧 Технические детали:

### Хук `useVignetteBanner`:
```typescript
useVignetteBanner(show: boolean, delay: number = 500)
```

**Параметры:**
- `show` - показывать ли баннер (например, `!!results`)
- `delay` - задержка перед показом в миллисекундах (по умолчанию 500ms)

**Особенности:**
- Проверка Telegram Mini App (не показывается в Telegram)
- Проверка sessionStorage (один раз за сессию)
- Проверка на уже загруженный скрипт
- Автоматическая очистка timeout при размонтировании

### SessionStorage ключ:
- `monetag_vignette_shown` - помечает, что баннер уже показан в текущей сессии

## 🎯 Результат:

### ✅ До:
- Баннер появлялся на всех страницах
- Раздражал пользователей на дашборде
- Плохой UX

### ✅ После:
- Баннер показывается только после завершения теста/дуэли
- Один раз за сессию
- Не мешает пользователям на дашборде
- Хороший UX
