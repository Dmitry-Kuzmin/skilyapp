## Обзор

### Цель промпта
Преобразовать сырые вопросы DGT (только на испанском) в премиальный триязычный образовательный контент с:
- Профессиональными переводами (ES/EN/RU)
- Экспертными объяснениями
- Критическими отличиями ПДД (РФ vs Испания)
- Мнемотехниками для запоминания

### Целевая аудитория
Водители категории B (легковой автомобиль), готовящиеся к экзамену DGT в Испании, с опытом вождения в РФ/СНГ.

## Архитектура промпта

### 1. Конфигурация модели

```javascript
const model = genAI.getGenerativeModel({
    model: "gemini-3.1-pro-preview",  // Самая умная модель
    generationConfig: {
        responseMimeType: "application/json",
        responseSchema: responseSchema,  // Гарантированная структура
        temperature: 0.2  // Строгое следование инструкциям
    }
});
```

**Почему эти параметры:**
- **gemini-3.1-pro-preview**: Максимальное качество рассуждений + мультимодальность
- **temperature 0.2**: Минимум "творчества", максимум точности к словарю терминов
- **JSON Schema**: 100% гарантия корректной структуры без постобработки

## Критические компоненты

### 🎯 Фильтр категории B

**Проблема:** AI описывал правила для всех ТС, создавая путаницу.  
**Решение:** Каждое правило начинается с позиции водителя легковушки.

**ПРАВИЛЬНО ✅:**
> "Как водитель **легкового автомобиля (Turismo)**, тебе это правило не касается, но ты должен знать..."

**НЕПРАВИЛЬНО ❌:**
> "Это правило действует для грузовиков..."

### 🚫 Запрещенные фразы

#### Для блока РФ (6 фраз):
```
❌ "В РФ нет аналога"
❌ "В РФ нет прямого аналога"
❌ "Правило идентично РФ"
❌ "В России экологические знаки не действуют"
❌ "В РФ аналогичное правило"
❌ Любые формальные отличия без опасности
```

**Почему:** Эти фразы создают "воду" без практической пользы.

#### Для самодостаточности текста (8 фраз):
```
❌ "как указано на схеме"
❌ "согласно таблице"
❌ "(см. схему)"
❌ "посмотрите на рисунок"
❌ "на изображении мы видим"
❌ "как показано на картинке"
❌ "согласно инфографике"
❌ "см. изображение А/Б"
```

**Почему:** Ученик НЕ ВИДИТ никаких схем в приложении, только текст.

### 🇷🇺 Conditional Russia Block

**Правило:** Блок создается ТОЛЬКО при наличии критического отличия.

**Критическое отличие = одно из:**
1. Противоположные приоритеты (пешеход/велосипедист)
2. Уникальная разметка/знаки, которых нет в РФ
3. Действия, которые в РФ законны, а в Испании — штраф
4. Отличие может привести к ДТП или штрафу

**Примеры критических отличий:**
- ✅ Велосипедисты на "зебре" (в РФ уступает пешеход, в ES — автомобиль)
- ✅ Ручной сигнал торможения (в РФ рука вверх, в ES — вверх-вниз)
- ✅ Нет порога +20 км/ч для превышения скорости

**Примеры НЕ критических:**
- ❌ Разные скорости (очевидно)
- ❌ Уникальные знаки (VAO, нет в РФ) без путаницы
- ❌ Одинаковые правила с другой формулировкой

## Правила модификации

### ⚠️ СТРОГО ЗАПРЕЩЕНО менять без тестирования:

1. **Словарь терминов** — может сломать 100+ уже переведенных вопросов
2. **JSON Schema** — может сломать импорт в базу
3. **Список запрещенных фраз** — могут вернуться старые проблемы
4. **Temperature < 0.2** — AI начнет фантазировать

### ✅ Безопасно модифицировать:

1. **Примеры мнемотехники** в инструкциях
2. **Дополнительные условия** для блока РФ (усиление)
3. **Новые термины** в словарь (с тестированием)
4. **Улучшения финальной проверки**

### 🧪 Процесс тестирования изменений:

```bash
# 1. Создать тестовый вопрос
node -e "
const { enrichQuestion } = require('./scripts/enrich-batch-v2.js');
const testQ = { /* ... */ };
enrichQuestion(testQ).then(console.log);
"

# 2. Проверить вручную:
# - Терминология корректна?
# - Нет запрещенных фраз?
```

## Troubleshooting

### Проблема: AI возвращает IDs как 1/2/3 вместо a/b/c

**Причина:** JSON Schema не строго ограничен enum  
**Решение:** Проверить responseSchema.answers.items.properties.id.enum

```javascript
id: { 
    type: "string",
    enum: ["a", "b", "c"]  // ОБЯЗАТЕЛЬНО!
}
```

---

### Проблема: AI ссылается на схемы "(см. схему)"

**Причина:** Не достаточно строгая финальная проверка  
**Решение:** Усилить блок "ФИНАЛЬНАЯ ПРОВЕРКА ПЕРЕД ВЫДАЧЕЙ JSON"

```javascript
⚠️ ФИНАЛЬНАЯ ПРОВЕРКА ПЕРЕД ВЫДАЧЕЙ JSON:
Перед тем как вернуть JSON, убедись что в explanation.ru НЕТ:
  • слов "схема" / "таблиц" / "рисунок"
Если нашел — перефразируй НЕМЕДЛЕННО.
```

---

### Проблема: Блок РФ пустой ("вода")

**Причина:** Нечеткие критерии критичности  
**Решение:** Добавить конкретные примеры в промпт

```javascript
Примеры КРИТИЧЕСКИХ отличий:
  • Противоположные приоритеты (пешеход на зебре)
  • Действия законны в РФ, штраф в ES (обочина)
  • Разные сигналы руками (опасность ДТП!)
```

---

### Проблема: Мнемотехника слабая

**Причина:** AI не понимает "хороший юмор"  
**Решение:** Дать 3-5 конкретных примеров в промпте

```javascript
3️⃣ **💡 Шпаргалка** (ПРИМЕРЫ):
✅ "Корова на обочине — нога на тормозе"
✅ "120 на спидометре — шансы на нуле"
✅ "Колеса к колесам — справа"
```
