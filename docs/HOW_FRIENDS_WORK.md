# 👥 Как определяются друзья в лидерборде

## 📋 Текущая реализация

### Основа: Таблица `referrals`

Друзья определяются через **реферальную систему** на основе таблицы `referrals`:

```sql
CREATE TABLE referrals (
  referrer_id UUID,  -- Тот, кто пригласил
  referred_id UUID,  -- Тот, кого пригласили
  ...
);
```

### Логика определения друзей

В функции `get_user_leaderboard_position` друзья определяются так:

```sql
-- Друзья = те, кого пользователь пригласил (referred_id)
SELECT referred_id AS friend_id
FROM referrals
WHERE referrer_id = p_user_id

UNION

-- + тот, кто пригласил пользователя (referrer_id)
SELECT referrer_id AS friend_id
FROM referrals
WHERE referred_id = p_user_id
```

**Простыми словами:**
- Если **ты пригласил** пользователя B → B твой друг
- Если **тебя пригласил** пользователь A → A твой друг

---

## 🤔 Проблема текущей реализации

### Это не совсем "друзья"

Текущая логика больше похожа на **"реферальная сеть"**, а не на друзей:

1. **Односторонняя связь:**
   - Если A пригласил B, они считаются друзьями
   - Но это не означает, что они реально друзья

2. **Нет взаимности:**
   - Нет подтверждения дружбы
   - Нет возможности добавить друга вручную

3. **Ограниченность:**
   - Друзьями считаются только те, кто связан через реферальную систему
   - Если пользователь не приглашал никого и его никто не приглашал → нет друзей

---

## 💡 Варианты улучшения

### Вариант 1: Отдельная таблица друзей (рекомендуется)

Создать отдельную таблицу для друзей с подтверждением:

```sql
CREATE TABLE user_friends (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id),
  friend_id UUID NOT NULL REFERENCES profiles(id),
  status TEXT NOT NULL CHECK (status IN ('pending', 'accepted', 'blocked')),
  requested_by UUID NOT NULL REFERENCES profiles(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  accepted_at TIMESTAMPTZ,
  UNIQUE(user_id, friend_id),
  CHECK(user_id != friend_id)
);
```

**Логика:**
- Пользователь A отправляет запрос дружбы пользователю B
- B подтверждает → статус `accepted`
- В лидерборде показываются только друзья со статусом `accepted`

**Преимущества:**
- ✅ Реальные друзья с подтверждением
- ✅ Можно добавлять друзей вручную
- ✅ Не зависит от реферальной системы

### Вариант 2: Расширить текущую систему

Оставить `referrals` как основу, но добавить:

```sql
-- Добавить поле "друзья" в referrals
ALTER TABLE referrals 
ADD COLUMN is_friend BOOLEAN DEFAULT false;

-- Или создать отдельную таблицу связей
CREATE TABLE user_connections (
  user_id UUID,
  connected_user_id UUID,
  connection_type TEXT CHECK (connection_type IN ('referral', 'friend', 'both')),
  ...
);
```

**Логика:**
- Реферальная связь автоматически создаёт "дружбу"
- Можно вручную добавить друзей
- В лидерборде показываются только `is_friend = true`

### Вариант 3: Гибридный подход

Использовать и `referrals`, и отдельную таблицу друзей:

```sql
-- Друзья = referrals + user_friends
SELECT friend_id FROM (
  -- Из referrals
  SELECT referred_id AS friend_id FROM referrals WHERE referrer_id = user_id
  UNION
  SELECT referrer_id AS friend_id FROM referrals WHERE referred_id = user_id
  UNION
  -- Из user_friends
  SELECT friend_id FROM user_friends 
  WHERE user_id = user_id AND status = 'accepted'
  UNION
  SELECT user_id AS friend_id FROM user_friends 
  WHERE friend_id = user_id AND status = 'accepted'
) AS all_friends;
```

## 📊 Текущее поведение

### Что показывается в фильтре "Друзья":

1. **Пользователи, которых ты пригласил:**
   - Если ты использовал свой реферальный код для приглашения пользователя B
   - B появится в твоём списке "друзей"

2. **Пользователь, который тебя пригласил:**
   - Если ты зарегистрировался по чьему-то реферальному коду
   - Тот, кто тебя пригласил, появится в твоём списке "друзей"

3. **Если нет связей:**
   - Показывается пустой список
   - Сообщение: "Пока нет игроков в рейтинге"

## 🔧 Как проверить текущих "друзей"

В Supabase SQL Editor:

```sql
-- Проверить, кого пригласил пользователь
SELECT 
  p.first_name AS friend_name,
  p.duel_pass_level,
  p.duel_pass_xp
FROM referrals r
JOIN profiles p ON p.id = r.referred_id
WHERE r.referrer_id = 'USER_ID'::uuid;

-- Проверить, кто пригласил пользователя
SELECT 
  p.first_name AS referrer_name,
  p.duel_pass_level,
  p.duel_pass_xp
FROM referrals r
JOIN profiles p ON p.id = r.referrer_id
WHERE r.referred_id = 'USER_ID'::uuid;
```
