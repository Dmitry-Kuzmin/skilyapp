# 📱 iOS Rich Push: Что не так и как исправить

## 🚨 Главная проблема

**iOS НЕ ПОДДЕРЖИВАЕТ кастомные стили уведомлений.**  
Это ограничение платформы Apple — уведомления всегда выглядят как системные.

## ✅ Что ВОЗМОЖНО на iOS (PWA с iOS 16.4+):

1. **Rich media** — картинка при долгом нажатии (Haptic Touch) ✅
2. **Actions** — до 4 кнопок (только текст, без иконок) ✅
3. **Emoji** — в `title` и `body` (НЕ в кнопках!) ✅
4. **Badge** — счётчик на иконке PWA ✅

## ❌ Что НЕ работает на iOS:

- CSS стили, градиенты, анимации
- Кастомные шрифты/цвета
- Иконки в кнопках (`icon` в `actions`)
- Emoji в кнопках (могут ломать отображение)

## 📋 Чек-лист "Почему не показывается картинка"

### 1️⃣ **Размер файла** (САМОЕ ЧАСТОЕ)
```bash
# Проверь в терминале:
ls -lh public/images/hero-lcp.webp
```

**Требования:**
- JPEG/PNG/WebP: **≤ 1 MB** (1048576 байт)
- GIF/видео: **≤ 2 MB**

❗ Если больше — iOS **молча игнорирует** картинку.

**Решение:** Сожми через:
- https://tinypng.com
- https://squoosh.app

### 2️⃣ **Формат изображения**
✅ Лучшие форматы:
- JPEG (самый стабильный)
- WebP (iOS 14+)
- PNG (тяжелее, но работает)

### 3️⃣ **Абсолютный URL**
❌ Плохо: `image: '/images/hero.jpg'`  
✅ Хорошо: `image: 'https://ваш-сайт.com/images/hero.jpg'`

В коде используй:
```javascript
const origin = window.location.origin;
const imageUrl = `${origin}/images/hero-lcp.webp`;
```

### 4️⃣ **Доступность URL**
Открой URL картинки в Safari на iPhone:
- Должна загрузиться **мгновенно**
- Проверь, что нет CORS-ошибок

## 🧪 Как тестировать на iPhone

### Вариант 1: Консоль Safari (самый быстрый)
1. Открой сайт на iPhone в Safari
2. Зайди в **Настройки → Safari → Дополнения → Веб-инспектор** (включи)
3. На Mac: Safari → **Разработка** → выбери iPhone → открой консоль
4. Вставь код из `/scripts/test-ios-push.js`
5. Запусти: `pushTests.runAllTests()`

### Вариант 2: Встроенная кнопка
1. Открой **Настройки** в приложении
2. Раздел **Уведомления**
3. Нажми **Отправить тестовое push-уведомление**

### Вариант 3: Ручной вызов
```javascript
// Вставь в консоль Safari на iPhone:
(async () => {
    const reg = await navigator.serviceWorker.ready;
    await reg.showNotification('Тест 🎨', {
        body: 'Долго нажми на уведомление!',
        image: 'https://picsum.photos/800/400.jpg', // Легкая тестовая картинка
        actions: [
            { action: 'open', title: 'Открыть' }
        ]
    });
})();
```

## 💡 Как добиться "вау-эффекта" на iOS

Поскольку стили недоступны, нужно работать с тем, что есть:

### 1. **Качественная картинка 2:1**
- Размер: 800×400px
- Формат: JPEG или WebP
- Вес: < 1 MB
- Соотношение: ландшафт (2:1) для красивого разворота

### 2. **Emoji в тексте** (НЕ в кнопках!)
```javascript
title: '🔥 Новый вызов!',
body: 'Дмитрий бросил тебе вызов ⚔️ Осталось 24 часа!'
```

### 3. **Понятные кнопки**
```javascript
actions: [
    { action: 'accept', title: 'Принять' },  // 1-2 слова
    { action: 'decline', title: 'Отклонить' }
]
```

### 4. **Badge для счётчика**
```javascript
badge: '/badge-72.png' // Красный кружок с цифрой на иконке
```

### 5. **Долгое нажатие = главный UX**
⚠️ **Объясни пользователям:** нужно ДОЛГО нажать на уведомление (Haptic Touch), чтобы увидеть картинку на пол-экрана и кнопки.

## 🎯 Итоговая философия

**iOS не даёт кастомные стили, но даёт нативный UX высокого качества.**

Это **философия Apple**:
- Все уведомления выглядят системно (единообразно)
- Но поддерживают rich media (картинки, видео)
- И интерактивность (кнопки)

**Не боритесь с платформой — используйте её возможности:**
1. Качественные изображения (< 1MB)
2. Emoji в тексте (не в кнопках)
3. Короткие понятные кнопки
4. Обучите пользователей долгому нажатию

## 🔍 Отладка: Пошаговая инструкция

### Если картинка не показывается:

1. **Проверь размер**:
   ```bash
   ls -lh public/images/hero-lcp.webp
   # Должно быть < 124K (как у вас сейчас — это ОК)
   ```

2. **Тест с внешней картинкой**:
   ```javascript
   // Если этот тест работает — проблема в вашей картинке
   reg.showNotification('Тест', {
       image: 'https://picsum.photos/800/400.jpg'
   });
   ```

3. **Проверь доступность URL**:
   - Открой в Safari: `https://ваш-сайт.com/images/hero-lcp.webp`
   - Должна загрузиться без ошибок 404/403

4. **Проверь формат**:
   ```bash
   file public/images/hero-lcp.webp
   # Должно быть: "WebP image data"
   ```

5. **Попробуй JPEG**:
   ```bash
   # Конвертируй WebP → JPEG для теста
   # (WebP может глючить на старых iOS 14)
   ```
