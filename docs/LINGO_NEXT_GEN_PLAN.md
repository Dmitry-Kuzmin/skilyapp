# Lingo Next Gen

Дата фиксации: 3 апреля 2026

## Что подтверждено

- В `language_terms` реально хранится `558` терминов.
- Текущий `/lingo` собран вручную через `src/data/lingo/chapter-1..6.ts`.
- Прогресс уже хранится отдельно в `user_lingo_progress`, значит новый курс можно развивать без потери механики прогресса.
- В `language_terms` уже есть `topic_id`, но фактическая раскладка терминов по темам пока не соответствует будущей учебной архитектуре:
  - `178` терминов сидят в topic 1.
  - `177` терминов сидят в topic 4.
  - `203` терминов вообще не привязаны к topic.
  - Остальные темы пока почти пустые.

## Главная проблема текущего курса

Сейчас есть хороший MVP, но не система:

- уроки статичны и не масштабируются на 558 терминов;
- словарь и курс живут отдельно;
- темп уроков не строится вокруг реальных дорожных сцен;
- нет чёткой структуры "что ученик освоит после модуля";
- нет ощущения премиального учебного продукта.

## Принципы вау-эффекта

1. Scenario-first
   - каждый урок стартует с реальной дорожной сцены, а не с сухого списка слов.
2. Visual memory
   - термин закрепляется через образ, контраст и короткий смысловой блок.
3. Exam decoder
   - ученик учится распознавать язык DGT: обязанность, исключение, запрет, ловушку.
4. Momentum design
   - короткие победы, понятный прогресс, сильный конец модуля и чувство роста.

## Формат урока нового поколения

Каждый урок должен собираться по одной логике:

1. Hook scene
   - короткий дорожный кейс или ситуация.
2. Core vocabulary
   - 6-10 ключевых терминов одного кластера.
3. Meaning lock
   - связки, карточки, быстрые сопоставления.
4. DGT phrasing
   - как этот термин звучит в реальном экзамене.
5. Stress check
   - 2-3 вопроса в стиле "а что если".
6. Micro win
   - сильный экран завершения с освоенным навыком.

## Что нужно сделать следующим этапом

1. Создать нормализованный слой распределения терминов по новым модулям.
2. Перевести `/lingo` с ручных файлов уроков на data-driven сборку.
3. Подключить к уроку термины из `language_terms`, а не дублировать контент руками.
4. Добавить метрики мастерства:
   - term exposure
   - lesson confidence
   - weak cluster detection
5. Встроить финальные "exam decoder" уроки как обязательный слой перед тестами.

## Практическое решение на текущем этапе

В код уже добавляется новый blueprint курса, чтобы:

- команда видела целевую архитектуру;
- дизайн `/lingo` отражал новый уровень продукта уже сейчас;
- следующая итерация могла раскладывать термины по модулям без повторного перепроектирования.
