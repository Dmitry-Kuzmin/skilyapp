# 🆘 Offline Mode FAQ - Support Guide

Простая шпаргалка для саппорта по вопросам offline режима.

---

## ❓ Часто задаваемые вопросы

### 1. "Почему приложение не работает без интернета?"

**Короткий ответ:**
Для работы offline нужно **хотя бы один раз** открыть приложение **с интернетом**.

**Что сказать пользователю:**
> "Привет! Для работы offline приложению нужно один раз загрузиться с интернетом, чтобы сохранить данные в кэш. После этого оно будет работать даже без сети!"

**Решение:**
1. Подключиться к интернету
2. Открыть приложение
3. Подождать 10-15 секунд (пока кэш заполнится)
4. Теперь можно использовать offline

---

### 2. "На iPhone перестало работать offline через пару дней"

**Короткий ответ:**
iPhone Safari **автоматически чистит кэш** через 2-7 дней бездействия. Это ограничение Apple, не баг приложения.

**Что сказать пользователю:**
> "iPhone Safari автоматически очищает кэш приложений через несколько дней. Для лучшего offline опыта рекомендуем добавить приложение на Home Screen (Safari → Share → Add to Home Screen). Так кэш сохраняется дольше!"

**Решение:**
1. **Вариант А (лучше):** Add to Home Screen
   - Safari → кнопка Share (квадрат со стрелкой)
   - "Add to Home Screen"
   - Иконка появится на главном экране
   - Открывать через эту иконку (не через Safari)

2. **Вариант Б:** Открывать хотя бы раз в 2-3 дня
   - Так iOS не почистит кэш

---

### 3. "Telegram Mini App не открывается без интернета"

**Короткий ответ:**
Холодный старт Telegram Mini App без интернета **НЕ гарантируется**. Это ограничение Telegram WebView.

**Что работает:**
✅ Открыл Mini App → используешь → интернет пропал → **продолжаешь использовать**

**Что НЕ работает:**
❌ Telegram закрыт → интернета нет → пытаешься открыть Mini App → **может не открыться**

**Что сказать пользователю:**
> "Telegram Mini Apps лучше всего работают когда есть интернет при первом открытии. Если связь пропадёт во время использования - приложение продолжит работать из кэша! Холодный запуск без интернета, к сожалению, не всегда возможен из-за ограничений Telegram."

**Успешность холодного старта offline:**
- Telegram Desktop: ~80%
- Telegram Android: ~60%
- Telegram iOS: ~40%

---

### 4. "Вижу белый экран / приложение не загружается"

**Быстрая диагностика:**

**Вопросы пользователю:**
1. Какое устройство? (iPhone/Android/Desktop)
2. Какой браузер? (Safari/Chrome/Telegram)
3. Есть интернет или нет?
4. Первый раз открываешь или уже использовал?

**Типичные решения по платформам:**

#### Desktop (Chrome/Safari):
```
Решение: Hard refresh
- Chrome: Ctrl+Shift+R (Win) / Cmd+Shift+R (Mac)
- Safari: Cmd+Option+R
```

#### iPhone Safari:
```
Решение 1: Очистить кэш
- Settings → Safari → Clear History and Website Data

Решение 2: Переподключиться к интернету
- Если был offline > 30 минут
```

#### Telegram (любой):
```
Решение: Переоткрыть Mini App
1. Закрыть Mini App (назад в чат)
2. Подключиться к интернету
3. Открыть Mini App снова
```

#### Если ничего не помогает:
```
Escalate тимлиду с информацией:
- Платформа + браузер
- Версия (если известно)
- Скриншот экрана
- Была ли обновлена версия приложения недавно
```

---

### 5. "Данные не обновляются / вижу старую информацию"

**Короткий ответ:**
Приложение кэширует данные на **5 минут** для быстрой загрузки. Обновление происходит при переподключении к сети.

**Что сказать пользователю:**
> "Для экономии трафика и быстрой загрузки приложение показывает данные из кэша (до 5 минут). Если нужны свежие данные - переподключитесь к интернету, и они обновятся автоматически!"

**Принудительное обновление:**
1. Закрыть приложение
2. Убедиться что есть интернет
3. Открыть снова
4. Данные обновятся

**Для критичных данных** (баланс монет, premium):
- Обновляются немедленно при изменении
- Refetch при каждом монтировании

---

### 6. "Приложение тормозит / долго загружается"

**Если ПЕРВАЯ загрузка медленная:**
> "Первая загрузка может занять 2-3 секунды, так как приложение сохраняет данные для работы offline. Последующие загрузки будут мгновенными (~0.5 сек)!"

**Если ПОВТОРНЫЕ загрузки медленные:**
```
Проблема: Кэш не работает или переполнен

Решение:
1. Очистить кэш браузера
2. Перезагрузить приложение с интернетом
3. Кэш создастся заново (оптимизированный)
```

---

### 7. "Приложение занимает много места на телефоне"

**Короткий ответ:**
Приложение использует **~10-20 МБ** для работы offline. Это нормально для современных приложений.

**Что сказать пользователю:**
> "Для работы offline приложение сохраняет ~10-20 МБ данных (учебные материалы, ваш прогресс, изображения). Это позволяет работать без интернета до недели!"

**Если нужно очистить:**
```
iPhone:
Settings → Safari → Advanced → Website Data → 
Find "skilyapp.com" → Swipe left → Delete

Android Chrome:
Settings → Site settings → Storage → 
Find "skilyapp.com" → Clear & reset
```

## 📱 Платформо-специфичные советы

### iPhone (iOS Safari)

**Лучший offline опыт:**
```
1. Safari → skilyapp.com
2. Share (квадрат со стрелкой внизу)
3. "Add to Home Screen"
4. Открывать через иконку на Home Screen
```

**Почему это лучше:**
- iOS меньше чистит кэш
- Fullscreen режим (больше места)
- Отдельный процесс (лучше производительность)

**Ограничения:**
- Кэш может очиститься через 2-7 дней бездействия
- При низком заряде (<20%) iOS чистит агрессивнее
- Low Power Mode ограничивает кэш

---

### Telegram (Mobile)

**Что работает надёжно:**
```
1. Открыл Mini App с интернетом
2. Используешь приложение
3. Интернет пропал
4. Продолжаешь использовать ✅
```

**Что может не работать:**
```
1. Закрыл Telegram полностью
2. Интернета нет
3. Открываешь Telegram → Mini App
4. Может не открыться ⚠️
```

**Решение:** Держать интернет включенным при открытии Telegram.

---

### Desktop (Chrome/Safari)

**Работает отлично:**
- Offline режим стабилен (дни/недели)
- Множественные reload работают
- Кэш не чистится агрессивно

**Если сломалось:**
- Hard refresh обычно помогает
- Очистка кэша - крайняя мера

## 📞 Когда escalate тимлиду

### Red Flags (срочно):

1. **Массовые жалобы** (>10 пользователей за час)
   - Возможно проблема с deploy
   - Нужен hotfix или rollback

2. **Белые экраны на Desktop** с интернетом
   - Не должно быть никогда
   - Version mismatch или серьёзный баг

3. **Telegram Web не загружается** вообще
   - Возможно сломались CSP headers
   - Проверить vercel.json

4. **Ошибки в production logs** (Rollbar)
   - Spike в "Importing a module script failed"
   - "FetchEvent.respondWith received an error"

### Можно подождать (не критично):

1. **iPhone offline перестал работать** через неделю
   - Это iOS limitation (задокументировано)
   - Решение: Add to Home Screen

2. **Telegram Mobile холодный старт** не работает
   - Это ожидаемо (~40-60% success)
   - Решение: Открывать с интернетом

3. **Старые данные** (< 10 минут)
   - Это by design (staleTime: 5 мин)
   - Обновятся при reconnect

## 📊 Expected Behavior (что нормально)

### ✅ Нормальное поведение (не баг):

1. **Первая загрузка 2-3 секунды**
   - Приложение кэширует данные
   - Последующие загрузки быстрые (~0.5 сек)

2. **Offline работает 10-30 минут на iPhone**
   - iOS агрессивно чистит кэш
   - Add to Home Screen помогает

3. **Telegram холодный старт offline не всегда работает**
   - WebView ограничения
   - Горячий режим (во время использования) работает отлично

4. **Некоторые изображения не загружаются offline**
   - Кэшируются только просмотренные изображения
   - Runtime cache (по запросу)

5. **Realtime features не работают offline**
   - WebSocket требует сеть
   - Duel notifications, online players - только online

---

### ❌ НЕ нормальное (баг):

1. **Белый экран на Desktop с интернетом**
   - Должно работать всегда
   - Escalate

2. **Telegram Web iframe ошибка**
   - "X-Frame-Options" или CSP проблема
   - Escalate

3. **Приложение крашится при reload** (любая платформа, с интернетом)
   - Version mismatch или chunk load failure
   - Escalate

4. **Offline работает < 1 минуты** на Desktop
   - Кэш должен держаться дни
   - Escalate

```

```
1. Подключиться к интернету

```

```
Отправить тимлиду:
- Всю собранную информацию
- Скриншоты
- Какие решения пробовали
- Timestamp проблемы

## 🎓 Для саппорта (что важно знать)

### Offline режим - это НЕ "работает всегда без интернета"

**Правильные ожидания:**

✅ **Desktop:** Offline работает дни/недели стабильно
✅ **iPhone (A2HS):** Offline работает часы/дни
⚠️ **iPhone Safari:** Offline работает 10-30 минут надёжно, потом может почиститься
⚠️ **Telegram Mobile:** Offline работает в горячем режиме, холодный старт не гарантирован

### Это не баги, это ограничения платформ

**iOS Safari:**
- Apple специально ограничивает PWA (~50 МБ кэш)
- Чистит кэш агрессивно (для нативных приложений из App Store)
- Add to Home Screen улучшает ситуацию

**Telegram Mobile:**
- WebView может не сохранять кэш между сессиями
- Зависит от версии Telegram и OS
- Это компромисс Mini Apps (vs нативные приложения)

### Главное правило саппорта

**Если offline не работает:**
1. Сначала: попробовать universal fix (reconnect + reload)
2. Если не помогло: platform-specific fix (clear cache)
3. Если всё ещё не работает: escalate с деталями

**Не обещать "будет работать вечно offline"** - это невозможно на iOS/Telegram.

**Тимлид/Tech Lead:**
- Для критичных проблем (white screens, mass outages)

- Для проблем с deployment
- Для Vercel/CDN issues
- Для certificate/SSL problems

**Backend:**
- Для Supabase API errors (400/500 codes)
- Для Edge Functions failures
