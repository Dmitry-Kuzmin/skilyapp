# Тестирование буста Data Leak (🛢️ Масло)

## Шаг 1: Запуск приложения

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173` (или другому порту, который укажет Vite)

## Шаг 2: Добавление буста в инвентарь

### Вариант A: Через Supabase Dashboard (SQL Editor)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Перейдите в **SQL Editor**
3. Выполните один из запросов ниже:

#### Найти свой profile_id:
```sql
-- Для веб-пользователей (замените YOUR_SUPABASE_USER_ID)
SELECT id, user_id, telegram_id, name 
FROM profiles 
WHERE user_id = 'YOUR_SUPABASE_USER_ID'::uuid;

-- Для Telegram-пользователей (замените YOUR_TELEGRAM_ID)
SELECT id, user_id, telegram_id, name 
FROM profiles 
WHERE telegram_id = YOUR_TELEGRAM_ID;
```

#### Добавить буст в инвентарь:
```sql
-- Замените YOUR_PROFILE_ID на ваш profile_id из предыдущего запроса
SELECT public.modify_boost_inventory(
  'YOUR_PROFILE_ID'::uuid, 
  'screen_injector', 
  5  -- количество бустов
);
```

### Вариант B: Через консоль браузера (DevTools)

1. Откройте приложение в браузере
2. Откройте DevTools (F12)
3. Перейдите на вкладку **Console**
4. Выполните:

```javascript
// Получить текущий profile_id
const { data: profile } = await supabase
  .from('profiles')
  .select('id')
  .eq('user_id', (await supabase.auth.getUser()).data.user?.id)
  .single();

console.log('Profile ID:', profile?.id);

// Добавить буст
const { data, error } = await supabase.rpc('modify_boost_inventory', {
  p_user_id: profile.id,
  p_boost_type: 'screen_injector',
  p_change: 5
});

console.log('Boost added:', data, error);
```

## Шаг 3: Тестирование эффекта

1. **Создайте дуэль** или присоединитесь к существующей
2. **Дождитесь начала дуэли** (появления первого вопроса)
3. **Найдите кнопку буста "Data Leak"** 🛢️ в панели бустов (должна быть видна с количеством)
4. **Нажмите на кнопку буста** во время вопроса
5. **Наблюдайте эффект**:
   - ⚠️ Появится предупреждение "САБОТАЖ" с тряской экрана
   - 💥 Появятся трещины стекла
   - 🛢️ Экран зальется черной жижей (маслом) сверху вниз
   - 🧽 Появится интерактивная губка для очистки
   - 📊 Будет виден прогресс-бар очистки

## Шаг 4: Очистка масла

1. **Наведите курсор** (или палец на мобильном) на залитый экран
2. **Зажмите и водите** по экрану, чтобы стереть масло
3. **Стирайте масло**, пока не очистите **97%** экрана
4. После очистки эффект исчезнет, и вы сможете продолжить дуэль

## Проверка работы

✅ **Что должно работать:**
- Буст виден в панели бустов с иконкой 🛢️ и названием "Data Leak"
- При активации появляется предупреждение "САБОТАЖ"
- Трещины стекла анимируются
- Масло стекает сверху вниз с физикой жидкости
- Губка следует за курсором/пальцем
- Прогресс очистки отображается в реальном времени
- После очистки 97% экран становится доступным

❌ **Если что-то не работает:**
- Проверьте консоль браузера на ошибки (F12 → Console)
- Убедитесь, что миграция применена: `npm run supabase:apply`
- Проверьте, что буст добавлен в инвентарь (см. Шаг 2)
- Убедитесь, что вы используете правильный `profile_id`

## Быстрая проверка инвентаря

```sql
-- Проверить наличие буста в инвентаре
SELECT bi.*, bd.name_ru, bd.icon
FROM boost_inventory bi
JOIN boost_definitions bd ON bi.boost_type = bd.type
WHERE bi.user_id = 'YOUR_PROFILE_ID'::uuid
  AND bi.boost_type = 'screen_injector';
```

## Отладка

Если эффект не появляется, проверьте:

1. **Консоль браузера** - есть ли ошибки JavaScript?
2. **Network tab** - успешно ли выполнен запрос `use_boost`?
3. **React DevTools** - обновляется ли состояние `activeExploits`?
4. **Компонент OilSplashAttack** - монтируется ли он? (добавьте `console.log` в начало компонента)

## Примечания

- Буст можно использовать только **один раз за вопрос**
- Буст работает только в **активной дуэли** (не в лобби)
- Эффект блокирует экран до полной очистки (97%)
- На мобильных устройствах используйте **тач-жесты** для очистки
