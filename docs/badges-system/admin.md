# Админка управления наградами

## 1. Схема данных
- `Badge`: `id`, `slug`, `name`, `short_description`, `description`, `category`, `rarity`, `points`, `is_active`, `season_id`, `created_at`, `updated_at`, `version`.
- `Trigger`: `id`, `badge_id`, `type` (event/aggregate/manual), `conditions` (JSON), `threshold`, `time_window`, `cooldown`, `created_by`.
- `RewardVisual`: `id`, `badge_id`, `icon_asset_id`, `animation_asset_id`, `theme_color`, `cta_label`.
- `Season`: `id`, `name`, `start_at`, `end_at`, `status`, `description`, `reward_multiplier`, `is_exclusive`.
- `BadgeVersion`: `id`, `badge_id`, `version`, `diff_summary`, `changed_by`, `changed_at`.
- `AuditLog`: `id`, `entity_type`, `entity_id`, `action`, `diff`, `performed_by`, `performed_at`.

## 2. UI-структура
1. **Dashboard**: summary по количеству активных/архивных бейджей, популярность, алерты.
2. **Список наград**: таблица с фильтрами по категории, редкости, статусу, сезону. Колонки: название, тип триггера, период, очки, активность.
3. **Карточка награды**: табы `Основное`, `Условия`, `Визуал`, `Статистика`, `Версии`.
4. **Конструктор условий**:
   - Builder блоков: выбор события, фильтров (например, `lesson_completed` где `course_id = X`), агрегатов (count/sum/max).
   - Визуальный timeline: окно проверки «за последние 7 дней» и т.п.
   - Тестовый прогон: ввод `user_id` → вывод прогресса и ожидаемое время до награды.
5. **Предпросмотр**: live view бейджа (мобильный/веб), отображение прогресс-бара, CTA, анимация.
6. **Статистика**: графики выдачи по дням, conversion rate, среднее время до unlock, состав аудитории (новые/возвращающиеся пользователи).
7. **Импорт/экспорт**: модалка, drag&drop CSV/JSON, проверка ошибок, отображение diff перед сохранением.

## 3. Права доступа
- **Контент-редактор**: создание/редактирование, запуск тестов, но без публикации.
- **Администратор**: все права, включая активацию/архивирование, импорт, управление ролями.
- **Аналитик**: доступ к статистике, экспорту данных, запуск тестового прогона (только чтение).
- **Аудит лог**: виден всем ролям; фильтрация по дате, типу сущности, пользователю.

## 4. API и backend
- REST или GraphQL слой с эндпоинтами:
  - `GET /badges`, `POST /badges`, `PUT /badges/:id`, `PATCH /badges/:id/state`.
  - `POST /badges/:id/test-run` — прогон условия.
  - `GET /badges/:id/stats?range=...`.
  - `POST /badges/import`, `GET /badges/export`.
  - `GET /seasons`, `POST /seasons`, `PATCH /seasons/:id`.
- Webhooks/обработчики для обновления кэша клиента после изменения бейджа.
- Валидации: уникальность `slug`, корректность JSON-условий, контроль пересечения сезонов.
- Версионирование: автоматический инкремент `version`, сохранение diff в `BadgeVersion`.

## 5. UX посвящение
- Onboarding внутри админки: подсказки по настройке первой награды, пример шаблона.
- Сохранение черновиков, автосохранение раз в минуту.
- Undo/redo для конструктора условий.
- Preview изменений перед публикацией с указанием, что изменится.

## 6. Импорт/экспорт
- **CSV**: базовые поля, преобразование в JSON условий через сопутствующий wizard.
- **JSON**: полное описание награды (включая визуал и версии).
- Поддержка bulk-операций: выбор нескольких бейджей → «архивировать», «дублировать».
- Хранение бэкапов в S3/облаке, автоматический nightly экспорт.

## 7. Контроль качества
- Юнит-тесты на парсер условий и api.
- E2E-сценарии: создание награды → публикация → симуляция событий → получение бейджа.
- Линтеры/pre-commit для JSON/конфигураций.
- Набор демо-данных для локальной разработки.
