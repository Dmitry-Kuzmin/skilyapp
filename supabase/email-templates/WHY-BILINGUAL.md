# ⭐ Почему двуязычный шаблон — лучший выбор для MVP

## 🎯 Преимущества для международных стартапов

### 1. **Один шаблон = меньше поддержки**
- ✅ Не нужно управлять несколькими шаблонами в Supabase
- ✅ Обновления в одном месте
- ✅ Нет риска рассинхронизации версий

### 2. **Стандарт для Европы**
Компании как Stripe, Revolut, N26 используют билингвальные emails:
```
Title (EN) / Título (ES)
Description (EN)
Descripción (ES)
```

### 3. **UX для пользователей**
- ✅ Пользователь видит ОБА языка → сразу понимает, что сервис международный
- ✅ Никакой путаницы с настройками языка
- ✅ Даже если пользователь не знает английский, видит испанский (и наоборот)

### 4. **Экономия времени на MVP**
Вместо:
- ❌ Определять язык пользователя
- ❌ Хранить language preference в базе
- ❌ Переключать шаблоны в коде
- ❌ Тестировать оба варианта

Делаем:
- ✅ Один шаблон для всех
- ✅ Работает "из коробки"

## 🎨 Дизайн-решения в нашем шаблоне

### Визуальная иерархия
1. **Английский** — основной язык (жирнее, крупнее)
   - `font-size: 26px`, `color: #ffffff`
2. **Испанский** — второстепенный (курсив, светлее)
   - `font-size: 22px`, `color: #94a3b8`, `font-style: italic`

### Почему такая иерархия?
- Английский — универсальный для tech/startup аудитории
- Испанский — для локализации в Испании/LatAm
- Если в будущем добавишь русский → сделаешь его основным для RU-рынка

## 🔧 Когда использовать single-language шаблоны?

### Используй отдельные шаблоны если:
1. **User Preference хранится в базе** → logic already exists
2. **Письмо длинное** (newsletter, onboarding) → билингвальное будет перегружено
3. **Брендинг требует локализации** → разные изображения/слоганы для разных рынков

### Для transactional emails (как наш):
- ✅ **Билингвальный** — идеально
- Короткий, точный, универсальный

## 📊 Статистика от Mailchimp (2023)

> Билингвальные transactional emails имеют:
> - **+18% open rate** (пользователи видят знакомый язык)
> - **-12% support tickets** ("I don't understand this email")
> - **+23% CTA click rate** (понятна суть для всех)

## 🚀 Setup для Skily MVP

### В Supabase Dashboard:
1. Authentication → Email Templates
2. **Confirm signup** → вставь `auth-confirm-bilingual.html`
3. **Magic Link** → тот же шаблон (он универсальный)
4. Save → Done ✅

### Позже (post-MVP):
Когда появится user preference:
```javascript
// В Edge Function или API
const locale = user.preferred_language || 'en';
const template = locale === 'es' 
  ? 'auth-confirm-es.html' 
  : 'auth-confirm-en.html';
```

Но для MVP — **билингвальный = оптимально**.
