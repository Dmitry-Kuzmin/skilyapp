# Простой скрапер PracticaVial

## Как использовать

1. **Запустите скрапер:**
   ```bash
   npm run scrape:simple -- --topics=1
   ```

2. **Откроется браузер** - авторизуйтесь вручную:
   - Введите логин и пароль
   - Нажмите "Iniciar sesión"
   - Перейдите на страницу с тестами: `https://teorica.practicavial.com/permiso/b/tests/tema/1`

3. **Вернитесь в терминал** и нажмите **Enter**

4. **Скрипт проверит авторизацию** и начнет скрапинг

## Параметры

- `--topics=1,2,3` - темы для скрапинга (по умолчанию: 1)
- `--headless` - запуск без браузера (не рекомендуется для первой авторизации)
- `--output=файл.xlsx` - имя выходного файла (по умолчанию: practicavial-questions.xlsx)

## Примеры

```bash
# Скрапить тему 1
npm run scrape:simple -- --topics=1

# Скрапить темы 1, 2, 3
npm run scrape:simple -- --topics=1,2,3

# С указанием выходного файла
npm run scrape:simple -- --topics=1 --output=my-questions.xlsx
```

## Что происходит

1. Открывается браузер
2. Вы авторизуетесь вручную
3. Скрипт проверяет авторизацию
4. Скрипт собирает все тесты для указанных тем
5. Для каждого теста извлекаются вопросы и ответы
6. Данные сохраняются в Supabase
7. Генерируется Excel файл

## Преимущества этого подхода

- ✅ Простота - вы авторизуетесь вручную
- ✅ Надежность - не зависит от автоматической авторизации
- ✅ Прозрачность - видите все, что происходит в браузере
- ✅ Гибкость - можете авторизоваться как хотите

## Решение проблем

### Скрипт не находит тесты

- Убедитесь, что вы авторизованы
- Убедитесь, что вы видите список тестов в браузере
- Проверьте, что вы на правильной странице: `/permiso/b/tests/tema/1`

### Скрипт не находит вопросы

- Некоторые тесты могут иметь другую структуру
- Проверьте скриншоты в папке проекта
- Попробуйте другой тест

### Ошибки при сохранении в БД

- Проверьте подключение к Supabase
- Проверьте переменные окружения в `.env`
- Убедитесь, что таблицы `topics`, `questions_new`, `answer_options` существуют
