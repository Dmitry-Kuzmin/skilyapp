# 🔧 Исправление ошибки 503 Service Unavailable

## Проблема

Ошибка `503 Service Unavailable` означает, что функция либо:
1. Не задеплоена правильно
2. Падает при выполнении (синтаксическая ошибка)
3. Не может загрузиться из-за проблем с импортами

## Решение

### Что исправлено:

1. **Изменён импорт**: Заменён `serve` на `Deno.serve` (как в других функциях)
2. **Улучшена обработка ошибок**: Добавлена более детальная обработка запросов
3. **Улучшено логирование**: Добавлены логи для отладки
4. **Упрощена структура**: Убрано дублирование кода

## Что нужно сделать

### Шаг 1: Задеплой обновлённую функцию

1. Открой **Supabase Dashboard**: https://supabase.com/dashboard
2. Выбери проект **sdadim-dgt-prep**
3. Перейди в **Edge Functions** → **ai-chat**
4. Нажми **"Edit"** или **"View Code"**
5. **УДАЛИ ВЕСЬ СТАРЫЙ КОД**
6. Скопируй **ВЕСЬ КОД** из файла `supabase/functions/ai-chat/index.ts`
7. Вставь в редактор функции
8. Нажми **"Deploy"** или **"Save"**

### Шаг 2: Проверь логи

1. После деплоя перейди в **Logs** функции
2. Должно быть видно:
   ```
   [AI Chat] Request received: { method: 'POST', ... }
   [AI Chat] Processing request with X messages
   [AI Chat] Trying Groq API...
   [AI Chat] ✅ Groq API success
   ```

### Шаг 3: Проверь работу

1. Перезагрузи страницу приложения
2. Задай вопрос AI помощнику
3. Должен прийти ответ

## Если ошибка 503 сохраняется

### Проблема 1: Функция не задеплоилась

**Решение:**
1. Проверь, что код полностью скопирован
2. Убедись, что нет синтаксических ошибок
3. Попробуй задеплоить через CLI (если установлен):
   ```bash
   supabase functions deploy ai-chat
   ```

### Проблема 2: Ошибка в логах

**Решение:**
1. Открой логи функции в Supabase Dashboard
2. Найди ошибку в логах
3. Проверь, что секрет `GROQ_API_KEY` добавлен
4. Проверь, что секрет `LOVABLE_API_KEY` добавлен (опционально)

### Проблема 3: Функция падает при выполнении

**Решение:**
1. Проверь логи на наличие ошибок
2. Убедись, что все импорты правильные
3. Проверь, что используется `Deno.serve`, а не `serve`

## Проверка

### В логах Supabase должно быть:

✅ **Успешный деплой:**
```
Function deployed successfully
```

✅ **Успешный запрос:**
```
[AI Chat] Request received: { method: 'POST', url: '...', hasAuth: true }
[AI Chat] Processing request with 1 messages
[AI Chat] Trying Groq API...
[AI Chat] ✅ Groq API success
```

❌ **Если видишь ошибку:**
```
Error: ...
at ...
```

## Важно

- ✅ Используй `Deno.serve`, а не `serve`
- ✅ Убедись, что секрет `GROQ_API_KEY` добавлен
- ✅ Проверь логи после каждого деплоя
- ✅ Перезагрузи страницу после деплоя

## Готово! 🎉

После исправления функция должна работать без ошибок 503!
