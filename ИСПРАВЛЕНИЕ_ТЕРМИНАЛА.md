# 🔧 Исправление проблемы с терминалом

## Проблема

Терминал показывает ошибку:
```
npm error enoent Could not read package.json: Error: ENOENT: no such file or directory, open '/Users/dimka/package.json'
```

Это происходит потому, что вы запускаете команду из неправильной директории.

## Решение

### Шаг 1: Откройте терминал

Откройте терминал на вашем Mac (Terminal.app или iTerm).

### Шаг 2: Перейдите в директорию проекта

Выполните команду:

```bash
cd /Users/dimka/Desktop/Sdadim/sdadim-dgt-prep
```

Или короче:

```bash
cd ~/Desktop/Sdadim/sdadim-dgt-prep
```

### Шаг 3: Проверьте, что вы в правильной директории

Выполните:

```bash
pwd
```

Должно показать:
```
/Users/dimka/Desktop/Sdadim/sdadim-dgt-prep
```

### Шаг 4: Проверьте, что package.json существует

Выполните:

```bash
ls package.json
```

Если файл существует, вы увидите `package.json`.

### Шаг 5: Теперь запустите скрапер

```bash
npm run scrape:practicavial
```

## Быстрый способ (одной командой)

Вы можете объединить все в одну команду:

```bash
cd ~/Desktop/Sdadim/sdadim-dgt-prep && npm run scrape:practicavial
```

## Альтернативный способ: Использовать абсолютный путь

Если не хотите переходить в директорию, можете запустить напрямую:

```bash
cd /Users/dimka/Desktop/Sdadim/sdadim-dgt-prep && npm run scrape:practicavial -- --username=kuzmin.public@gmail.com --password=yVSnp
```

## Как понять, что вы в правильной директории?

Проверьте приглашение терминала. Если вы видите путь к проекту в приглашении, значит вы в правильной директории.

Или выполните:
```bash
pwd
```

Должно показать путь к проекту, а не просто `~` или `/Users/dimka`.

## Видео-инструкция (текстовая)

1. Откройте терминал
2. Наберите: `cd ` (с пробелом после cd)
3. Откройте Finder и перейдите в папку проекта
4. Перетащите папку проекта в терминал
5. Нажмите Enter
6. Теперь вы в правильной директории!

## Готово!

После того, как вы перешли в правильную директорию, все команды `npm run` будут работать правильно.
