# 🔧 Решение проблемы с таймаутом

## Проблема

Скрипт выдает ошибку "Navigation timeout" при попытке авторизации, даже если вы уже авторизовались вручную.

## Решение

Я улучшил скрипт, чтобы он:

1. ✅ **Сначала проверяет, авторизованы ли вы** - если да, пропускает авторизацию
2. ✅ **Пробует открыть страницу с тестами напрямую** - если вы авторизованы, она откроется
3. ✅ **Обнаруживает ручную авторизацию** - если вы авторизовались вручную, скрипт это обнаружит
4. ✅ **Уменьшил таймауты** - быстрее определяет проблемы

## Как использовать

### Вариант 1: Авторизоваться вручную (рекомендуется)

1. **Запустите скрипт:**
   ```bash
   npm run scrape:practicavial -- --username=ваш-email --password=ваш-пароль --headless=false
   ```

2. **Когда откроется браузер:**
   - Авторизуйтесь вручную (введите логин и пароль)
   - Дождитесь успешной авторизации
   - Скрипт автоматически обнаружит, что вы авторизованы, и продолжит работу

3. **Если скрипт все еще выдает ошибку:**
   - Перейдите на страницу: https://teorica.practicavial.com/permiso/b/tests/tema/all
   - Если страница открылась - вы авторизованы
   - Скрипт должен продолжить работу

### Вариант 2: Использовать cookies (альтернатива)

Если автоматическая авторизация не работает, используйте старый способ с cookies:

1. Авторизуйтесь на сайте в обычном браузере
2. Скопируйте cookies из браузера
3. Используйте Edge Function через админ-панель (старый способ)

## Что изменилось в скрипте

1. **Проверка авторизации перед логином** - скрипт сначала проверяет, авторизованы ли вы
2. **Прямой переход на страницу тестов** - если вы авторизованы, страница откроется
3. **Обнаружение ручной авторизации** - скрипт обнаружит, если вы авторизовались вручную
4. **Улучшенная обработка ошибок** - более понятные сообщения об ошибках
5. **Меньшие таймауты** - быстрее определяет проблемы

## Если проблема остается

1. **Проверьте подключение к интернету**
2. **Попробуйте авторизоваться вручную в браузере**
3. **Перейдите на страницу тестов: https://teorica.practicavial.com/permiso/b/tests/tema/all**
4. **Запустите скрипт снова**

## Отладка

Если скрипт все еще не работает:

1. **Запустите с видимым браузером:**
   ```bash
   npm run scrape:practicavial -- --username=ваш-email --password=ваш-пароль --headless=false
   ```

2. **Наблюдайте за процессом:**
   - Что происходит в браузере?
   - На какой странице вы находитесь?
   - Есть ли ошибки на странице?

3. **Проверьте логи в терминале:**
   - Какие сообщения выводит скрипт?
   - На каком этапе возникает ошибка?

## Готово!

Теперь скрипт должен работать лучше и обнаруживать, если вы уже авторизованы.
