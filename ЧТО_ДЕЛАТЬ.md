## Шаг 1: Запустите скрапер

**Только после того, как вы перешли в директорию проекта**, выполните одну из команд:

### Самый простой способ (интерактивный):

```bash
npm run scrape:practicavial
```

Скрипт попросит ввести логин и пароль для PracticaVial.

### Или с параметрами:

```bash
npm run scrape:practicavial -- --username=ваш-email@example.com --password=ваш-пароль
```

### Для отладки (с видимым браузером):

```bash
npm run scrape:practicavial -- --username=ваш-email@example.com --password=ваш-пароль --headless=false
```

## Шаг 2: Дождитесь завершения

Скрипт автоматически:
1. ✅ Откроет браузер
2. ✅ Авторизуется на сайте PracticaVial
3. ✅ Загрузит все темы и тесты
4. ✅ Извлечет все вопросы и ответы
5. ✅ Сохранит в базу данных Supabase
6. ✅ Создаст Excel файл `practicavial-scrape.xlsx`

## Шаг 3: Проверьте результаты

После завершения:
- **Excel файл**: `practicavial-scrape.xlsx` в корне проекта
- **База данных**: Все вопросы в таблице `questions_new`, ответы в `answer_options`

## Если что-то не работает

1. **Проверьте логин и пароль** - они должны быть правильными
2. **Запустите с `--headless=false`** - увидите, что происходит в браузере
3. **Проверьте подключение к интернету**
4. **Убедитесь, что у вас есть премиум доступ** к PracticaVial

## Дополнительные опции

### Скрапинг только определенных тем:

```bash
npm run scrape:practicavial -- --username=ваш-email --password=ваш-пароль --topics=1,2,3
```

### Сохранение в другой файл:

```bash
npm run scrape:practicavial -- --username=ваш-email --password=ваш-пароль --output=my-data.xlsx
```

## Важно!

⚠️ **Для сохранения в базу данных может понадобиться SERVICE_ROLE_KEY**

Если вы видите ошибки при сохранении в базу данных, добавьте в `.env`:

```env
SUPABASE_SERVICE_ROLE_KEY=ваш-service-role-key
```

Получить SERVICE_ROLE_KEY можно в Supabase Dashboard:
1. Откройте проект в Supabase
2. Перейдите в Settings → API
3. Скопируйте `service_role` key (секретный ключ)
