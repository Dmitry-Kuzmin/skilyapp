# ✅ ЧТО ПРОВЕРИТЬ ПРЯМО СЕЙЧАС

## 🔍 Шаг 1: Проверь логи в Supabase (ВАЖНО!)

1. Открой **Supabase Dashboard**: https://supabase.com/dashboard
2. Выбери проект **sdadim-dgt-prep**
3. Перейди в **Edge Functions** → **ai-chat**
4. Нажми **"Logs"** или **"View Logs"**
5. Задай вопрос в приложении (например: "привет")
6. Посмотри, что появляется в логах

### Что должно быть в логах:

**✅ Если функция работает:**
```
[AI Chat] Request received: { method: 'POST', ... }
[AI Chat] Processing request with 1 messages
[AI Chat] Trying Groq API...
[AI Chat] ✅ Groq API success
```

**❌ Если секрет не добавлен:**
```
[AI Chat] Request received: { method: 'POST', ... }
[AI Chat] Processing request with 1 messages
[AI Chat] Groq API key not found, skipping Groq
[AI Chat] Lovable API key not found
[AI Chat] ❌ No AI provider available
```

**❌ Если функция не запускается:**
- В логах ничего не появляется
- Или ошибка при деплое

---

## 🔍 Шаг 2: Проверь секрет GROQ_API_KEY

1. В Supabase Dashboard перейди в **Settings** → **Edge Functions** → **Secrets**
2. Проверь, что есть секрет с названием **`GROQ_API_KEY`**
3. Если его нет - добавь:
   - Нажми **"Add a new secret"**
   - **Name**: `GROQ_API_KEY`
   - **Value**: вставь ключ из Groq Console
   - Нажми **"Save"**

---

## 🔍 Шаг 3: Проверь, что функция задеплоена

1. В **Edge Functions** → **ai-chat**
2. Проверь статус функции:
   - Должно быть **"Active"** или **"Deployed"**
   - Если статус другой - функция не задеплоена

---

## 🔍 Шаг 4: Перезадеплой функцию

1. В функции **ai-chat** нажми **"Edit"** или **"View Code"**
2. Убедись, что код начинается с:
   ```typescript
   import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';
   ```
3. Убедись, что используется `Deno.serve`, а не `serve`
4. Если нужно - скопируй весь код из файла `supabase/functions/ai-chat/index.ts`
5. Нажми **"Deploy"** или **"Save"**

---

## 🔍 Шаг 5: Проверь работу

1. Перезагрузи страницу приложения (Ctrl+R или Cmd+R)
2. Задай вопрос AI помощнику
3. Посмотри логи в Supabase - должно быть видно, что происходит

---

## ❓ Если ничего не помогает

### Вариант 1: Создай функцию заново

1. Удали функцию **ai-chat** (если возможно)
2. Создай новую функцию с названием **ai-chat**
3. Скопируй код из `supabase/functions/ai-chat/index.ts`
4. Задеплой

### Вариант 2: Проверь через CLI

Если у тебя установлен Supabase CLI:
```bash
supabase functions deploy ai-chat
```

### Вариант 3: Проверь конфигурацию

Убедись, что в `supabase/config.toml` есть:
```toml
[functions.ai-chat]
verify_jwt = false
```
